Привет! Мне нужна твоя помощь в составлении комплексного плана коучинга для нашей команды поддержки. Мы отслеживаем все данные по тикетам в хранилище данных ClickHouse центра поддержки, и я хочу превратить это в конкретные планы коучинга для каждого агента.

Сначала изучи файлы Coaching_Framework.pdf и guide.md в твоей рабочей директории. В них подробно описано, как мы определяем соблюдение SLA, как его рассчитывать на основе целевых значений времени ответа из политик SLA, а также как назначать уровни эффективности (performance tiers) и зоны развития (focus areas).

Когда разберёшься с методологией, обратись к хранилищу данных ClickHouse, чтобы получить метрики эффективности по каждому агенту. Поле reporter в таблице tickets идентифицирует каждого агента. Тебе нужно будет соединить (join) данные с таблицей политик SLA (SLA_POLICIES), чтобы получить целевые значения времени ответа (response target hours) для каждого уровня приоритета. Рассчитай для каждого агента процент соблюдения SLA, средний балл удовлетворённости клиентов (CSAT), среднее время ответа, общее количество тикетов и количество решённых тикетов.

После вычисления этих метрик создай Excel-файл под названием Agent_Scorecard.xlsx с тремя листами. Первый лист должен называться Performance Metrics и содержать колонки Agent_Name, Total_Tickets, Resolved_Tickets, SLA_Compliance_Pct, Avg_CSAT и Avg_Response_Hours, по одной строке на агента, отсортированные по алфавиту. Второй лист должен называться Coaching Plan с колонками Agent_Name, Performance_Tier, Coaching_Frequency, Primary_Focus_Area и Secondary_Focus_Area, снова по одной строке на агента, отсортированные по алфавиту. Примени определения уровней (tier) и правила назначения зон развития (focus area) из документа с методологией. Третий лист должен называться Summary с двумя колонками Metric и Value, где показаны Total_Agents, Total_Tickets, Avg_SLA_Compliance, Avg_CSAT и Tier_Distribution в виде текстовой строки, показывающей, сколько агентов в каждом уровне.

Далее мне нужно, чтобы ты создал в Teamly пространство (space) под названием Agent Coaching Tracker. Создай в этом пространстве по одной странице на каждого агента. Заголовок (title) страницы должен совпадать с именем агента. В теле страницы укажи следующие поля: Team со значением General Support, Tier — их уровень эффективности (performance tier), Coaching_Frequency — их частота коучинга согласно методологии, Focus_Area — их основная зона развития (primary focus area) и Notes — краткая сводка по их метрикам, например процент соблюдения SLA и балл CSAT. Каждое поле подпиши его английским названием (Team, Tier, Coaching_Frequency, Focus_Area, Notes), а значение укажи рядом.

Наконец, запланируй сессии коучинга в Google Calendar. Создай по одному событию на каждого агента в течение недели с 16 по 20 марта 2026 года. Каждая сессия должна длиться 30 минут. Озаглавь каждое событие как Coaching Session, после чего в скобках укажи имя агента. Распредели события по неделе, по одному в день, если возможно, начиная с понедельника 16 марта. В описании события укажи уровень эффективности (performance tier) агента и его основную зону развития (primary focus area).
