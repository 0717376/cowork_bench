Вы — профессор, выступающий в роли рецензента на Конференции по машинному обучению 2026. Вам назначены для рецензирования три статьи. Портал рецензирования конференции доступен по адресу http://localhost:30214/review_criteria.html и содержит официальную шкалу оценивания, включая критерии технической обоснованности, новизны, ясности и итоговой рекомендации.

Прочитайте файл Review_Guidelines.pdf в вашей рабочей директории, чтобы получить дополнительный контекст о том, как структурировать рецензии и что конференция ожидает от рецензентов.

Сначала откройте портал рецензирования конференции и внимательно изучите шкалу оценивания. Обратите внимание на шкалы оценок и на то, что означает каждый уровень оценки.

Затем получите информацию о трёх назначенных статьях из базы данных научных статей. Статьи называются "Scaling Laws for Neural Language Models", "Training language models to follow instructions with human feedback" и "OPT: Open Pre-trained Transformer Language Models". Прочитайте полный текст и аннотацию каждой статьи. Дополнительно изучите исходный код LaTeX каждой статьи, чтобы детальнее проанализировать разделы о методологии, рассматривая уравнения, экспериментальную постановку и структуру разделов.

Создайте три отдельных документа Word в вашей рабочей директории — по одному на каждую рецензию.

Первый документ должен называться "Review_Scaling_Laws.docx". Он должен содержать заголовок с названием статьи и авторами. Затем включите разделы: Summary (Краткое описание — 2-3 предложения о вкладе статьи), Technical Soundness (Техническая обоснованность — оценка 1-5 с обоснованием на основе шкалы), Novelty (Новизна — оценка 1-5 с обоснованием), Clarity (Ясность — оценка 1-5 с обоснованием) и Overall Recommendation (Итоговая рекомендация — Accept/Принять, Weak Accept/Слабое принятие, Borderline/На грани, Weak Reject/Слабое отклонение или Reject/Отклонить с кратким обоснованием). Для этой статьи присвойте Technical Soundness оценку 5, Novelty оценку 4 и Clarity оценку 4. Итоговая рекомендация должна быть Accept.

Второй документ должен называться "Review_InstructGPT.docx" — для статьи про RLHF. Используйте ту же структуру. Присвойте Technical Soundness 5, Novelty 5 и Clarity 4. Итоговая рекомендация должна быть Accept.

Третий документ должен называться "Review_OPT.docx" — для статьи OPT. Используйте ту же структуру. Присвойте Technical Soundness 4, Novelty 3 и Clarity 5. Итоговая рекомендация должна быть Weak Accept.

Заголовки разделов и текст обоснований могут быть на русском языке. Однако числовые оценки, метки рекомендаций (Accept / Weak Accept), arxiv ID, имена столбцов и имена файлов должны оставаться точно такими, как указано выше.

Важно: инструмент Word MCP теряет текущую рабочую директорию, поэтому при создании или сохранении каждого .docx передавайте АБСОЛЮТНЫЙ путь к файлу внутри вашей рабочей директории (например, <workspace>/Review_Scaling_Laws.docx).

Создайте таблицу Google Sheet с названием "Conference Review Tracker" с единственным листом "Reviews". Лист должен иметь столбцы: Paper_ID, Paper_Title, Technical_Soundness, Novelty, Clarity, Average_Score, Recommendation, Review_Status. Включите по одной строке для каждой из трёх статей. Paper_ID — это arxiv ID. Average_Score — среднее из трёх оценок, округлённое до 1 десятичного знака. Review_Status должен быть "Completed" для всех трёх. Отсортируйте строки по Average_Score по убыванию.

Когда закончите, просто ответьте без вызова каких-либо инструментов, чтобы обозначить завершение.
