Исследовательский инженер собирает конвейер анализа статей для команды, которая изучает генерацию кода (code generation). Выполните следующие шаги.

Во-первых, найдите в базе arxiv статьи о генерации кода. Должно найтись около четырёх релевантных статей по таким темам, как оценка больших языковых моделей на коде, предобученные модели для кода, генерация кода соревновательного уровня и открытые модели для кода. Для каждой статьи получите полные данные: название (title), авторов (authors) и аннотацию (abstract). В базе есть посторонняя обзорная статья — не включайте её.

Во-вторых, дополните сведения по каждой статье числом цитирований и площадкой публикации (venue) с помощью инструмента scholarly. Запишите citation_count и venue для каждой статьи.

В-третьих, во внутреннем сервисе по адресу http://localhost:30151/api/supplements.json через терминал получите дополнительные метаданные (ссылку на репозиторий с кодом и наборы данных) и при желании учтите их.

В-четвёртых, сохраните результаты в граф знаний memory. Создайте по одной сущности (entity) на каждую статью с типом `paper` и наблюдениями (observations), включающими Title, ArXiv ID, Authors и Citation count. Дополнительно создайте сущность `research_session` (тип `session`) с темой «code generation», числом найденных статей и суммарным числом цитирований, и свяжите её со статьями. Названия сущностей, поля (Title, ArXiv ID, ...) и тему оставляйте на английском.

В-пятых, создайте в рабочей папке файл `paper_data.json` — массив объектов, по одному на релевантную статью. Каждый объект должен содержать поля: `arxiv_id`, `title`, `authors` (список строк-имён), `citation_count` (число) и `abstract`. Включайте только статьи про генерацию кода, без посторонней обзорной.

В-шестых, в рабочей папке уже есть скрипт `analysis_pipeline.py`. Запустите его через терминал (`python analysis_pipeline.py`); он читает `paper_data.json` и пишет `analysis_results.json` со статистикой: `total_papers`, `total_citations`, `average_citations`, `most_cited_paper` (название), `paper_titles`, `paper_arxiv_ids`. При необходимости можете доработать скрипт.

Наконец, создайте файл `pipeline_report.txt` с кратким отчётом на русском: тема исследования, найденные статьи, ключевая статистика и наблюдения о ландшафте исследований по генерации кода.

Названия статей, фамилии авторов, идентификаторы arxiv, имена файлов и названия JSON-полей оставляйте на английском — так задана схема.
