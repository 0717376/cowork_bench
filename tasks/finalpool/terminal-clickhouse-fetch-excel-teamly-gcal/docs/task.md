Вы — руководитель операций поддержки (Отдел поддержки), и вам нужно построить дашборд соответствия SLA, сравнивающий внутренние показатели поддержки с отраслевыми бенчмарками. Цель — проанализировать данные по обращениям, получить внешние бенчмарки, рассчитать уровни соответствия (compliance rate) и опубликовать результаты на нескольких платформах.

Начните с чтения файла SLA_Framework.pdf в вашей рабочей директории — он задаёт внутренние цели SLA и методологию расчёта уровней соответствия. Также прочитайте файл benchmark_config.json, в котором указан URL для получения отраслевых бенчмарков и параметры сравнения.

Получите отраслевые бенчмарки SLA со страницы-панели по адресу http://localhost:30185. На странице есть таблица отраслевых метрик, включая среднее время реакции (average response time), оценку удовлетворённости клиентов (customer satisfaction), долю решений при первом контакте и общий процент соответствия SLA (SLA compliance). Зафиксируйте эти значения для сравнения с вашими внутренними данными.

Из корпоративного облачного хранилища данных ClickHouse запросите базу данных центра поддержки (SUPPORT_CENTER). Получите данные по обращениям, сгруппированные по уровню приоритета (PRIORITY), включая общее число обращений, среднее время реакции в часах и среднюю оценку удовлетворённости клиентов для каждого приоритета. Также запросите таблицу политик SLA, чтобы получить целевое время реакции в часах для каждого уровня приоритета. Получите данные по эффективности агентов, показывающие имя агента (AGENT_NAME), команду (TEAM), число обработанных обращений и среднее время реакции.

С помощью инструментов командной строки напишите и выполните Python-скрипт под именем sla_analyzer.py в вашей рабочей директории. Скрипт должен рассчитать уровни соответствия SLA для каждого уровня приоритета. Обращение считается соответствующим SLA, если его среднее время реакции не превышает целевое значение SLA для данного приоритета. Рассчитайте уровень соответствия в процентах. Затем сравните внутренние сводные метрики с отраслевыми бенчмарками, полученными с панели: вычислите gap (разрыв) по каждой метрике и пометьте, соответствует ли внутреннее значение бенчмарку или отстаёт от него.

Создайте книгу Excel под именем SLA_Compliance_Report.xlsx в вашей рабочей директории с четырьмя листами. Первый лист с именем Ticket_Summary должен содержать столбцы: приоритет, число обращений, среднее время реакции в часах, средняя оценка удовлетворённости, целевое время SLA в часах и уровень соответствия в процентах. Включите по одной строке на каждый уровень приоритета. Второй лист с именем Benchmark_Comparison должен содержать столбцы: название метрики, внутреннее значение, значение отраслевого бенчмарка, gap и статус (status), где статус — "Meets", если внутреннее значение не хуже бенчмарка, и "Below" в противном случае. Включите строки для среднего времени реакции, удовлетворённости клиентов и уровня соответствия SLA. Третий лист с именем Agent_Performance должен содержать столбцы: имя агента, команда, число обработанных обращений, среднее время реакции и признак выполнения целей SLA. Включите по одной строке на каждого агента. Четвёртый лист с именем Monthly_Trend должен содержать столбцы: месяц, всего обращений и оценочный процент соответствия. Включите строки минимум за три последних месяца с правдоподобной динамикой на основе наблюдаемых вами общих паттернов соответствия.

Создайте в Teamly пространство (space) под названием "SLA Compliance Dashboard" и добавьте в него по одной странице (page) на каждую ключевую метрику сравнения из вашего анализа бенчмарков. На каждой странице укажите название метрики (Metric Name), числовое значение (Value), значение бенчмарка (Benchmark), статус (Status) — "Meets" или "Below" — и дату обновления (Updated). Статусы должны совпадать со сравнением на листе Benchmark_Comparison.

Запланируйте ежемесячные совещания по обзору SLA в общем календаре. Создайте три события в первый понедельник апреля, мая и июня 2026 года в 10:00, каждое длительностью полтора часа. Названия событий должны содержать "Monthly SLA Review" и название месяца. Добавьте описание, упоминающее, что совещание посвящено обзору соответствия SLA и плану действий.
