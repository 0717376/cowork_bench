Я ML-исследователь нашей лаборатории и отслеживаю state-of-the-art методы по разным бенчмаркам. Мне нужно собрать сводный трекер, который связывает результаты с лидербордов бенчмарков с лежащими в их основе статьями и их методологиями.

Начните с того, что откройте сайт с лидербордом бенчмарков по адресу http://localhost:30229 и посмотрите текущие методы-лидеры по разным задачам. Лидерборд показывает методы, их оценки (scores) и, где это указано, идентификаторы статей за каждым методом. Зафиксируйте все методы, их оценки, задачи, на которых они оцениваются, и любые ссылки на статьи.

Затем для каждой статьи, упомянутой на лидерборде, найдите полную информацию о ней в базе arxiv: название, авторов и аннотацию. После этого прочитайте LaTeX-исходник каждой статьи, чтобы извлечь дополнительные детали методологии — ключевой технический вклад, использованные датасеты и основные архитектурные решения, описанные в разделах статьи.

Когда вся информация будет собрана, создайте в рабочей директории два результата.

Во-первых, создайте Excel-книгу с именем Method_Benchmark.xlsx с тремя листами. Первый лист должен называться "Leaderboard" со столбцами Task, Method, Score и Paper_ID. Включите все методы с лидерборда, указывая идентификатор статьи, если он связан с методом, или оставляя поле пустым, если у метода нет привязанной статьи. Второй лист должен называться "Method Details" со столбцами Paper_ID, Title, Key_Contribution и Dataset_Used. Для каждой найденной статьи кратко (одним предложением) сформулируйте ключевой вклад на основе LaTeX-исходника и укажите основной упомянутый датасет. Третий лист должен называться "Summary" со столбцами Metric и Value и содержать строки Total_Methods (количество всех методов на лидерборде), Methods_With_Papers (количество методов со связанной статьёй), Total_Tasks (число различных задач-бенчмарков) и Top_Score (наибольшая оценка по всем задачам).

При работе с Excel-инструментом всегда передавайте АБСОЛЮТНЫЙ путь к файлу Method_Benchmark.xlsx внутри рабочей директории агента, иначе файл может быть записан не туда.

Во-вторых, создайте страницу в командной базе знаний Teamly с заголовком "ML Benchmark Method Tracker". Страница должна давать структурированный обзор: заголовок для каждой задачи-бенчмарка, лучшие методы по каждой задаче с их оценками, описания ключевых вкладов из связанных статей и любые наблюдения о трендах по результатам бенчмарков — например, какие архитектуры доминируют или какие подходы наиболее конкурентоспособны.

Пожалуйста, выполните все шаги и убедитесь, что страница в базе знаний содержит содержательный материал, полезный для постоянного отслеживания исследований. Ориентируйтесь на методические рекомендации по сравнению методов и шаблон в рабочей директории для оформления.
