# Политика эскалации по проблемам качества товаров

## Обзор
Когда товар признан имеющим системные проблемы качества по нескольким источникам данных, его необходимо эскалировать в соответствии с классификацией серьёзности.

## Уровни серьёзности

### Critical
Товары, у которых severity score превышает 80-й перцентиль всех выявленных проблемных товаров. Требуют немедленного внимания команды качества (Quality Team) и должны фиксироваться в командной базе знаний со статусом Open.

### High
Товары с severity score не ниже 50. Подлежат рассмотрению в течение 5 рабочих дней.

### Medium
Товары с severity score от 25 до 49. Ежемесячный цикл пересмотра.

### Low
Товары с severity score ниже 25. Ежеквартальный цикл пересмотра.

## Контакты для эскалации
- Руководитель команды качества (Quality Team Lead): quality_lead@company.com
- Команда по продукту (Product Team): product_team@company.com
- Команда поддержки (Support Team): support_team@company.com
- Вице-президент по клиентскому опыту (VP Customer Experience): vp_cx@company.com

## Процесс
1. Выявить проблемные товары по данным магазина (возвраты/сбои заказов) и отзывам клиентов
2. Сопоставить с данными тикетов центра поддержки для дополнительного контекста
3. Вычислить severity score по взвешенной формуле из audit_criteria.json
4. Создать карточки отслеживания для критических позиций
5. Уведомить соответствующие команды по почте со сводкой результатов
