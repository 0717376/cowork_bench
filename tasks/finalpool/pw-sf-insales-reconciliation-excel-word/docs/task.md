Мне нужна помощь со сверкой (реконсиляцией) данных по продажам нашего интернет-магазина на InSales в разрезе регионов. Внешние эталонные (бенчмарк) данные по рынкам опубликованы по адресу http://localhost:30326 — зайди на эту страницу и выгрузи все приведённые там метрики (Region, Market_Size_M, Market_Penetration_Pct).

Затем выгрузи наши внутренние данные из корпоративного хранилища данных ClickHouse и из базы магазина InSales для сравнения. Внутренние показатели нужно агрегировать по тому же первичному измерению — региону (Region): для каждого региона посчитай количество заказов (Order_Count) и выручку (Revenue) в рублях.

С помощью терминала создай и запусти Python-скрипт под названием sf_wc_reconcile_processor.py в рабочей директории. Скрипт должен читать собранные данные из создаваемых тобой JSON-файлов, выполнять анализ и выводить результат в файл sf_wc_reconcile_results.json. Набор регионов в sf_wc_reconcile_results.json должен совпадать с набором регионов в итоговом Excel-файле.

Создай Excel-файл под названием Wc_Reconciliation_Report.xlsx с тремя листами.

Лист Data_Analysis должен содержать основную сравнительную таблицу со столбцами: Region (первичное измерение), Order_Count и Revenue (наши внутренние показатели), Market_Size_M и Market_Penetration_Pct (внешние эталонные значения с дашборда). Отсортируй строки по столбцу Region в алфавитном порядке. В таблице должны присутствовать все пять регионов: Asia Pacific, Europe, Latin America, Middle East, North America.

Лист Metrics должен иметь два столбца — Metric и Value — и суммировать ключевые статистики: как минимум Total_Regions, Total_Orders (сумма Order_Count по всем регионам), Total_Revenue (сумма Revenue по всем регионам) и Avg_Market_Penetration. Значения Total_Orders и Total_Revenue должны быть согласованы с данными листа Data_Analysis (равны соответствующим суммам).

Лист Recommendations должен иметь столбцы Priority, Action и Region и перечислять приоритетные действия на основе анализа отклонений (gap). Следуй правилу из файла analysis_guide.md: регион с наибольшим отрицательным отклонением (gap) получает Priority 1, следующий — Priority 2 и так далее.

Также создай документ Word под названием Wc_Reconciliation_Analysis.docx с тремя разделами: краткое резюме (executive summary / Краткое резюме), ключевые выводы (key findings / Ключевые выводы) и рекомендации (recommendations / Рекомендации). Включи в отчёт фактические числа и регионы.

Сохрани все выходные файлы в рабочей директории. ВАЖНО: при создании файлов Wc_Reconciliation_Report.xlsx и Wc_Reconciliation_Analysis.docx через соответствующие MCP-инструменты (excel и word) передавай АБСОЛЮТНЫЙ путь к файлу внутри рабочей директории, так как эти инструменты не сохраняют текущую рабочую директорию.
