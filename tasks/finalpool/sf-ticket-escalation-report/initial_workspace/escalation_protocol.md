# Протокол эскалации поддержки

## Обзор

Этот документ описывает текущие процедуры эскалации для команды поддержки. Вся обработка заявок должна следовать этим правилам, чтобы обеспечить своевременное разрешение и удовлетворённость клиентов.

## Уровни эскалации

### Уровень 1 — Стандартная обработка
- Все новые заявки начинаются с Уровня 1
- Обрабатываются основной командой поддержки
- Цель: разрешить в рамках сроков SLA

### Уровень 2 — Усиленная поддержка
- Заявки, приближающиеся к пределам SLA (75% от отведённого времени)
- Низкие оценки удовлетворённости клиентов (ниже порога)
- Повторяющиеся проблемы от одного и того же автора

### Уровень 3 — Эскалация руководству
- Нарушения SLA
- Заявки критического приоритета, не разрешённые в пределах порога
- Удовлетворённость клиентов стабильно ниже допустимого уровня

## Рекомендации по времени ответа

Время ответа регулируется правилами эскалации, которые хранятся в системе правил эскалации компании. Актуальные пороговые значения следует получать из API правил.

## Отчётность

Еженедельные отчёты об эскалациях должны включать:
- Сводку по заявкам, приближающимся к порогам SLA или превышающим их
- Анализ заявок с низкой удовлетворённостью по приоритету и типу проблемы
- Анализ авторов заявок для выявления закономерностей
- Рекомендуемые пункты действий для команды поддержки
