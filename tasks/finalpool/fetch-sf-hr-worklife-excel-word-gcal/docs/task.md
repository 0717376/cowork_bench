Мне нужна помощь с анализом баланса работы и личной жизни (sf worklife). Есть внешние эталонные (бенчмарк) данные, которые нужно получить по адресу http://localhost:30339/api/data.json, и извлечь из них релевантные метрики (отраслевые средние зарплаты по отделам).

Затем выгрузите наши внутренние данные из корпоративного хранилища данных ClickHouse (схема HR_ANALYTICS, таблица сотрудников) для сравнения. Сгруппируйте данные по отделам и для каждого отдела вычислите численность сотрудников, среднюю зарплату, средний стаж и среднюю оценку результативности.

С помощью терминала создайте и запустите Python-скрипт под названием sf_worklife_processor.py в рабочей директории, который читает собранные данные из создаваемых вами JSON-файлов, выполняет анализ и записывает результат в sf_worklife_results.json.

Создайте файл Excel под названием Hr_Worklife_Report.xlsx с тремя листами. Первый лист Data_Analysis должен содержать основные сравнительные данные. Используйте отдел в качестве основного измерения и включите столбцы: Department, Employee_Count, Our_Avg_Salary, Avg_Experience, Avg_Performance, Industry_Benchmark и Salary_Gap. Industry_Benchmark — это отраслевая эталонная средняя зарплата по отделу из полученного API, а Salary_Gap = Our_Avg_Salary - Industry_Benchmark (наша внутренняя средняя минус внешний эталон). Отсортируйте строки по столбцу Department в алфавитном порядке.

Второй лист Metrics должен суммировать ключевые метрики и содержать два столбца: Metric и Value. Включите как минимум: Total_Departments (общее число отделов), Total_Employees (общая численность сотрудников) и Avg_Salary_Gap (среднее значение зарплатного разрыва по всем отделам), а также другие сводные статистики.

Третий лист Recommendations должен перечислять приоритетные действия на основе анализа разрывов и содержать столбцы: Priority, Action, Department и Impact. В первую очередь обращайте внимание на отделы с наибольшим отрицательным разрывом.

Также создайте документ Word под названием Hr_Worklife_Analysis.docx с разделами: краткое резюме для руководства (executive summary), ключевые выводы (key findings) и рекомендации (recommendations).

Наконец, запланируйте в календаре встречу с названием "Analysis Review" 14 марта 2026 года с 14:00 до 15:00 UTC.

Сохраните все выходные файлы в рабочей директории. ВАЖНО: при создании файлов .xlsx и .docx через соответствующие MCP-инструменты передавайте АБСОЛЮТНЫЙ путь к файлу внутри рабочей директории, так как эти инструменты не сохраняют текущую рабочую директорию.
