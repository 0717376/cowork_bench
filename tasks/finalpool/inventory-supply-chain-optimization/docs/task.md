Нашей команде по управлению цепочкой поставок нужно оптимизировать товарные запасы: снизить затраты на хранение, сохранив при этом доступность товаров. Сегодняшняя дата — 1 марта 2026 года. Выполни шесть этапов ниже и подготовь итоговые отчёты.

Этап 1. Сбор данных. По адресу http://localhost:30306/api/supplier_catalog.json доступен API каталога поставщиков (на русском, цены в рублях) со сроками поставки, минимальными объёмами заказа и оценкой надёжности по каждому нашему поставщику. Получи эти данные. Затем проверь наш магазин InSales: текущие остатки товаров (stock_quantity), суммарные объёмы продаж (total_sales) и названия товаров. Дополнительно выгрузи историю продаж из хранилища данных ClickHouse (логическая база данных SALES_DW, таблица SALES_DW.PUBLIC.ORDERS) — нам нужны исторические заказы по товарным категориям для оценки спроса.

Этап 2. Прогноз спроса. Оцени дневной темп продаж по каждому товару InSales исходя из 90-дневного окна продаж: Daily_Rate = Total_Sales / 90. Затем рассчитай, на сколько дней хватит текущего остатка: Days_Remaining = Current_Stock / Daily_Rate. Товар требует пополнения, если Days_Remaining меньше 30 (Needs_Restock = "Yes", иначе "No"). По данным ClickHouse SALES_DW агрегируй спрос по каждой категории товаров (CATEGORY): суммарное количество заказанных единиц и число заказов за всю историю.

Этап 3. Точки заказа. На основе сроков поставки из каталога поставщиков и волатильности спроса определи, какие товары нужно дозаказать в первую очередь. Поставщики с большим сроком поставки и низкой надёжностью требуют более раннего размещения заказа.

Этап 4. Через терминал создай и запусти Python-скрипт `inventory_forecaster.py` в рабочей директории. Скрипт должен читать `supplier_data.json` и `product_stock.json` (оба файла тоже создай в рабочей директории на основе полученных данных), вычислять Daily_Rate и Days_Remaining по приведённым выше формулам и выводить результат в `restock_plan.json`. Используй абсолютные пути к файлам рабочей директории при создании и запуске скрипта.

Этап 5. Отчётность. Подготовь Excel-файл `Inventory_Forecast_Report.xlsx` с четырьмя листами:

- Лист `Stock_Status` со столбцами Product, Current_Stock, Total_Sales, Daily_Rate (округлить до 2 знаков), Days_Remaining (округлить до 1 знака) и Needs_Restock ("Yes"/"No"). Отсортируй строки по названию товара (Product) по алфавиту.
- Лист `Supplier_Info` со столбцами Supplier, Lead_Time_Days, Min_Order_Qty и Reliability_Score — по каждому поставщику из каталога.
- Лист `Demand_History` со столбцами Category, Total_Units_Ordered и Order_Count — агрегированные по категориям данные истории заказов из ClickHouse SALES_DW.
- Лист `Restock_Summary` со столбцами Metric и Value со строками Total_Products_Analyzed, Products_Need_Restock, Products_Healthy и Avg_Days_Remaining (округлить до 1 знака).

ВАЖНО: при работе с Excel и Word передавай инструменту АБСОЛЮТНЫЙ путь к файлам `Inventory_Forecast_Report.xlsx` и `Supply_Chain_Risk_Assessment.docx` внутри рабочей директории, так как у этих инструментов может не быть доступа к текущей рабочей директории.

Подготовь документ Word `Supply_Chain_Risk_Assessment.docx` с оценкой рисков цепочки поставок. Документ должен содержать разделы с заголовками «Risk Assessment / Оценка рисков», «Suppliers / Поставщики» и «Contingency Plan / План реагирования», упоминать поставщиков с наибольшим сроком поставки и риски концентрации.

Этап 6. Распространение планов. Создай Google-таблицу с названием «Inventory Tracking» с листом, в шапке которого присутствуют столбцы Product, Current_Stock, Days_Remaining и Needs_Restock, и занеси туда товары для текущего мониторинга остатков. Создай событие в календаре «Inventory Review Meeting» 12 марта 2026 года с 9:00 до 10:00 по UTC с описанием, перечисляющим товары, которые требуют пополнения. Также отправь письмо на адрес procurement@company.com с темой «Urgent Restock Alert», перечислив в теле письма товары, которые требуют немедленного пополнения запасов.
