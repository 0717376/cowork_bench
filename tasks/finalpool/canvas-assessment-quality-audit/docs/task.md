Отдел оценки качества обучения российского университета должен провести аудит качества тестовых заданий (квизов) по всем курсам. Начните с того, что зайдите на портал стандартов качества оценивания по адресу http://localhost:30221 и получите допустимые диапазоны для индекса трудности, индекса дискриминативности, целевые показатели надёжности и минимальное число вопросов. Также перейдите по ссылке на страницу с методикой, чтобы понять, как рассчитывается трудность.

Прочитайте документ с экзаменационной политикой в рабочей директории (Exam_Policy_2026.pdf) — это официальные требования университета к оцениванию, и изучите глоссарий по оцениванию (assessment_glossary.md) для определений психометрических терминов, используемых в аудите.

Затем обратитесь к системе управления обучением (LMS) и соберите данные по каждому квизу всех курсов. Для каждого квиза подсчитайте число вопросов, рассчитайте средний балл по всем сдачам и вычислите индекс трудности. Индекс трудности рассчитывается как средний балл, делённый на максимально возможное число баллов, где максимально возможное число баллов равно points_possible, умноженному на число вопросов. В результате получается значение от 0 до 1, отражающее среднюю долю набранных от максимума баллов.

Названия курсов и квизов берите «как есть» из LMS и не переводите и не изменяйте — записывайте в файлы ровно в том виде, в каком они хранятся в системе.

Создайте файл Excel с именем Assessment_Quality.xlsx с тремя листами. Первый лист с именем "Quiz Overview" должен содержать колонки Course, Quiz, Question_Count, Avg_Score и Avg_Difficulty (индекс трудности как десятичная дробь от 0 до 1). Отсортируйте по названию курса, затем по названию квиза. Второй лист с именем "Flagged Items" должен содержать только те квизы, чей средний индекс трудности выходит за пределы допустимого диапазона с портала стандартов, с колонками Course, Quiz, Question_Count, Difficulty_Index и Issue (значение "Too Easy", если выше верхнего порога, или "Too Hard", если ниже нижнего порога). Третий лист с именем "Summary" должен содержать строки Total_Quizzes, Total_Questions (сумма всех чисел вопросов), Flagged_Quizzes и Flag_Rate (процент квизов, попавших в список помеченных).

ВАЖНО: имена файлов, имена листов, имена колонок и значения Issue ("Too Easy" / "Too Hard") должны оставаться строго на английском языке в указанном виде — не переводите их. На русском языке пишется только повествовательный текст отчёта Word (см. ниже).

Также создайте документ Word с именем Assessment_Report.docx — повествовательный отчёт об аудите качества на русском языке. Включите: краткое резюме (executive summary) с общим числом проанализированных квизов и числом выявленных пометок; раздел с методикой, поясняющий расчёт индекса трудности; раздел с результатами, где обсуждаются закономерности среди помеченных квизов; и раздел с рекомендациями по улучшению качества оценивания.

Примечание: MCP-инструменты для Excel и Word теряют текущую рабочую директорию, поэтому при создании и сохранении файлов Assessment_Quality.xlsx и Assessment_Report.docx передавайте им ПОЛНЫЙ (абсолютный) путь к файлам внутри вашей рабочей директории.
