Наш университет хочет дополнить курсы по программированию и вычислительным дисциплинам подобранными видеоматериалами с образовательного технологического YouTube-канала.

Из системы управления обучением (Canvas) получи все активные курсы, в названии которых встречается одно из ключевых слов: «вычисления» или «данных». Затем получи все доступные видео канала Fireship из базы данных видео.

Для каждого найденного курса подбери ровно два наиболее релевантных видео Fireship, сопоставляя ключевые слова категории курса с заголовками видео (поиск подстроки без учёта регистра). Используй следующие списки ключевых слов:
- если в названии курса есть слово «вычисления» (категория Computing), ключевые слова: «code», «computing», «tech», «programming»;
- если в названии курса есть слово «данных» (категория Data), ключевые слова: «data», «database», «code», «analytics».

Ключевые слова в каждом списке перечислены в порядке убывания приоритета. Для каждого видео совпавшим считается самое приоритетное (первое по списку) ключевое слово, встречающееся в его заголовке. Ранжируй подходящие видео сначала по приоритету совпавшего ключевого слова, а при равном приоритете — по числу просмотров (View_Count) по убыванию; возьми два первых видео из этого ранжирования.

Для каждого совпадения зафиксируй конкретное ключевое слово (на английском, как оно встречается в заголовке видео), вызвавшее совпадение.

Создай файл Excel с именем Curriculum_Video_Map.xlsx в рабочей директории. Файл должен содержать два листа.

Первый лист должен называться Course_Videos и содержать следующие столбцы в указанном порядке: Course_Name, Course_Code, Video_ID, Video_Title, View_Count, Duration_Min, Relevance_Match. Course_Code — это числовой идентификатор курса из системы управления обучением. Duration_Min — длительность видео в минутах, округлённая до одного знака после запятой. Relevance_Match — английская строка ключевого слова, по которому произошло совпадение (например, «data» или «code»). Отсортируй данные по Course_Name по возрастанию, затем внутри каждого курса по View_Count по убыванию. У каждого курса должно быть ровно две строки с видео.

Второй лист должен называться Summary и содержать ровно три строки со следующими метками в первом столбце: Total_Courses_Mapped, Total_Videos_Recommended и Avg_Views_Recommended. Второй столбец каждой строки содержит соответствующее числовое значение. Total_Courses_Mapped — количество различных курсов, у которых есть хотя бы одно совпадение с видео. Total_Videos_Recommended — общее количество строк на листе Course_Videos без учёта заголовка. Avg_Views_Recommended — среднее число просмотров по всем рекомендованным видео, округлённое до ближайшего целого.

Создай страницу в корпоративной базе знаний teamly с заголовком «Tech Course Video Resources». На странице должен быть раздел для каждого сопоставленного курса: название курса в виде заголовка и перечень рекомендованных заголовков видео под ним. Страницу можно разместить в одном из существующих пространств.
