Наш отдел поддержки клиентов хочет запустить структурированную инициативу по повышению удовлетворённости на основе недавней обратной связи. В I квартале 2026 года был проведён опрос удовлетворённости клиентов, и его результаты доступны через REST API по адресу http://localhost:30205/api/survey_results.json. Опрос включает 20 ответов клиентов с оценками общей удовлетворённости, времени отклика, качества решения и профессионализма агентов, а также уровень приоритета обращения и свободные текстовые комментарии.

Мы также ведём систему управления обращениями (тикетами) в облачном хранилище данных ClickHouse с историческими данными поддержки, охватывающими более 31 000 решённых обращений. Каждое обращение содержит уровень приоритета, время отклика в часах и оценку удовлетворённости клиента.

Сначала получите результаты опроса с указанного API-эндпоинта. Затем выполните запрос к системе ClickHouse для получения агрегированных метрик с группировкой по уровню приоритета (High, Medium, Low): среднее время отклика в часах и средняя оценка удовлетворённости клиентов. Также получите общее количество обращений.

Создайте в рабочей директории файл Excel с именем "Support_Satisfaction_Analysis.xlsx" с четырьмя листами.

Первый лист с названием "Survey Results" должен содержать все 20 ответов опроса со столбцами Respondent_ID, Overall_Satisfaction, Response_Time_Rating, Resolution_Quality_Rating, Agent_Professionalism, Priority, Issue_Type и Comment.

Второй лист с названием "Survey Summary" должен иметь столбцы Metric и Value со следующими строками: Total_Respondents (20), Avg_Overall_Satisfaction (среднее всех оценок общей удовлетворённости), Avg_Response_Time_Rating, Avg_Resolution_Quality, Avg_Agent_Professionalism, Lowest_Rated_Priority (уровень приоритета с наименьшей средней общей удовлетворённостью по опросу) и Highest_Rated_Priority (уровень приоритета с наибольшей средней общей удовлетворённостью по опросу).

Третий лист с названием "Ticket System Comparison" должен иметь столбцы Priority, Survey_Avg_Satisfaction, Ticket_Avg_Satisfaction, Ticket_Avg_Response_Hours и Ticket_Count. Этот лист сопоставляет удовлетворённость по опросу в разрезе приоритетов с фактическими данными системы обращений ClickHouse в разрезе приоритетов.

Четвёртый лист с названием "Improvement Areas" должен выделять направления, требующие внимания. Он должен иметь столбцы Area, Current_Score, Target_Score и Gap. Включите строки для каждой метрики (Response Time, Resolution Quality, Agent Professionalism), где среднее значение по опросу ниже 4.0. Целевое значение (Target_Score) должно быть 4.5 для всех метрик. Разрыв (Gap) — это целевое значение минус текущее значение.

Далее создайте форму (Google Form) для постоянного сбора обратной связи. Форма должна называться "Customer Support Feedback Form" и включать следующие вопросы: оценка общей удовлетворённости (шкала 1-5), оценка времени отклика (шкала 1-5), оценка качества решения (шкала 1-5), выбор уровня приоритета (High, Medium, Low) и поле для свободных комментариев.

Запланируйте четыре ежеквартальные встречи-обзора в Google Calendar на 2026 год: 15 марта, 15 июня, 15 сентября и 15 декабря, с 10:00 до 11:30 в часовом поясе America/New_York. Заголовок каждого события должен быть "Support Satisfaction Review - Q[N] 2026", где N — номер квартала (от 1 до 4).

Сохраните все выходные файлы в рабочую директорию.
