Учебному отделу нужен анализ учебной нагрузки преподавателей за текущий семестр. В рабочей директории есть PDF-документ Course_Workload_Standards.pdf с внутренними нормативами по максимальному числу заданий на курс и допустимым диапазонам нагрузки. Сначала прочитайте этот PDF, чтобы понять нормативы.

Затем извлеките данные из системы управления обучением (LMS). Получите список всех курсов и для каждого курса узнайте количество заданий (assignments), количество тестов (quizzes) и число зачисленных студентов. Нужно понять, какие курсы имеют наибольшую нагрузку по суммарному числу элементов (задания плюс тесты) и по общему числу студентов.

С помощью инструментов командной строки создайте и запустите Python-скрипт workload_analyzer.py в рабочей директории. Скрипт должен читать данные о курсах из course_data.json (который вы создаёте на основе данных из LMS) и нормативы нагрузки из standards.json (извлечённые из PDF), вычислять метрики нагрузки по предметам (группируйте курсы по названию предмета, удаляя семестр в скобках) и выводить файл workload_analysis.json. Метрики для расчёта: среднее число заданий на курс, общее число студентов и факт превышения нормативов.

Создайте облачную таблицу "Faculty_Workload_Tracker" с двумя листами. Первый лист Course_Workload должен содержать столбцы Course_Name, Subject, Semester, Assignment_Count, Quiz_Count, Total_Items и Enrollment. Включите все курсы, отсортированные по алфавиту по Course_Name.

Второй лист Subject_Summary должен содержать столбцы Subject, Num_Courses, Avg_Assignments, Total_Enrollment, Workload_Rating (значение "Heavy", если среднее число заданий 12 или более; "Moderate", если от 8 до 11; "Light", если меньше 8). Отсортируйте по алфавиту по Subject.

Создайте Word-документ Faculty_Workload_Report.docx с заголовком "Faculty Workload Analysis Report" и разделами: Overview (общее число курсов, общее число заданий, общее число зачислений), Subject Analysis (разбор нагрузки по каждому предмету), High Workload Areas (предметы с рейтингом Heavy) и Recommendations (минимум 3 практических рекомендации, например перераспределение заданий или привлечение ассистентов преподавателя). Ссылайтесь на конкретные числа из анализа. ВАЖНО: при сохранении документа через инструмент Word передавайте АБСОЛЮТНЫЙ путь к файлу Faculty_Workload_Report.docx внутри рабочей директории (инструмент Word не сохраняет текущую директорию).

Отправьте два письма. Первое — на адрес department-chairs@university.edu с темой "Faculty Workload Analysis Complete", в котором кратко указано общее число проанализированных курсов, предметы с рейтингом Heavy и главная рекомендация. Второе — на адрес academic-affairs@university.edu с темой "Workload Standards Compliance Report", где отмечено, какие предметы соответствуют нормативам, а какие их превышают.
