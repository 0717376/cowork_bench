Команде поддержки нужно подготовить программу сбора обратной связи от клиентов на основе реальных данных по обращениям из корпоративного облачного хранилища данных ClickHouse. В хранилище есть база данных центра поддержки с записями об обращениях: тип проблемы, приоритет, время отклика и оценка удовлетворённости клиентов.

Начните с запроса к хранилищу данных, чтобы определить пять самых частых типов проблем по общему количеству обращений из таблицы SUPPORT_CENTER.PUBLIC.TICKETS. Для каждого из этих пяти типов проблем зафиксируйте количество обращений, среднее время отклика в часах и среднюю оценку удовлетворённости клиентов.

Затем создайте Google-форму с названием "Customer Service Improvement Survey" и описанием на русском языке, например: "Помогите нам улучшить качество поддержки, поделившись отзывом о вашем недавнем обращении." Форма должна содержать не менее пяти вопросов, и ВСЕ вопросы должны быть обязательными (required). Включите:
- короткий текстовый вопрос, запрашивающий электронную почту клиента (в заголовке вопроса должно присутствовать слово "email");
- вопрос с выбором одного варианта о том, с каким типом проблемы столкнулся клиент, где вариантами выступают пять самых частых типов проблем (используйте те же названия типов, что и в хранилище данных);
- вопрос с выбором одного варианта с просьбой оценить общий уровень удовлетворённости (overall satisfaction) по шкале от 1 до 5;
- текстовый вопрос с просьбой описать, что можно улучшить (suggestion/feedback), в свободной форме;
- текстовый вопрос для любых дополнительных комментариев или предложений в свободной форме.

Используйте инструменты сервера forms: list_forms, create_form, add_text_question, add_multiple_choice_question, get_form. Обратите внимание: сервер форм поддерживает только текстовые вопросы (textQuestion) и вопросы с выбором варианта (choiceQuestion); отдельного типа "флажки/checkbox" нет, поэтому вопросы с вариантами оформляйте как выбор варианта.

Далее создайте книгу Excel с именем файла Support_Analysis.xlsx в рабочей директории. В ней должно быть два листа.

Первый лист назовите "Issue Summary" со столбцами Issue_Type, Ticket_Count, Avg_Response_Time и Avg_Satisfaction. Заполните его ВСЕМИ типами проблем (по одной строке на каждый тип), отсортированными по количеству обращений (Ticket_Count) по убыванию. Avg_Response_Time — среднее время отклика в часах, округлённое до двух знаков; Avg_Satisfaction — средняя удовлетворённость, округлённая до двух знаков.

Второй лист назовите "Priority Breakdown" со столбцами Priority, Count и Avg_Satisfaction. Заполните его по одной строке на каждый уровень приоритета. Значения приоритета оставляйте на английском, как в хранилище: Critical, High, Medium, Low. Count — количество обращений данного приоритета; Avg_Satisfaction — средняя удовлетворённость, округлённая до двух знаков.

Создайте документ Word с именем файла Improvement_Report.docx в рабочей директории. ВАЖНО: при работе с инструментом Word передавайте АБСОЛЮТНЫЙ путь к файлу .docx внутри рабочей директории. Документ должен включать заголовок (например, "Support Service Improvement Report") и не менее четырёх разделов с заголовками (стиль Heading):
- "Executive Summary" (Краткое резюме) — раздел, в котором упоминается общее число проанализированных обращений и общая средняя оценка удовлетворённости;
- "Issue Analysis" (Анализ проблем) — раздел с обсуждением самых частых типов проблем и их оценок удовлетворённости;
- "Recommendations" (Рекомендации) — раздел с конкретными действиями по каждому из пяти самых частых типов проблем;
- "Conclusion" (Заключение) — завершающий раздел.

В тексте документа должны упоминаться как минимум три самых частых типа проблем и общее число обращений. Текст разделов пишите на русском языке, но заголовки разделов оставьте такими, чтобы они содержали указанные английские формулировки (Executive Summary, Issue Analysis, Recommendations, Conclusion) — допускается дублировать русский перевод рядом.

Наконец, отправьте письмо на адрес quality-team@company.com с темой, содержащей слова "Support Analysis" (например: "Support Analysis Report"). Тело письма (можно на русском языке) должно содержать сводку ключевых выводов: общее число обращений, самый частый тип проблемы по объёму и общую среднюю оценку удовлетворённости. Упомяните в теле как минимум два-три самых частых типа проблем.
