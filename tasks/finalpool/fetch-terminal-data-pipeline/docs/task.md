Команда инженеров данных недавно получила сырые дампы ответов API из двух внутренних систем. Файлы `sales_api_response.json` и `inventory_api_response.json` уже находятся в вашей рабочей директории. Файл продаж содержит записи о заказах с полями `order_id`, `product_id`, `product_name`, `quantity`, `unit_price` и `order_date`. Файл склада содержит записи об остатках товаров с полями `product_id`, `product_name`, `current_stock`, `reorder_point` и `warehouse`. Оба файла могут содержать дублирующиеся записи и непоследовательный регистр в названиях товаров. Выручка считается в рублях.

В вашей рабочей директории также есть пустой файл памяти `memory/memory.json` для отслеживания прогресса.

Выполните следующие шаги.

Во-первых, прочитайте оба JSON-файла, чтобы понять их структуру и содержимое.

Во-вторых, напишите Python-скрипт под названием `clean_data.py`, который выполняет следующие операции очистки. Для данных продаж удалите дублирующиеся записи на основе `order_id`, сохраняя первое вхождение каждой. Для данных склада удалите дублирующиеся записи на основе `product_id`, сохраняя первое вхождение. Приведите все значения `product_name` к виду с заглавными буквами (title case) в обоих наборах данных. Для каждого уникального товара в данных продаж рассчитайте суммарную выручку как `quantity`, умноженное на `unit_price`, по всем заказам этого товара. Скрипт должен создать два файла: `cleaned_sales.json` с дедуплицированными записями продаж и `cleaned_inventory.json` с дедуплицированными записями склада. Запустите скрипт после его написания.

В-третьих, сохраните в памяти заметку с кратким описанием того, что вы сделали к данному моменту, включая количество уникальных записей продаж и записей склада после очистки.

В-четвёртых, создайте Excel-файл под названием `Data_Pipeline_Report.xlsx` в вашей рабочей директории с тремя листами. При создании файла через инструмент Excel укажите АБСОЛЮТНЫЙ путь к файлу `Data_Pipeline_Report.xlsx` внутри вашей рабочей директории, так как инструмент может не сохранять текущую директорию.

Первый лист должен называться `Sales Analysis` со столбцами `Product_Name`, `Total_Quantity_Sold`, `Total_Revenue` и `Avg_Unit_Price`. Должна быть одна строка на товар, показывающая суммарное проданное количество по всем заказам, суммарную выручку (сумму `quantity`, умноженного на `unit_price`, для каждого заказа) и среднюю цену за единицу. Включайте только товары, которые присутствуют в очищенных данных продаж.

Второй лист должен называться `Inventory Status` со столбцами `Product_Name`, `Current_Stock`, `Reorder_Point` и `Status`. `Status` должен быть `LOW`, если текущий остаток меньше или равен точке дозаказа, и `OK` в противном случае. Включите все товары из очищенных данных склада.

Третий лист должен называться `Combined View` со столбцами `Product_Name`, `Total_Sold`, `Current_Stock`, `Revenue` и `Stock_Status`. Этот лист объединяет данные из анализа продаж и статуса склада. `Total_Sold` — суммарное проданное количество для этого товара. `Current_Stock` и `Stock_Status` берутся из данных склада. `Revenue` — суммарная выручка для этого товара. Включите все товары, которые присутствуют хотя бы в одном из наборов данных.

В-пятых, создайте Google Sheet с названием `Pipeline Dashboard`, содержащую те же данные, что и лист `Combined View` из Excel-файла.
