Вы — диетолог, который составляет план здорового питания для корпоративной программы благополучия сотрудников. Вам нужно собрать рецепты, сравнить их с рекомендуемыми суточными нормами питания и подготовить итоговый документ с планом питания.

Сначала прочитайте файл Wellness_Program_Brief.pdf в вашей рабочей директории — в нём описаны цели корпоративной программы благополучия и требования к рациону.

Соберите рецепты из кулинарной базы (MCP `kulinar`). Используйте инструменты `mcp_kulinar_getAllRecipes`, `mcp_kulinar_getRecipesByCategory` и `mcp_kulinar_getRecipeById`. В базе доступны категории: салат, закуска, суп, горячее, гарнир, выпечка, десерт, напиток. Распределите блюда по трём приёмам пищи:
- завтрак: не менее трёх блюд (например, из категорий выпечка, десерт, гарнир — каши, сырники, блины, ватрушки);
- обед: не менее трёх блюд (например, из категорий суп, салат, закуска);
- ужин: не менее трёх блюд (например, из категорий горячее, гарнир).
Для каждого выбранного блюда получите его полные данные, включая ингредиенты и шаги приготовления. Всего у вас должно быть не менее девяти рецептов по трём приёмам пищи.

Откройте справочную страницу с нормами питания по адресу http://localhost:30404/nutrition_standards.html. На этой странице приведена таблица рекомендуемых суточных значений по ключевым нутриентам: калории, белки, углеводы, жиры, клетчатка и натрий. Извлеките все рекомендуемые суточные значения с этой страницы (читайте их честно со страницы, не придумывайте).

Напишите и запустите Python-скрипт nutrition_calculator.py в вашей рабочей директории. Скрипт должен задавать оценки питательной ценности для каждого собранного рецепта (можно использовать разумные оценки на основе ингредиентов рецепта: например, блюдо на основе риса может содержать примерно 400 калорий, 12 г белка, 65 г углеводов, 8 г жиров, 3 г клетчатки и 800 мг натрия). Для каждого приёма пищи (завтрак, обед, ужин) выберите блюдо с наиболее сбалансированным профилем питания. Скрипт должен рассчитать суммарную суточную питательную ценность при употреблении по одному блюду на каждый приём пищи и сравнить её с рекомендуемыми суточными значениями со страницы норм. Выведите процент покрытия рекомендуемого суточного значения по каждому нутриенту.

Создайте облачную таблицу с названием "Nutrition Dashboard". Добавьте лист "Recipe Comparison" со столбцами Recipe_Name, Meal_Type, Calories, Protein_g, Carbs_g, Fat_g, Fiber_g и Sodium_mg. Включите по одной строке для каждого из девяти или более собранных рецептов. Добавьте второй лист "Daily Plan" со столбцами Meal, Selected_Recipe, Calories, Protein_g, Carbs_g, Fat_g, Fiber_g, Sodium_mg и Pct_Daily_Calories. Включите три строки для завтрака, обеда и ужина с выбранными блюдами, а также итоговую строку (totals).

Примечание по названиям: название таблицы, имена листов ("Recipe Comparison", "Daily Plan"), все заголовки столбцов, имя скрипта nutrition_calculator.py и имя документа Wellness_Diet_Plan.docx должны оставаться на английском в точности так, как указано выше.

Создайте документ Word с именем Wellness_Diet_Plan.docx в вашей рабочей директории. У документа должен быть заголовок "Программа корпоративного благополучия: план питания". Документ должен содержать:
- раздел с рекомендуемыми суточными нормами питания и значениями со справочной страницы (калории, белки, углеводы, жиры, клетчатка, натрий);
- раздел со списком всех рассмотренных рецептов, сгруппированных по приёму пищи (завтрак, обед, ужин);
- раздел с рекомендуемым суточным планом питания, где показаны выбранные блюда на завтрак, обед и ужин вместе с их профилями питания;
- завершающий раздел с анализом того, насколько предложенный план покрывает суточные рекомендуемые значения, и какие нутриенты в дефиците и требуют восполнения (gap-анализ).

Важно: инструмент работы с Word теряет текущую рабочую директорию, поэтому при создании/сохранении документа передавайте АБСОЛЮТНЫЙ путь к файлу Wellness_Diet_Plan.docx внутри вашей рабочей директории.
