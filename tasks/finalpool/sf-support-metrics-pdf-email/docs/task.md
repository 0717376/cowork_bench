Привет! Помоги, пожалуйста, подготовить отчёт по метрикам поддержки на основе нашей базы тикетов. Мне нужно проанализировать работу центра поддержки и поделиться результатами с командой.

Запроси таблицу тикетов из базы данных центра поддержки в ClickHouse (логическая база `SUPPORT_CENTER`, схема `sf_data`, таблица `SUPPORT_CENTER__PUBLIC__TICKETS`), чтобы собрать подробный отчёт с метриками.

Создай PDF-файл с именем `Support_Metrics.pdf` в рабочей директории со следующим содержимым.

Заголовок документа: «Support Metrics Report».

Во-первых, раздел «Overview» (Обзор), где указаны: Total Tickets (общее количество всех тикетов), среднее время ответа Average Response Time в часах, округлённое до 2 знаков после запятой, и средняя удовлетворённость клиентов Average Customer Satisfaction, округлённая до 2 знаков после запятой.

Во-вторых, раздел «Metrics by Issue Type» (Метрики по типу обращения) с таблицей. Колонки: Issue Type (тип обращения), Tickets (количество), Avg Response (hrs) — среднее время ответа в часах, округлённое до 2 знаков, и Avg Satisfaction — средняя удовлетворённость, округлённая до 2 знаков. Сортировка по количеству тикетов по убыванию. Используй значения типов обращений ровно так, как они хранятся в базе данных.

В-третьих, раздел «Metrics by Priority» (Метрики по приоритету) с таблицей. Колонки: Priority (приоритет), Tickets (количество), Avg Response (hrs) — среднее время ответа в часах, округлённое до 2 знаков, и Avg Satisfaction — средняя удовлетворённость, округлённая до 2 знаков. Сортировка по Priority в алфавитном порядке.

Когда PDF готов, отправь его письмом менеджеру поддержки на адрес `manager@supportcenter.com` с темой «Monthly Support Metrics Report» и кратким текстом, в котором упомянуты общее количество тикетов и средняя оценка удовлетворённости.
