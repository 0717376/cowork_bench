Вы — специалист по планированию запасов интернет-магазина InSales. Вам нужно спрогнозировать, когда товары начнут заканчиваться на складе, и запланировать даты повторного заказа у поставщиков с учётом их сроков поставки.

Начните с чтения документа политики повторного заказа (reorder_policy.pdf) в вашем рабочем каталоге. В нём описаны уровень страхового запаса, горизонт планирования и пошаговый расчёт того, какие товары нуждаются в повторном заказе и когда. Также есть шаблон таблицы скорости продаж (sales_velocity.xlsx), который вы заполните данными из магазина.

Получите из магазина InSales все опубликованные товары (published), у которых текущий остаток на складе больше нуля. Для каждого товара зафиксируйте его Product_ID, Product_Name, Category, текущий остаток (Current_Stock) и обычную цену. Также получите все заказы со статусом completed и processing, чтобы рассчитать скорость продаж. Для каждого товара определите общее количество проданных единиц по всем заказам со статусами completed и processing, изучив позиции заказов (line items).

Рассчитайте количество дней, охваченных историей заказов, как разницу между самой ранней и самой поздней датами заказов со статусами completed и processing. Затем вычислите среднесуточные продажи (Avg_Daily_Sales) для каждого товара, разделив общее количество проданных единиц на количество дней в этом периоде.

Запросите сроки поставки поставщиков по эндпоинту API http://localhost:30202/api/lead_times.json, который возвращает сроки поставки в днях для каждой категории товаров. Обратите внимание: ключами в этом JSON являются названия категорий товаров так, как они хранятся в магазине, поэтому сопоставление товаров с их сроком поставки выполняется по полю Category товара.

Примените расчёты политики повторного заказа. Для каждого товара вычислите эффективный остаток как текущий остаток минус страховой запас в 5 единиц, но не менее нуля. Рассчитайте количество дней до достижения страхового запаса (Days_Until_Safety_Stock), разделив эффективный остаток на среднесуточные продажи. Если количество дней до достижения страхового запаса составляет 90 дней или меньше и у товара есть какая-либо история продаж, то он нуждается в повторном заказе. Дата повторного заказа рассчитывается как «сегодня плюс количество дней до достижения страхового запаса минус срок поставки поставщика для данной категории». Если рассчитанная дата повторного заказа выпадает на сегодня или раньше, запланируйте её на завтра.

Создайте Excel-файл с именем Inventory_Forecast.xlsx в вашем рабочем каталоге с двумя листами. Первый лист с названием "Stock Analysis" должен содержать колонки Product_ID, Product_Name, Category, Current_Stock, Avg_Daily_Sales, Days_Until_Safety_Stock, Lead_Time, Needs_Reorder и Reorder_Date. Включите все товары с остатком больше нуля. Помечайте Needs_Reorder значением Yes или No, а Reorder_Date устанавливайте равным рассчитанной дате либо N/A для товаров, не нуждающихся в повторном заказе. Второй лист с названием "Reorder Schedule" должен содержать только товары, нуждающиеся в повторном заказе, с колонками Product_Name, Category, Current_Stock, Reorder_Date и Lead_Time_Days.

Для каждого товара, нуждающегося в повторном заказе, создайте событие в Google Calendar на рассчитанную дату повторного заказа с 09:00 до 10:00. Установите заголовок (summary) события как "Reorder: " с последующим названием товара, а в описание события включите текущий остаток на складе и срок поставки.

Важно: при работе с Excel-инструментом передавайте АБСОЛЮТНЫЙ путь к файлу Inventory_Forecast.xlsx внутри вашего рабочего каталога, поскольку относительные пути могут не разрешиться корректно.

Когда все задачи будут выполнены, просто завершите работу без вызова каких-либо инструментов.
