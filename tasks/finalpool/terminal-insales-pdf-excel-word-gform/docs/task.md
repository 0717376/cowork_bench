Вы — аналитик по контролю качества, проводящий аудит качества товаров для интернет-магазина электроники на платформе InSales. Ваша цель — проанализировать отзывы о товарах и закономерности возвратов, выявить проблемы с качеством, подготовить отчёты по аудиту и создать опрос обратной связи для поставщиков.

Начните с чтения файла Quality_Standards.pdf в вашей рабочей директории. В нём определены допустимые пороги доли брака, минимальные требования к среднему рейтингу и контрольные показатели по объёму отзывов для каждой категории товаров. Также прочитайте файл audit_scope.json, в котором указаны категории товаров, включаемые в аудит, и пороговые значения рейтинга, при которых возникают флаги качества.

Из магазина InSales извлеките данные по отзывам о товарах по всем категориям. Вам нужны оценки отзывов, привязка к товарам и разбивка по категориям. Также выгрузите данные по заказам, включая статусы заказов (order status), суммы и любую связанную информацию о возвратах. В магазине шесть категорий товаров, и вы должны проанализировать все из них.

С помощью инструментов командной строки напишите и выполните Python-скрипт под названием quality_audit.py в вашей рабочей директории. Скрипт должен выполнять следующий анализ. Для каждой категории товаров рассчитайте количество товаров, общее количество отзывов и средний рейтинг (avg_rating). Вычислите процент отзывов с оценкой ниже трёх звёзд из пяти (pct_below_3stars). Для данных по заказам и возвратам рассчитайте количество и общую сумму для каждого статуса заказа, уделяя особое внимание заказам со статусами refunded и cancelled. Рассчитайте показатель качества (quality_score) для каждой категории по шкале от нуля до ста по формуле: средний рейтинг, делённый на пять и умноженный на сто, затем уменьшенный на удвоенный процент низких отзывов, то есть (avg_rating / 5) * 100 - 2 * pct_below_3stars. Назначьте уровни риска (risk level) на основе этих показателей: высокий риск (High), если показатель ниже 70; средний (Medium), если от 70 до 85; низкий (Low), если выше 85.

Создайте книгу Excel под названием Product_Quality_Audit.xlsx в вашей рабочей директории с четырьмя листами. Первый лист с именем Review_Summary должен содержать столбцы для названия категории, количества товаров, среднего рейтинга, общего количества отзывов и процента отзывов с оценкой ниже трёх звёзд. Включите по одной строке для каждой из шести категорий товаров. Второй лист с именем Refund_Analysis должен содержать столбцы для статуса заказа, количества заказов, общей суммы и процента от общего числа заказов. Включите по одной строке для каждого статуса заказа, найденного в данных. Третий лист с именем Quality_Scorecard должен содержать столбцы для названия категории, показателя качества и уровня риска. Включите по одной строке на категорию. Четвёртый лист с именем Survey_Questions должен содержать столбцы для номера вопроса, текста вопроса и типа вопроса. Перечислите не менее пяти вопросов, подходящих для опроса обратной связи по качеству для поставщиков, охватывающих такие темы, как доля брака, время отклика, планы по улучшению качества, стандарты упаковки и общая удовлетворённость.

Создайте документ Word под названием Quality_Audit_Report.docx в вашей рабочей директории. Документ должен содержать заголовок, абзац исполнительного резюме (executive summary), описывающий общие выводы, раздел с анализом отзывов по категориям, выделяющий категории с тревожными показателями качества, раздел о закономерностях возвратов с указанием доли возвращённых заказов и раздел рекомендаций с не менее чем тремя конкретными действиями по улучшению качества.

Наконец, создайте форму опроса обратной связи под названием «Обратная связь по качеству поставщиков» (Supplier Quality Feedback) с описанием, упоминающим оценку качества товаров. Форма должна содержать не менее пяти вопросов, соответствующих тем, которые перечислены на листе Survey_Questions в вашем файле Excel. Включите разные типы вопросов, такие как шкалы оценок, множественный выбор и открытые текстовые вопросы.

Важно: инструменты Word и Excel могут терять текущую рабочую директорию, поэтому при работе с файлами Product_Quality_Audit.xlsx и Quality_Audit_Report.docx всегда передавайте АБСОЛЮТНЫЙ путь к файлу внутри вашей рабочей директории.
