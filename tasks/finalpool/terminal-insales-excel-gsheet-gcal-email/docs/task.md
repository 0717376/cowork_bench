Вы — менеджер по управлению запасами, отвечающий за контроль уровня товарных остатков и координацию пополнения склада для интернет-магазина. Ваша цель — проанализировать текущие складские запасы, спрогнозировать спрос, сформировать отчёты и обеспечить своевременное планирование закупок.

Начните с прочтения файла Inventory_Policy.pdf в вашей рабочей директории — в нём описаны пороги для повторного заказа и ожидаемые сроки поставки. Также изучите файл warehouse_config.json, содержащий ограничения по вместимости склада и контактные данные поставщика.

Запросите у магазина InSales полный каталог товаров. Для каждого товара получите название (product_name), категорию (category), обычную цену (price), количество на складе (stock_qty), статус наличия (stock_status) и суммарное число продаж (total_sales). Эти данные понадобятся для оценки состояния запасов и расчёта приоритетов повторного заказа.

Напишите Python-скрипт под названием demand_forecast.py в рабочей директории и выполните его с помощью инструментов командной строки. Скрипт должен рассчитать простую метрику скорости спроса для каждого товара: разделите суммарное число продаж на условный период продаж в 180 дней, чтобы получить дневной темп продаж. Затем вычислите запас в днях (days_of_supply), разделив текущий остаток на дневной темп продаж (используйте значение 999 для товаров с нулевым темпом продаж). Определите точку повторного заказа (reorder_point) для каждого товара как 14-кратный дневной темп продаж (буфер на 14-дневный срок поставки). Отметьте товары приоритетом Critical, если их текущий остаток меньше либо равен точке повторного заказа и при этом товар не закончился; товарам, которых нет в наличии, присвойте приоритет Out_of_Stock; всем остальным — Normal.

Создайте в рабочей директории книгу Excel под названием Inventory_Lifecycle_Report.xlsx с четырьмя листами.

ВАЖНО: инструмент Excel может терять рабочую директорию, поэтому при работе с файлом передавайте АБСОЛЮТНЫЙ путь к Inventory_Lifecycle_Report.xlsx внутри рабочей директории агента.

Первый лист должен называться Product_Inventory и содержать колонки product_name, category, price, stock_qty, stock_status, total_sales и days_of_supply. Включите по одной строке на каждый товар каталога, отсортировав по days_of_supply по возрастанию, чтобы наиболее срочные позиции были вверху.

Второй лист должен называться Reorder_Alerts и содержать колонки product_name, current_stock, reorder_point и urgency. Включите только товары с приоритетом Critical или Out_of_Stock, отсортированные по urgency так, чтобы позиции Out_of_Stock шли первыми.

Третий лист должен называться Category_Summary и содержать колонки category, product_count, avg_stock и total_value. Значение total_value — это сумма произведений цены на количество на складе для каждого товара категории. Включите по одной строке на каждую категорию товаров.

Четвёртый лист должен называться Restock_Schedule и содержать колонки product_name, reorder_date, quantity и supplier. Для каждого товара с приоритетом Critical или Out_of_Stock установите дату повторного заказа 10 марта 2026 года. Заказываемое количество — это точка повторного заказа минус текущий остаток (минимум 1 единица). Поставщик для всех позиций — "Primary Supplier".

Опубликуйте данные Category_Summary в общей системе электронных таблиц, чтобы команда могла видеть состояние запасов в реальном времени. Создайте таблицу с названием "Inventory Dashboard" с листом, содержащим сводку по категориям.

Запланируйте встречу по пересмотру пополнения запасов в общем календаре на 11 марта 2026 года в 10:00 продолжительностью один час. Заголовок (summary) события — "Inventory Restock Review Meeting", а в описании должно упоминаться число выявленных критичных позиций.

Отправьте письмо на адрес purchasing@company.com с темой "Critical Inventory Alert - Restock Required", в котором обобщается число отсутствующих на складе и критичных товаров, а также перечислены три наиболее срочные позиции, требующие немедленного пополнения.
