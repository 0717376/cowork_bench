# Руководство по анализу совместных покупок (бандлинг)

## Извлечение данных о совместных покупках

Анализ совместных покупок (co-purchase) выявляет товары, которые покупатели часто берут вместе в одном заказе. Источник данных — позиции заказа (line items), где каждый заказ может содержать несколько товаров.

### Методология

1. **Извлеките позиции заказов**: каждый заказ содержит поле `line_items` (JSONB-массив) с записями вида `{"product_id": 16, "name": "Canon M50...", "quantity": 1, "price": 698.67}`.

2. **Отфильтруйте подходящие заказы**: учитывайте только заказы со статусом 'completed' или 'processing'.

3. **Сформируйте пары товаров**: для каждого заказа с 2+ различными товарами сгенерируйте все уникальные пары (A, B), где product_id у A меньше product_id у B, чтобы избежать дублирования.

4. **Посчитайте совместные вхождения**: подсчитайте, в скольких заказах каждая пара товаров встречается вместе. Это и есть co-purchase count.

5. **Вычислите метрики выручки**: для каждой пары рассчитайте среднюю совместную выручку (average combined revenue) — сумму цен обоих товаров в заказах, где они встречаются вместе.

### Приоритетный скоринг

Priority Score = Co_Purchase_Count * Avg_Combined_Revenue

Эта метрика балансирует частоту покупок и потенциал выручки. Более высокие значения указывают на бандлы, которые одновременно популярны и прибыльны.

### Аналитика по категориям (Category Insights)

Отслеживайте, какие категории товаров наиболее активно участвуют в бандлах. Рассчитайте:
- Количество уникальных товаров из каждой категории, попавших хотя бы в один бандл
- Bundle participation rate: процент товаров категории, попавших в бандлы
- Top bundle partner category: с какой другой категорией чаще всего образуют пары товары данной категории
