Вы помогаете исследователю подготовиться к работе программного комитета российской конференции по искусственному интеллекту. В рабочей директории находятся три файла, с которыми нужно ознакомиться перед началом анализа.

Сначала откройте PDF-файл review_guidelines.pdf в рабочей директории. В нём описаны критерии оценки, которые комитет использует для выставления баллов статьям, включая пять измерений (новизна, строгость методологии, полнота экспериментов, ясность и значимость), каждое из которых оценивается по шкале от 1 до 5. Изучите балльную шкалу перед тем, как продолжить.

Затем прочитайте файл conference_papers.txt в рабочей директории. В нём перечислены четыре идентификатора статей, назначенных вам на рецензирование. Это единственные статьи, которые вы должны оценивать. В архиве могут быть и другие статьи, но игнорируйте любые, которых нет в вашем списке.

Для каждой из четырёх назначенных статей получите полный исходный код LaTeX из архива статей. Внимательно изучите структуру разделов, аннотацию и содержание методологии каждой статьи.

Напишите Python-скрипт под названием extract_methods.py, который читает содержимое LaTeX каждой назначенной статьи и извлекает методологические ключевые слова и техники, упомянутые в каждой статье. Скрипт должен выводить файл methodology_analysis.json в рабочую директорию. Каждая запись должна содержать идентификатор статьи, название (Title), список найденных методологических ключевых слов и количество разделов в статье. Запустите этот скрипт через терминал.

Напишите второй Python-скрипт под названием score_papers.py, который использует анализ методологии и балльную шкалу из PDF для построения сравнительной матрицы. Для каждой статьи назначьте целочисленные баллы от 1 до 5 по каждому из пяти критериев (новизна, строгость методологии, полнота экспериментов, ясность, значимость). Используйте следующие рекомендации по выставлению баллов из PDF: статьи с большим числом разделов и подробными экспериментами получают более высокий балл за полноту; статьи, вводящие действительно новые архитектуры или техники, получают более высокий балл за новизну; статьи с чёткими математическими формулировками получают более высокий балл за строгость; статьи с хорошо структурированными разделами и читабельными аннотациями получают более высокий балл за ясность; статьи, решающие важные проблемы с сильными результатами, получают более высокий балл за значимость. Скрипт должен выводить comparison_matrix.json, содержащий идентификатор статьи, название (Title) и словарь баллов по каждому критерию, плюс итоговый балл (total). Запустите этот скрипт.

Создайте облачную таблицу под названием "Paper Review Matrix" с тремя листами. Первый лист должен называться "Review Scores" со столбцами Paper_ID, Title, Novelty, Methodology_Rigor, Experimental_Completeness, Clarity, Significance и Total_Score. Заполните его четырьмя назначенными статьями, отсортированными по Total_Score по убыванию. Второй лист должен называться "Methodology Comparison" со столбцами Paper_ID, Title, Methods_Used и Section_Count, перечисляя методологические ключевые слова, найденные для каждой статьи. Третий лист должен называться "Rankings" со столбцами Rank, Paper_ID, Title, Total_Score и Recommendation. Столбец Recommendation должен содержать "Accept" для статей с баллом 20 и выше, "Revise" для статей с баллом от 15 до 19 и "Reject" для статей с баллом ниже 15.

Напишите третий Python-скрипт под названием generate_rankings.py, который читает comparison_matrix.json и выводит final_rankings.json со статьями, упорядоченными по итоговому баллу по убыванию, включая ранг (rank), идентификатор статьи, название, итоговый балл и рекомендацию (recommendation) для каждой. Запустите этот скрипт.

Создайте документ Word под названием Conference_Review_Summary.docx. Он должен начинаться с раздела "Overview" (Обзор), кратко описывающего процесс рецензирования и четыре оцениваемые статьи. Затем включите раздел "Per-Paper Review" (Обзор по статьям) с подразделом для каждой статьи, содержащим название статьи, краткое описание её сильных сторон (strengths), краткое описание её слабых сторон (weaknesses), баллы по каждому критерию и итоговый балл. После этого добавьте раздел "Comparative Analysis" (Сравнительный анализ), обсуждающий, как статьи соотносятся друг с другом методологически и какие подходы наиболее перспективны. Завершите разделом "Recommendations" (Рекомендации), перечисляющим каждую статью с её итоговой рекомендацией accept, revise или reject, а также с кратким обоснованием.

Названия разделов в документе Word можно оформлять на русском языке (Обзор, Обзор по статьям, Сравнительный анализ, Рекомендации) или на английском, как указано выше — допустимы оба варианта. Литералы рекомендаций Accept/Revise/Reject оставляйте на английском.

Важно: документ Word создаётся через MCP-сервер word, который не сохраняет текущую рабочую директорию. Поэтому при создании/сохранении Conference_Review_Summary.docx передавайте АБСОЛЮТНЫЙ путь к файлу внутри рабочей директории агента.

Также прочитайте файл review_template.md в рабочей директории для дополнительных указаний по форматированию структуры документа Word.
