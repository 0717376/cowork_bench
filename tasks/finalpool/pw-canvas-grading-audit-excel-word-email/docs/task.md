С помощью терминала создай и запусти в рабочей директории Python-скрипт course_grading_processor.py, который читает собранные данные из создаваемых тобой JSON-файлов, выполняет анализ и выводит результат в course_grading_results.json. В course_grading_results.json обязательно сохрани извлечённые с дашборда эталонные значения (например, национальные показатели успеваемости и/или метки уровней курсов) и рассчитанные расхождения (gap) между нашими внутренними значениями и эталоном — чтобы было видно, что сравнение с эталоном действительно проводилось.

Создай Excel-файл Grading_Audit_Report.xlsx с тремя листами.

1. Лист Data_Analysis с основными данными сравнения. Заголовки столбцов должны быть строго следующими (на английском): Course, Code, Enrollment, Avg_Score, Pass_Rate. Здесь Course — название курса (основное измерение), Code — код курса, Enrollment — число записанных студентов, Avg_Score — наш внутренний средний балл, Pass_Rate — наш внутренний процент успешной сдачи. Должно быть не менее 6 строк с курсами. Отсортируй строки по алфавиту по столбцу Course. Значения Avg_Score и Pass_Rate должны быть числами в диапазоне 0-100, Enrollment — положительным целым числом.

2. Лист Metrics с двумя столбцами Metric и Value, обобщающий ключевую статистику. Включи как минимум строки: Total_Courses (число курсов в Data_Analysis), Total_Enrollment (сумма столбца Enrollment), Avg_Score (среднее по столбцу Avg_Score), Avg_Pass_Rate (среднее по столбцу Pass_Rate). Значения должны быть согласованы с данными листа Data_Analysis.

3. Лист Recommendations с приоритетными действиями на основе анализа расхождений. Заголовки столбцов: Priority, Action, Course. Не менее 2 строк. Упорядочь по столбцу Priority; в строке с наивысшим приоритетом должен фигурировать курс с наибольшим отрицательным расхождением / низким Pass_Rate (то есть тот, кому в первую очередь нужна поддержка). В столбце Course указывай реальные названия (или коды) курсов из листа Data_Analysis.
