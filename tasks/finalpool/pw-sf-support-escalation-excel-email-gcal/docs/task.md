Мне нужна помощь с анализом эскалаций поддержки (sf escalation). Есть внешние эталонные (benchmark) данные: зайди на http://localhost:30332 и выгрузи оттуда соответствующие метрики.

Затем запроси наши внутренние данные из корпоративного хранилища данных ClickHouse для сравнения.

С помощью terminal создай и запусти Python-скрипт sf_escalation_processor.py в рабочей директории, который читает собранные данные из создаваемых тобой JSON-файлов, выполняет анализ и выводит результат в sf_escalation_results.json.

Создай Excel-файл Support_Escalation_Report.xlsx с тремя листами. Первый лист Data_Analysis должен содержать основные данные сравнения с соответствующими колонками. Второй лист Metrics должен суммировать ключевые метрики. Третий лист Recommendations должен перечислять конкретные действия.

Лист Data_Analysis должен содержать колонки Priority (приоритет, как первичное измерение), Ticket_Count (количество тикетов), Our_Avg_Response (наше среднее время отклика), Industry_Avg (эталонное среднеотраслевое значение) и Gap (разница между нашим значением и эталоном: Gap = Our_Avg_Response - Industry_Avg), а также CSAT (наш показатель удовлетворённости). Отсортируй данные по Priority в алфавитном порядке. Лист Metrics должен иметь две колонки Metric и Value, суммирующие общее количество тикетов, средние значения и ключевую статистику. Лист Recommendations должен содержать колонки Priority и Action и перечислять приоритетные действия на основе анализа отклонений (gap). Отправь письмо на team-lead@company.com с темой "Analysis Report Complete", резюмируя ключевые выводы. Запланируй встречу для обзора с заголовком "Analysis Review" на 14 марта 2026 года с 14:00 до 15:00 UTC.

Сохрани все выходные файлы в рабочей директории. ВАЖНО: при создании файла Support_Escalation_Report.xlsx через MCP-инструмент excel передавай АБСОЛЮТНЫЙ путь к файлу внутри рабочей директории, так как этот инструмент не сохраняет текущую рабочую директорию.
