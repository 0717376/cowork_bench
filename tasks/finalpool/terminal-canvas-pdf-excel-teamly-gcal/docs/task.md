Я заведующий кафедрой и готовлю отчёт о самообследовании к предстоящей аккредитации нашей образовательной программы (аккредитация Рособрнадзора). В рабочей директории есть PDF-документ accreditation_standards.pdf, в котором описаны восемь критериев аккредитации, которым должна соответствовать наша программа, и для каждого критерия задан конкретный минимальный порог. Начните с чтения этого PDF, чтобы понять все критерии и их требования.

Затем выгрузите данные о курсах из нашей системы управления обучением (LMS). Получите полный список курсов нашей программы, и для каждого курса извлеките число записанных студентов (enrollment), средний балл за работы (average submission grade) и средний балл за тесты (average quiz score). Особое внимание уделите двум секциям курса «Creative Computing and Culture» (курсы 7 и 8) — это наши флагманские дисциплины, и эксперты по аккредитации всегда смотрят на них пристально.

В рабочей директории также есть файл previous_review.csv с результатами прошлого цикла аккредитации и файл faculty_credentials.json со списком квалификаций преподавателей. Используйте их как справочные материалы при оценке соответствия.

Напишите и запустите Python-скрипт compute_metrics.py в рабочей директории, который читает собранные данные о курсах и вычисляет метрики уровня программы. Скрипт должен вычислить:
- общий средний балл (avg_gpa) — как среднее по всем курсам из их средних баллов за работы по шкале 0–100 (то есть среднее значение по-курсовых средних Avg_Grade);
- прокси удержания (retention) — суммарное число записанных студентов, делённое на суммарное число сданных работ по всем курсам;
- индекс разнообразия оценивания (assessment diversity) — число курсов, у которых есть и задания (assignments), и тесты (quizzes), делённое на общее число курсов.
Скрипт должен записать результаты в program_metrics.json в рабочей директории.

Затем напишите и запустите второй скрипт evaluate_compliance.py, который читает критерии аккредитации из accreditation_standards.pdf (пороговые значения, которые вы извлекли ранее, можно зашить в код) и файл program_metrics.json, и оценивает каждый из восьми критериев. Для каждого критерия он должен определить статус: Compliant, если метрика достигает порога или превышает его; Partial, если она в пределах 10 процентов от порога; Non-Compliant, если ниже. Результаты записать в compliance_assessment.json.

Создайте в рабочей директории книгу Excel под названием Accreditation_Self_Study.xlsx с четырьмя листами. Первый лист Course_Data должен содержать столбцы Course_Name, Enrollment, Avg_Grade, Quiz_Avg и Assignment_Count, по одной строке на курс, отсортированные по алфавиту. Второй лист Accreditation_Matrix должен содержать столбцы Criterion, Threshold, Actual_Value и Status, по одной строке на каждый критерий аккредитации. Третий лист Gap_Analysis должен содержать столбцы Criterion, Gap_Description, Remediation_Action и Timeline, со строками только для тех критериев, у которых статус Partial или Non-Compliant. Четвёртый лист Resource_Needs должен содержать столбцы Resource, Justification, Estimated_Cost и Priority, и перечислять не менее трёх ресурсов, необходимых для устранения пробелов в соответствии (например, найм дополнительных преподавателей, программное обеспечение для оценивания или консультации по пересмотру учебного плана).

Названия файлов (Accreditation_Self_Study.xlsx, compute_metrics.py, evaluate_compliance.py, generate_summary.py, program_metrics.json, compliance_assessment.json, accreditation_summary.txt), названия листов (Course_Data, Accreditation_Matrix, Gap_Analysis, Resource_Needs), названия столбцов и значения статусов (Compliant, Partial, Non-Compliant) сохраняйте на английском языке без изменений. При работе с Excel и при запуске Python-скриптов передавайте АБСОЛЮТНЫЙ путь к файлам внутри рабочей директории, так как у MCP-сервера Excel может не быть надёжной текущей директории.

Создайте в корпоративной базе знаний Teamly страницу под названием «Accreditation Action Items». На этой странице оформите таблицу, где каждая строка — отдельный критерий аккредитации, всего восемь строк (по одной на каждый из восьми критериев). Таблица должна содержать столбцы Criterion, Status, Owner, Due_Date и Evidence. В столбце Criterion укажите название критерия, в столбце Status — статус из результатов оценки соответствия (Compliant, Partial или Non-Compliant), в столбце Owner — ответственного преподавателя из списка в faculty_credentials.json, в столбце Due_Date — срок (распределите сроки по ближайшим трём месяцам), а в столбце Evidence — какие доказательства нужно собрать по этому критерию. Название страницы «Accreditation Action Items», названия столбцов и значения статусов сохраняйте на английском языке без изменений.

Проверьте общий календарь на наличие уже существующих событий, связанных с аккредитацией. Затем создайте три новых события в календаре. Первое — событие под названием «Evidence Collection Deadline» на 30 дней от сегодняшней даты. Второе — событие под названием «Draft Report Due» на 60 дней от сегодняшней даты. Третье — событие под названием «Final Submission» на 90 дней от сегодняшней даты. Каждое событие должно содержать описание, в котором упоминается самообследование по аккредитации (accreditation self-study). Названия событий сохраняйте на английском языке без изменений.

Наконец, напишите и запустите скрипт generate_summary.py, который читает результаты оценки соответствия и создаёт текстовый файл accreditation_summary.txt в рабочей директории. В этой сводке должен быть перечислен каждый критерий с его статусом, выделены найденные пробелы и указан общий процент соответствия программы.
