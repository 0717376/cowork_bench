# Методология анализа первопричин возвратов

## Обзор
Этот гайд описывает методологию расследования паттернов возвратов и выявления первопричин путём сопоставления данных о возвратах с отзывами на товары.

## Шаг 1: Сбор данных
- Выгрузите все записи о возвратах, включая идентификатор возврата, идентификатор заказа, сумму, причину и дату.
- Для каждого заказа с возвратом определите вовлечённые товары из позиций заказа (line items).
- Соберите данные об отзывах на товары (рейтинги и тексты отзывов) для всех возвращённых товаров.

## Шаг 2: Классификация серьёзности
Классифицируйте каждый возврат по серьёзности на основе суммы возврата:
- **Critical**: сумма возврата больше $50
- **Major**: сумма возврата от $20 до $50 (включая $20)
- **Minor**: сумма возврата меньше $20

## Шаг 3: Определение типа проблемы
Для каждого возвращённого товара определите тип проблемы:
- Рассчитайте средний рейтинг отзывов по всем отзывам на товар.
- Если по товару есть возвраты И средний рейтинг < 3.0 → **Quality Issue**
- Если по товару есть возвраты И средний рейтинг >= 3.0 → **Service Issue**
- Если по товару есть возвраты, но нет отзывов → **No Reviews**

## Шаг 4: Сопоставление первопричин
Сопоставьте причины возвратов с категориями первопричин (значения причин в данных хранятся на английском):
- "Product defective" → Manufacturing Defect
- "Product arrived damaged" → Shipping Damage
- "Wrong item received" → Wrong Item
- "Changed my mind" или "Item not as described" → Customer Expectations
- "Duplicate order" или прочее → Other

## Шаг 5: Пометка к проверке
Товары с 2 и более возвратами требуют официальной проверки поставщика.

## Шаг 6: Отчётность
- Создайте Excel-книгу с листами Refund Details, Product Impact и Summary.
- Напишите Word-документ с разделами Executive Summary, Refund Trend Analysis, Product-Level Findings, Root Cause Classification, Recommendations и Action Plan.
- Отправьте находки отделу качества и отделу по работе с поставщиками.
