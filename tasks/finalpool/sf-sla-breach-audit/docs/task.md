Команда эксплуатации хочет понять, как часто служба поддержки не укладывается в установленное время реакции, зафиксированное в наших соглашениях об уровне обслуживания (SLA). Наше облачное хранилище данных ClickHouse (схема sf_data) содержит все записи обращений в поддержку (SUPPORT_CENTER) вместе с определениями SLA-политик, задающими максимально допустимое время первичной реакции для каждого уровня приоритета. Пожалуйста, запросите оба набора данных, соедините их по приоритету (PRIORITY) и рассчитайте количество и долю обращений, в которых время реакции превысило целевой показатель, для каждого уровня приоритета.

Создайте Excel-файл с именем SLA_Breach_Audit.xlsx в рабочей директории, состоящий из двух листов. Первый лист должен называться "Breach Analysis" и содержать столбцы Priority, Total_Tickets, Avg_Response_Hours (среднее фактическое время реакции, округлённое до двух знаков после запятой), SLA_Target_Hours (целевое время реакции из SLA-политики), Breached_Count (число обращений, где фактическое время реакции превысило целевой показатель SLA) и Breach_Rate (доля нарушений в процентах, округлённая до двух знаков после запятой). Отсортируйте строки по столбцу SLA_Target_Hours по возрастанию. Включайте только те приоритеты, по которым действительно есть данные об обращениях.

Второй лист должен называться "Summary" со столбцами Metric и Value. Включите следующие записи: Total_Tickets (общий итог), Total_Breached (всего нарушений по всем приоритетам), Overall_Breach_Rate (общая доля нарушений в процентах, округлённая до двух знаков после запятой), Worst_Priority (приоритет с наибольшей долей нарушений) и Best_Priority (приоритет с наименьшей долей нарушений). Значения приоритетов оставляйте на английском языке (High, Medium, Low), как они хранятся в источнике.

Когда отчёт будет готов, отправьте письмо с адреса sla-audit@company.com на адрес ops-director@company.com с темой "SLA Breach Audit Results", указав в теле письма общую долю нарушений (Overall_Breach_Rate) и приоритет с худшими показателями (Worst_Priority).

Примечание: при создании .xlsx через инструмент Excel передавайте абсолютный путь к файлу внутри рабочей директории (рабочий каталог инструмента может отличаться), чтобы файл гарантированно был записан в рабочую директорию.
