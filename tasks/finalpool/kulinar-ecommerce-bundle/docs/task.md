У нас есть интернет-магазин (на платформе InSales), который продаёт кухонную технику, и мы хотим собрать товарные наборы на основе рецептов, чтобы увеличить продажи. Идея в том, чтобы соединить нашу бытовую технику с популярными рецептами, для которых эта техника пригодится, и упаковать их как промо-наборы.

Вот что нужно сделать.

Во-первых, загляните в каталог товаров нашего интернет-магазина и выберите все товары из категории «Бытовая техника» (в системе она помечена как Home Appliances). По каждому товару мне нужны его название, обычная цена и количество на складе.

Во-вторых, найдите в кулинарной базе рецептов (Кулинар) по одному рецепту из трёх разных категорий: один рецепт из категории напитков (напиток), один из категории гарниров (гарнир) и один из категории горячих блюд (горячее). По каждому рецепту получите полную информацию, включая название, список ингредиентов и уровень сложности. Подбирайте рецепты так, чтобы они естественно сочетались с кухонной техникой (например, рецепт напитка, для которого пригодится блендер; гарнир, который удобно готовить в мультиварке; горячее блюдо, которое хорошо получается на пару или в вакууматоре). Сами решите, какие пары будут удачными.

В-третьих, проверьте свежие котировки сырьевого эмитента ЛУКОЙЛ (тикер LKOH.ME) — его цена влияет на наши закупочные издержки на комплектующие. Возьмите цены закрытия за последние 5 торговых дней по этому тикеру.

Когда все данные собраны, создайте Excel-файл с именем Bundle_Pricing.xlsx в рабочей директории с тремя листами.

Первый лист должен называться Store Products. В нём перечислите все товары категории «Бытовая техника» со следующими колонками: Product_Name, Price, Stock_Quantity. Отсортируйте строки по колонке Price по возрастанию.

Второй лист должен называться Gold Trend. У него должны быть колонки: Date, Close_Price. Включите цены закрытия по тикеру LKOH.ME за последние 5 торговых дней, отсортированные по дате по убыванию (самая свежая дата сверху). Значения Close_Price округлите до 2 знаков после запятой.

Третий лист должен называться Recipe Bundles. У него должны быть колонки: Recipe_Name, Recipe_Category, Difficulty, Paired_Product, Bundle_Price. Каждая строка — это один набор, объединяющий рецепт с товаром-техникой из нашего магазина. Строк должно быть ровно 3, по одной на каждый выбранный рецепт. Recipe_Name — это название рецепта на русском языке ровно так, как оно записано в кулинарной базе. Recipe_Category — название категории на русском (напиток, гарнир или горячее). Difficulty — уровень сложности из рецепта (целое число от 1 до 4). Paired_Product — название товара из магазина, который вы объединяете с этим рецептом. Bundle_Price — цена парного товара, умноженная на 0.85 и округлённая до 2 знаков после запятой, что соответствует скидке 15% на набор.

Примечание: при работе с Excel-инструментом передавайте ему абсолютный путь к файлу Bundle_Pricing.xlsx внутри рабочей директории.
