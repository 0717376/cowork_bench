Вы — менеджер сообщества разработчиков. Вы хотите провести опрос сообщества об их предпочтениях по видео, проанализировать, какие темы по программированию вызывают наибольшую вовлечённость на канале Fireship, создать онлайн-форму опроса для сообщества, собрать статистику по видео в таблицу и запланировать звонок сообщества. Менеджер сообщества по адресу community@devclub.io попросил вас подготовить этот ежемесячный отчёт о вовлечённости.

Ваш рабочий процесс должен выглядеть следующим образом.

Во-первых, найдите на видеоплатформе видео Fireship, относящиеся к контенту по JavaScript и TypeScript. Из этих результатов, а также из более широкого поиска по каналу, выберите топ-8 видео по количеству просмотров, охватывающих такие темы, как JavaScript, TypeScript, React, AI and machine learning, systems programming и DevOps. Для каждого видео соберите идентификатор видео, заголовок, количество просмотров, количество лайков, длительность в секундах, дату публикации и основной тематический тег.

Во-вторых, создайте файл таблицы с именем Community_Report.xlsx. На первом листе с именем Top_Videos укажите следующие столбцы по порядку: Rank, Video_ID, Title, Views, Likes, Duration_Sec, Topic_Tags. Заполните этот лист по одной строке на видео, отсортировав от наибольшего к наименьшему по количеству просмотров, причём Rank начинается с 1. Также вычислите коэффициент вовлечённости (engagement rate) для каждого видео как количество лайков, делённое на количество просмотров, умноженное на 100, и добавьте его как дополнительный столбец с именем Engagement_Rate. На втором листе с именем Engagement_Analysis укажите следующие столбцы: Topic, Avg_Engagement_Rate, Total_Views, Video_Count. Заполните его по одной строке на каждую отдельную тему, агрегируя видео по их Topic_Tags. Должно быть не менее 4 строк с темами.

В-третьих, создайте онлайн-форму опроса с заголовком "Fireship Community Preference Survey". Добавьте в форму следующие вопросы. Первый вопрос должен спрашивать "Which Fireship topic interests you most?" в виде вопроса с единственным выбором (radio) со следующими вариантами: JavaScript, TypeScript, React, AI and ML, Systems and Rust, DevOps and Cloud, Other. Второй вопрос должен спрашивать "How often do you watch Fireship?" в виде radio-вопроса с вариантами: Daily, Weekly, Monthly, Rarely. Третий вопрос должен спрашивать "What format do you prefer?" в виде radio-вопроса с вариантами: Short under 5 minutes, Medium 5 to 15 minutes, Long over 15 minutes. Четвёртый вопрос должен спрашивать "Your current role" в виде radio-вопроса с вариантами: Student, Junior Developer, Senior Developer, Engineering Manager, Other. Пятый вопрос должен спрашивать "What topic should Fireship cover next?" в виде вопроса со свободным текстовым ответом.

В-четвёртых, проверьте общий календарь на наличие любых уже существующих событий сообщества в апреле 2026 года. Затем создайте новое событие в календаре с заголовком "Community Standup", запланированное на 1 апреля 2026 года с 18:00 до 19:00 UTC.

В-пятых, отправьте письмо на адрес community@devclub.io. Тема должна ссылаться на ежемесячный отчёт о вовлечённости. В теле письма кратко изложите топ-темы видео, упомяните найденные коэффициенты вовлечённости, укажите идентификатор формы опроса, чтобы участники могли получить к ней доступ, и отметьте запланированный на 1 апреля community standup.

Файл конфигурации в вашем рабочем пространстве называется community_config.json и содержит целевые темы и заголовок отчёта. Шаблон отчёта report_template.md объясняет форматы столбцов, как вычисляется коэффициент вовлечённости и логику ранжирования.

Когда все результаты будут готовы, отметьте задачу как выполненную.
