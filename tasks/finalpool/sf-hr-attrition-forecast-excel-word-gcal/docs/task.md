Нашей HR-команде нужен комплексный анализ риска оттока, чтобы выявить сотрудников, которые могут уйти из организации. В рабочем каталоге есть PDF-документ Retention_Policy.pdf, описывающий используемую нами методику оценки риска, а также файл guide.md, который подробнее объясняет методологию. Пожалуйста, внимательно прочитайте оба документа перед началом анализа.

Нужно выгрузить данные о сотрудниках и историю изменения зарплат из нашего корпоративного хранилища данных ClickHouse. Для каждого сотрудника рассчитайте балл риска на основе четырёх факторов, описанных в регламенте удержания: стагнация роста зарплаты, низкая удовлетворённость работой, низкая результативность и наличие опытных сотрудников, которым недоплачивают по сравнению с коллегами по подразделению. Для обработки данных используйте terminal, чтобы запускать любые нужные аналитические скрипты.

После вычисления баллов риска создайте в рабочем каталоге Excel-файл с именем Attrition_Risk_Analysis.xlsx с тремя листами. Первый лист должен называться "Risk Assessment" и содержать по одной строке для каждой комбинации Department и Risk_Level, отсортированных сначала по Department по алфавиту, а затем по Risk_Level по алфавиту внутри каждого подразделения. Столбцы: Department, Risk_Level, Employee_Count, Avg_Satisfaction, Avg_Performance и Avg_Salary_Growth_Pct, средние значения округлены до одного знака после запятой. Второй лист должен называться "Department Summary" с одной строкой на подразделение, отсортированных по алфавиту, со столбцами Department, Total_Employees, High_Risk_Count, Medium_Risk_Count, Low_Risk_Count, High_Risk_Pct (доля сотрудников высокого риска, округлённая до одного знака) и Total_Intervention_Budget (рассчитывается как количество сотрудников высокого риска, умноженное на 5000, плюс количество сотрудников среднего риска, умноженное на 2000). Третий лист должен называться "Top Risk Factors" с одной строкой на подразделение, со столбцами Department, Top_Factor (название фактора, который даёт наибольшую суммарную сумму баллов риска по всем сотрудникам этого подразделения), Factor_Score (средний вклад этого фактора на одного сотрудника, округлённый до одного знака) и Recommended_Action (краткое описание рекомендуемой меры для этого фактора).

Значения Risk_Level (Low/Medium/High) и Top_Factor оставьте на английском — это технические идентификаторы. Названия подразделений (Department) используйте ровно в том виде, в каком они хранятся в базе данных.

Далее создайте в рабочем каталоге документ Word с именем Intervention_Report.docx. Этот отчёт должен включать следующие разделы: краткое резюме (Executive Summary) с обзором общего числа сотрудников в каждой категории риска и общего бюджета мер; раздел анализа по подразделениям (Department-by-Department Analysis) с подразделом по каждому из семи подразделений, показывающим разбивку по риску и главный фактор; раздел обзора бюджета (Budget Overview) с таблицей распределения бюджета по подразделениям; раздел рекомендуемых мер (Recommended Actions) с общеорганизационными рекомендациями; и раздел сроков (Timeline) с описанием дальнейших шагов. ВАЖНО: при работе с инструментом Word передавайте АБСОЛЮТНЫЙ путь к файлу .docx внутри рабочего каталога.

Наконец, запланируйте в календаре семь часовых встреч-обзоров подразделений, по одной на каждое подразделение. Эти встречи должны проходить в течение недели с 16 по 20 марта 2026 года в рабочее время. Каждая встреча должна называться по шаблону «<название подразделения> — обзор риска оттока» и содержать краткое описание с указанием количества сотрудников высокого риска и рекомендуемого бюджета мер для этого подразделения. Распределите встречи так, чтобы не было пересечений по времени.
