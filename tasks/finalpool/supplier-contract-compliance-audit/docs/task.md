Вашей команде по закупкам необходимо провести комплексный аудит соблюдения договоров с поставщиками, чтобы убедиться, что подрядчики выполняют свои договорные обязательства, и выявить возможности для повышения эффективности. Исходные данные уже находятся в вашей рабочей директории: список поставщиков `supplier_list.csv` (заголовки столбцов на английском: `Supplier_Name`, `Contract_Start_Date`, `Annual_Contract_Value`, `Primary_Contact`, `Contact_Email`, `Service_Type`), шаблон договора `contract_template.pdf` со стандартными разделами (предмет услуг, SLA, условия оплаты, интеллектуальная собственность, конфиденциальность, ограничение ответственности), а также индивидуальные договоры всех 5 поставщиков — файлы `contract_supplier*.pdf` рядом с шаблоном. Соответствие каждого поставщика оценивается путём сравнения его договора со стандартным шаблоном. Реестр аналитических данных по запросам доступен через ClickHouse (при необходимости).

Первый этап — собрать все действующие договоры и соглашения об услугах. Сгруппируйте договоры по поставщику, категории услуг и дате начала договора. Создайте централизованный реестр всех договорных документов, обеспечив удобный доступ и контроль версий. Зафиксируйте базовую информацию по каждому поставщику: контактные данные, оказываемые услуги, стоимость договора и даты продления.

На втором этапе извлеките и задокументируйте все ключевые показатели эффективности и требования соответствия, указанные в каждом договоре. Для каждого соглашения определите обязательства по уровню обслуживания: сроки поставки, стандарты качества продукции, время реакции службы поддержки. Извлеките финансовые условия: цены, условия оплаты, объёмные скидки. Задокументируйте требования соответствия: страхование, сертификаты, разрешения регуляторов. Составьте сводную матрицу всех договорных обязательств по каждому поставщику.

Третий этап — сопоставление фактического исполнения с договорными обязательствами. По каждому поставщику соберите данные об исполнении: метрики качества, доля своевременных поставок, отзывчивость и другие показатели. Сравните фактическое исполнение с договорными стандартами. Определите, где поставщики превосходят ожидания, а где не дотягивают до обязательств. Задокументируйте конкретные случаи несоответствия с подтверждающими данными.

На четвёртом этапе создайте детальные оценочные карты соответствия по каждому поставщику. Рассчитайте баллы соответствия (Compliance Score, 0–100) по различным измерениям. Представьте результаты в наглядном виде, показав, какие поставщики выполняют обязательства, а какие требуют внимания. Агрегируйте данные по всем поставщикам, чтобы выявить общеорганизационные закономерности.

Пятый этап — детальный анализ первопричин по выявленным пробелам. Определите, вызваны ли нарушения недостатком возможностей поставщика, нехваткой ресурсов, внешними факторами или нечёткими ожиданиями. Оцените влияние несоответствий на бизнес и приоритизируйте проблемы для устранения. Разработайте конкретные практические рекомендации по улучшению работы поставщиков.

Шестой этап — подготовка итоговых выводов и отчётов аудита, а также рассылка результатов. Задокументируйте все выводы с подтверждающими данными и оценкой влияния. Отправьте письма поставщикам на их контактные адреса из `supplier_list.csv` (поле `Contact_Email`) и заинтересованным сторонам с темой об аудите соблюдения договоров; включите требования по устранению недостатков и сроки. Запланируйте встречи в Google Calendar с поставщиками, показавшими низкие результаты, для разработки планов улучшения с чёткой ответственностью и механизмами контроля.

## Обязательные результаты (файлы в рабочей директории)

Создайте РОВНО следующие четыре файла со строго указанными именами, листами и столбцами (идентификаторы листов и столбцов оставляйте на английском; содержимое-прозу пишите на русском):

1. `compliance_assessment.xlsx` — лист **`Compliance Assessment`** со столбцами: `Supplier`, `Scope Clarity`, `SLA Present`, `Payment Terms`, `IP Rights`, `Confidentiality`, `Liability Limits`, `Compliance Score`. По одной строке на каждого из 5 поставщиков из `supplier_list.csv`. Используйте словарь статусов: `Complete`/`Incomplete`, `Compliant`/`Non-Compliant`, `Missing`. `Compliance Score` — число 0–100. Дополнительно можно включить блок оценки рисков (Risk Assessment) с уровнями `Low`/`Medium`/`High`.

2. `corrective_actions.xlsx` — лист **`Corrective Actions`** со столбцами: `Supplier`, `Issue`, `Required Action`, `Owner`, `Target Date`, `Est. Cost`, `Status`. Минимум по одной корректирующей мере на каждого поставщика с выявленными несоответствиями. `Target Date` — дата в формате ISO (`YYYY-MM-DD`) в будущем. `Status` — например `In Progress`/`Planned`.

3. `audit_findings.docx` — детальные выводы аудита: по разделу на каждого из 5 поставщиков с указанием конкретного пробела (например, отсутствие условия об ограничении ответственности у ООО ТехСервис, отсутствие SLA у Глобал Логистика), баллом соответствия и рекомендацией.

4. `audit_summary.docx` — исполнительное резюме: средний балл соответствия и количество поставщиков по уровням риска (высокий/средний/низкий), согласованные с листом оценки.

ВАЖНО: при работе с MCP-сервером `word` (создание `.docx`) передавайте АБСОЛЮТНЫЙ путь к файлу внутри рабочей директории, так как рабочая директория инструмента может не совпадать с вашей.
