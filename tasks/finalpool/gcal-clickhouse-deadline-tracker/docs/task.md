Нашей команде поддержки требуется полный аудит соблюдения SLA по времени ответа на тикеты с приоритетами High и Medium. Все данные по тикетам хранятся в нашем корпоративном хранилище данных ClickHouse вместе с определениями политик SLA, которые задают максимально допустимое время ответа для каждого уровня приоритета.

Запросите данные из хранилища ClickHouse и найдите все тикеты с приоритетом High или Medium. Для каждого тикета сравните фактическое время ответа с целевым временем ответа, заданным в таблице политик SLA. Тикет считается нарушившим SLA ("Breached"), если время ответа превышает целевое значение SLA; близким к нарушению ("Near Breach"), если время ответа превышает 80 процентов целевого значения, но не превышает само целевое значение; и соответствующим SLA ("Compliant") в остальных случаях.

Создайте Excel-файл с именем SLA_Compliance_Audit.xlsx в рабочей директории с двумя листами:

Лист 1 с именем "Breached Tickets" должен содержать каждый тикет с приоритетом High или Medium, нарушивший целевое время ответа по SLA. Включите следующие столбцы в этом порядке: Ticket_ID, Priority, Issue_Type, Created_At (в формате YYYY-MM-DD HH:MM), Response_Time_Hours (фактическое время ответа, округлённое до 2 знаков после запятой), SLA_Target_Hours (целевое время ответа из политики SLA), Hours_Over_SLA (время ответа минус целевое значение, округлённое до 2 знаков после запятой), Customer_Satisfaction (оценка CSAT). Отсортируйте все строки сначала по Priority (High перед Medium), затем по Hours_Over_SLA по убыванию.

Лист 2 с именем "Summary" должен содержать столбцы: Priority, Total_Tickets, Breached_Count, Near_Breach_Count, Compliant_Count, Breach_Rate_Pct (количество нарушений, делённое на общее число и умноженное на 100, округлённое до 1 знака после запятой), Avg_Response_Hours (округлённое до 2 знаков после запятой), Avg_CSAT (средняя оценка удовлетворённости клиентов, округлённая до 2 знаков после запятой). Включите одну строку для приоритета High и одну строку для приоритета Medium, именно в этом порядке.

Далее создайте в Google Calendar события-напоминания о разборе для топ-5 худших нарушений SLA среди тикетов приоритета High (5 тикетов приоритета High с наибольшим Hours_Over_SLA). Каждое событие должно быть запланировано на 2026-03-07 с 09:00 до 09:30, с заголовком "SLA Review: [Ticket_ID]" и описанием "Response SLA breach review for [Ticket_ID]. Priority: [Priority]. Response time: [Response_Time_Hours]h vs target: [SLA_Target_Hours]h. [Hours_Over_SLA]h over SLA."

Создайте в базе знаний команды (Teamly) страницу с заголовком "SLA Breach Dashboard - March 2026" (Дашборд нарушений SLA - Март 2026), которая обобщает результаты аудита. Включите общее количество нарушений по приоритетам, основные типы проблем (Issue_Type), приводящие к нарушениям, и среднюю удовлетворённость клиентов по нарушившим тикетам.

Наконец, отправьте оповещение по электронной почте на адрес support-lead@company.com с темой "SLA Compliance Alert - Response Time Breaches" от sla-monitor@company.com. В тексте письма укажите общее количество нарушений по приоритету High, общее количество нарушений по приоритету Medium, доли нарушений (breach rate) для каждого уровня приоритета и перечислите 5 худших по нарушениям Ticket_ID среди приоритета High.

Примечание: при создании Excel-файла через инструмент excel передавайте абсолютный путь к файлу SLA_Compliance_Audit.xlsx внутри указанной рабочей директории.
