Затем выгрузите фактические данные по тикетам нашей поддержки из корпоративного хранилища данных ClickHouse. Мне нужно понять, как наше время отклика соотносится с отраслевыми средними значениями, в разбивке по приоритету тикетов.

Используя terminal, создайте и запустите в рабочей директории Python-скрипт sla_analyzer.py, который читает sla_raw_data.json (создайте его с объединёнными данными из веб-страницы и хранилища), выполняет сравнительный анализ и сохраняет результат в sla_results.json.

Создайте Excel-файл SLA_Benchmark_Report.xlsx с тремя листами. Первый лист SLA_Comparison должен содержать колонки Priority, Ticket_Count, Our_Avg_Response_Hrs, Industry_Avg_Response_Hrs, Response_Gap (наше значение минус отраслевое, округлить до 2 знаков), Avg_CSAT и Compliance_Status ("Compliant", если наше время отклика не превышает отраслевое среднее, иначе "Non-Compliant"). Отсортируйте по Priority в следующем порядке: Critical, High, Medium, Low.

Второй лист Action_Items должен содержать колонки Priority, Response_Gap, Improvement_Needed_Pct (Response_Gap, делённый на Industry_Avg, умноженный на 100, округлить до 1 знака — только для несоответствующих приоритетов, для соответствующих ставьте 0) и Recommended_Action ("Urgent review needed" при разрыве > 5, "Process optimization required" при разрыве 1-5, "On track" для соответствующих).

Третий лист Summary должен содержать две колонки Metric и Value: Total_Tickets, Compliant_Priorities, Non_Compliant_Priorities, Worst_Priority (приоритет с наибольшим положительным разрывом), Best_Priority (приоритет с наименьшим или наиболее отрицательным разрывом), Overall_CSAT (взвешенное по числу тикетов среднее CSAT, округлить до 2 знаков).
