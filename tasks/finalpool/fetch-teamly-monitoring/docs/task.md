Этот файл содержит 504 ежечасных проверки статуса для трёх сервисов за семь дней — с 27 февраля по 5 марта 2026 года. Каждая запись содержит timestamp, service_name, status (одно из значений up, down или degraded), response_time_ms и error_message (равно null, когда сервис в состоянии up).

Проанализируйте эти данные и создайте Excel-файл с именем Service_Availability_Report.xlsx в рабочей директории с двумя листами. Указывайте абсолютный путь к файлу Service_Availability_Report.xlsx внутри рабочей директории агента при работе с инструментами Excel.

Первый лист должен называться "Availability Summary" и содержать следующие столбцы: Service_Name, Total_Checks, Up_Count, Down_Count, Degraded_Count, Uptime_Pct и Avg_Response_Time_Ms. Uptime_Pct рассчитывается как количество проверок up, делённое на общее число проверок, умноженное на 100 и округлённое до одного знака после запятой. Avg_Response_Time_Ms округляется до двух знаков после запятой. По одной строке на сервис, то есть всего три строки.

Второй лист должен называться "Incidents" и перечислять каждый непрерывный период, когда сервис находился в состоянии down или degraded. Столбцы: Service_Name, Start_Time, End_Time, Duration_Minutes, Status, Error_Message. Новая строка инцидента создаётся каждый раз, когда сервис переходит из состояния up в down или degraded, и инцидент завершается, когда сервис возвращается в up. Duration_Minutes отражает, сколько минут длился инцидент, исходя из временных меток.
