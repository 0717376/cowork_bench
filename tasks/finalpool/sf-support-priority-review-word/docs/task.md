Вы — специалист по аналитике поддержки и готовите подробный отчёт о качестве обработки обращений в разбивке по уровню приоритета для директора по поддержке. Данные обращений и SLA-политики хранятся в облачном хранилище данных ClickHouse (база SUPPORT_CENTER).

Сначала выполните запрос к данным обращений в поддержку, чтобы получить статистику с группировкой по уровню приоритета. Для каждого приоритета (High, Medium, Low) вычислите общее количество обращений, среднее время отклика в часах и среднюю оценку удовлетворённости клиентов.

Также найдите SLA-политики, чтобы определить целевое время отклика для каждого уровня приоритета, и сравнить фактические показатели с целевыми.

Создайте документ Word с именем «Priority_Handling_Report.docx» в вашей рабочей директории. Документ должен иметь заголовок «Support Ticket Priority Analysis» и содержать дату 2026-03-06 в верхней части. Затем включите раздел для каждого уровня приоритета (High, Medium, Low) со следующими метриками: количество обращений, среднее время отклика в часах, средняя оценка удовлетворённости клиентов и целевое время отклика по SLA. После разделов по приоритетам добавьте итоговый раздел с общей сводкой.
Важно: при работе с инструментом word MCP передавайте АБСОЛЮТНЫЙ путь к файлу Priority_Handling_Report.docx внутри вашей рабочей директории.

Далее создайте Google Sheet с названием «Priority Metrics Dashboard» и листом «By Priority», содержащим столбцы: Priority, Ticket_Count, Avg_Response_Hours, Avg_Satisfaction, SLA_Target. Включите по одной строке на каждый уровень приоритета.

Наконец, отправьте письмо с адреса support-analytics@company.com на адрес support-director@company.com с темой «Priority Handling Performance Report». В теле письма кратко изложите ключевые выводы анализа, включая показатели времени отклика для каждого уровня приоритета.
