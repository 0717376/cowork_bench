Вы — вице-президент по клиентскому опыту (VP of Customer Experience), готовящий ежеквартальный обзор качества, который объединяет данные о работе магазина с метриками центра поддержки. Цель — понять, как проблемы с товарами в интернет-магазине коррелируют с паттернами обращений в поддержку, и представить выводы в формате, готовом для совета директоров.

Начните с посещения портала удовлетворённости клиентов по адресу http://localhost:30239, чтобы изучить текущие бенчмарки и целевые показатели удовлетворённости. На главной странице показаны общий целевой показатель удовлетворённости (CSAT), целевое время отклика, целевой уровень решения обращений и целевой NPS. Перейдите на страницу трендов http://localhost:30239/trends.html, чтобы увидеть ежемесячные тренды CSAT и конкурентные бенчмарки. Читайте данные портала честно через браузер — не выдумывайте значения.

Затем подключитесь к магазину InSales и получите все данные о заказах. Сосредоточьтесь на выявлении проблемных заказов: смотрите на заказы со статусом refunded или cancelled, так как они представляют собой проблемные транзакции. Сгруппируйте заказы по категориям товаров (категорию можно определить по данным о товаре в позициях каждого заказа) и рассчитайте общее количество заказов и количество проблемных заказов (refunded или cancelled) для каждой категории. Вычислите долю проблем (issue rate) как процент заказов с проблемами в каждой категории.

Далее запросите хранилище данных ClickHouse центра поддержки (логическая БД SUPPORT_CENTER, схема sf_data) для получения метрик производительности по обращениям. Для каждого уровня приоритета (High, Medium, Low) рассчитайте общее количество обращений, среднее время отклика в часах, среднюю оценку удовлетворённости клиента и долю решённых обращений. Сравните эти метрики с целевыми показателями с портала удовлетворённости.

Для перекрёстного анализа (cross-reference) посмотрите на общие паттерны. Определите, какие категории товаров имеют самую высокую долю проблем по данным магазина, и проверьте, варьируются ли метрики центра поддержки (в особенности время отклика и оценки удовлетворённости) по типам проблем (issue type). Это даёт понимание того, вызывают ли категории товаров с большим числом проблем также и сложности с качеством поддержки.

Создайте файл Excel под названием CX_Quality_Review.xlsx в вашем рабочем каталоге с четырьмя листами. Первый лист должен называться "Order Issues" со столбцами Category, Total_Orders, Issue_Orders и Issue_Rate_Pct. Включите все категории товаров, найденные в данных о заказах.

Второй лист должен называться "Support Metrics" со столбцами Priority, Ticket_Count, Avg_Response_Time_Hours, Avg_Satisfaction, Resolution_Rate_Pct, Response_Target_Hours (с портала) и Meets_Response_Target (Yes или No). Включите строки для каждого уровня приоритета.

Третий лист должен называться "Cross Reference" со столбцами Issue_Type, Ticket_Count, Avg_Response_Time, Avg_Satisfaction. Включите все типы проблем (issue type) из данных поддержки.

Четвёртый лист должен называться "Executive Summary" со столбцами Metric и Value. Включите строки для общего числа проанализированных заказов, общей доли проблем, общего количества обращений в поддержку, среднего времени отклика по всем приоритетам, средней удовлетворённости клиентов и того, достигаются ли общие целевые показатели.

Наконец, создайте презентацию PowerPoint под названием Quality_Review.pptx в вашем рабочем каталоге, содержащую не менее шести слайдов. Презентация должна включать титульный слайд, обзор бенчмарков и целевых показателей удовлетворённости, сводку проблем с заказами магазина по категориям, производительность центра поддержки по уровням приоритета, выводы перекрёстного анализа и рекомендации по улучшению качества клиентского опыта.

Важно: MCP-инструменты Excel и PowerPoint не сохраняют текущий рабочий каталог. При создании и записи CX_Quality_Review.xlsx и Quality_Review.pptx передавайте АБСОЛЮТНЫЙ путь к файлу внутри вашего рабочего каталога.

Когда все задачи выполнены, завершите работу, ответив без вызова каких-либо инструментов.
