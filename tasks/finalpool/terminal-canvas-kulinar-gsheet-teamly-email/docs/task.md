Отдел по работе со студентами проводит пилотное исследование того, связана ли осведомлённость о питании с успеваемостью. Исследование охватывает два курса «Экологическая экономика и этика» (course IDs 13 и 14) из системы управления обучением.

Начните с чтения файла study_protocol.md в рабочем каталоге, чтобы понять методологию исследования и требования IRB. Также изучите файлы nutritional_guidelines.csv и campus_dining_budget.json — они содержат сведения о суточных нормах питания и бюджетных ограничениях.

Запросите в системе управления обучением данные о зачислении и оценках по обоим курсам. Для каждого курса получите задания с их баллами и числом сданных работ. Также соберите данные по студентам об их паттернах сдачи работ — в частности, какие студенты сдают работы с опозданием, и их средние баллы.

Запросите в кулинарной базе рецептов (kulinar) категории здоровых рецептов, сосредоточившись на двух доступных категориях: «гарнир» (для лёгкого приёма пищи) и «горячее» (для основного приёма пищи). Доступные категории базы: салат, закуска, суп, горячее, гарнир, выпечка, десерт, напиток. Используйте инструмент получения рецептов по категории (getRecipesByCategory для категорий «гарнир» и «горячее»); получите количество рецептов в каждой из этих двух категорий. Не выдумывайте названия блюд — используйте реальные данные из базы.

Напишите Python-скрипт под названием categorize_students.py, который читает данные о сдаче работ и распределяет студентов по трём уровням вовлечённости. Студенты без опозданий — «High» (высокая вовлечённость), студенты с долей опозданий до 25% — «Medium» (средняя), студенты с долей опозданий выше 25% — «Low» (низкая). Скрипт должен вычислить для каждого уровня число студентов, средний балл и долю опозданий. Запустите этот скрипт через терминал и сохраните вывод как student_engagement.json — массив объектов с полями tier, student_count, avg_score и late_submission_rate.

Напишите второй Python-скрипт под названием design_meal_plans.py, который читает student_engagement.json и nutritional_guidelines.csv, а затем формирует рекомендации по планам питания для каждого уровня. Студенты с высокой вовлечённостью получают план «Control» без специальных блюд. Студенты со средней вовлечённостью получают план «Partial» только с рекомендациями для лёгкого приёма пищи (категория «гарнир»). Студенты с низкой вовлечённостью получают «Full Meal Plan» с рекомендациями и для лёгкого приёма пищи (категория «гарнир»), и для основного приёма пищи (категория «горячее»). Скрипт должен использовать категории рецептов «гарнир» и «горячее» из результатов кулинарной базы и оценить суточную стоимость на основе ограничений из campus_dining_budget.json. Запустите этот скрипт и сохраните вывод как meal_recommendations.json.

Создайте облачную таблицу (google_sheet) с названием "Nutrition Study Data" из двух листов. Первый лист «Student_Engagement» должен содержать колонки Tier, Student_Count, Avg_Score и Late_Submission_Rate, по одной строке на каждый уровень (High, Medium, Low), отсортированные по названию уровня. Второй лист «Meal_Plans» должен содержать колонки Tier, Intervention_Type, Recommended_Meals и Estimated_Daily_Cost. Уровень High получает вмешательство «Control» со значением «None» для блюд и стоимостью 0. Уровень Medium получает вмешательство «Partial» с перечисленными рецептами категории «гарнир» и оценкой стоимости из бюджета. Уровень Low получает вмешательство «Full Meal Plan» с перечисленными рецептами категорий «гарнир» и «горячее» и оценкой стоимости из бюджета.

Создайте в корпоративной базе знаний teamly страницу с заголовком "Wellness Pilot Program", документирующую программу. В теле страницы оформите таблицу или структурированный список по трём группам — по одной на каждый уровень вовлечённости — с полями Student_Group, Intervention_Type, Enrollment, Start_Date и Status. Группа уровня High имеет вмешательство Control с числом студентов высокой вовлечённости, дата старта 2026-04-01, статус Planning. Группа уровня Medium имеет вмешательство Partial с числом студентов средней вовлечённости, та же дата старта и статус. Группа уровня Low имеет вмешательство Full Meal Plan с числом студентов низкой вовлечённости, та же дата старта и статус. Обязательно укажите все три типа вмешательства (Control, Partial, Full Meal Plan) и соответствующие числа студентов.
