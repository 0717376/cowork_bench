Вы — HR-директор, разрабатывающий стратегию удержания сотрудников на основе исследований. Ваша цель — объединить внутренние данные о персонале с выводами академических исследований, чтобы подготовить комплексный анализ и план действий.

Начните с чтения файла retention_targets.pdf в вашей рабочей директории, который содержит целевые показатели удержания по отделам и допустимые пороги текучести. Также изучите файл analysis_methodology.md, описывающий, как рассчитывается оценка риска ухода (flight risk), и файл department_contacts.json со списком контактов руководителей отделов.

Запросите корпоративное хранилище данных ClickHouse — данные о сотрудниках из базы данных HR Analytics, конкретно из таблицы employees. Сотрудник с «риском ухода» (flight risk) определяется как тот, у кого оценка удовлетворённости работой (job satisfaction) равна 4 или ниже И рейтинг производительности (performance rating) равен 4 или выше. Это высокоэффективные, но неудовлетворённые сотрудники, которые с наибольшей вероятностью уйдут. Для каждого из семи отделов (Engineering, Finance, HR, Operations, R&D, Sales и Support) определите общую численность (headcount), число сотрудников с риском ухода, процент риска ухода и среднюю оценку удовлетворённости работой.

Напишите Python-скрипт под названием flight_risk_analysis.py в вашей рабочей директории, который подключается к хранилищу данных, вычисляет метрики риска ухода по каждому отделу и выводит результаты в файл flight_risk_analysis.json. Запустите этот скрипт с помощью инструментов командной строки.

Затем выполните поиск в архиве академических статей по запросу «employee retention», чтобы найти релевантные исследования. Прочитайте аннотации трёх наиболее релевантных статей (тех, которые конкретно посвящены стратегиям удержания, прогнозным моделям или программам вмешательства). Игнорируйте статьи на не связанные темы.

Напишите второй Python-скрипт под названием synthesis.py, который читает файл flight_risk_analysis.json, объединяет его с собранными вами выводами исследований и выводит файл synthesis.json, содержащий стратегии по отделам, основанные как на внутренних данных, так и на выводах исследований. Запустите этот скрипт с помощью инструментов командной строки.

Создайте книгу Excel под названием Retention_Strategy.xlsx в вашей рабочей директории с тремя листами.

Первый лист должен называться Department_Analysis и содержать столбцы department, headcount, flight_risk_count, flight_risk_pct и avg_satisfaction. Включите по одной строке на каждый отдел с вычисленной статистикой. Округлите flight_risk_pct до двух знаков после запятой и avg_satisfaction до двух знаков после запятой.

Второй лист должен называться Research_Summary и содержать столбцы paper_title, key_finding и applicability_score. Включите по одной строке для каждой из трёх релевантных статей, которые вы выбрали. applicability_score должен быть целым числом от 1 до 10, оценивающим, насколько выводы статьи применимы к ситуации организации.

Третий лист должен называться Action_Plan и содержать столбцы department, strategy, priority и estimated_cost. Включите по одной строке на каждый отдел. Приоритет (priority) назначается следующим образом: отделы с процентом риска ухода выше 8.3% получают приоритет «High», отделы со значением от 7.9% до 8.3% включительно получают приоритет «Medium», а отделы ниже 7.9% получают приоритет «Low». strategy должна быть кратким описанием рекомендуемой меры по удержанию для этого отдела. estimated_cost должен быть оценкой стоимости реализации стратегии в денежном выражении.

Создайте документ Word под названием Retention_Strategy_Report.docx в вашей рабочей директории. При вызове инструмента Word передавайте АБСОЛЮТНЫЙ путь к файлу .docx внутри рабочей директории. Документ должен иметь заголовок «Employee Retention Strategy Report». Он должен содержать раздел с кратким резюме (executive summary), описывающий общую ситуацию с риском ухода по всей организации; раздел анализа данных, представляющий выводы по каждому отделу, включая численность и процент риска ухода; раздел выводов исследований, обобщающий три релевантные академические статьи и их ключевые идеи; и раздел рекомендаций, перечисляющий конкретные стратегии для каждого отдела с назначенными им приоритетами.

Создайте в базе знаний Teamly страницу под названием «Retention Action Items». На этой странице перечислите все семь отделов, и для каждого отдела укажите: название отдела (Department), стратегию удержания (Strategy), приоритет (Priority — одно из значений High, Medium или Low), статус (Status — значение «Not Started») и ответственного (Owner). Приоритет устанавливайте на основе порогов процента риска ухода, описанных выше. Для всех отделов установите Status равным «Not Started». В качестве Owner укажите имя руководителя отдела из файла department_contacts.json. Стратегия (Strategy) должна совпадать со стратегией, которую вы указали на листе Action_Plan книги Excel.
