Нам нужно провести анализ удержания клиентов для нашего интернет-магазина на платформе InSales. Цель — выявить клиентов, которые рискуют уйти (churn), и заранее принять меры по удержанию самых ценных из них. Наш API отраслевых бенчмарков доступен по адресу http://localhost:30206/api/benchmarks.json и предоставляет ключевые метрики, с которыми нам следует сравнивать наши показатели.

Начните с чтения файла retention_strategy.md в рабочей директории, где описаны правила сегментации клиентов и подход к удержанию. Также прочитайте файл email_template.txt, содержащий шаблон, который мы используем для писем по удержанию клиентов. Изучите файл customer_tiers.xlsx, где указаны пороги по сумме покупок для каждого сегмента, и файл outreach_history.csv, в котором фиксируются предыдущие кампании по удержанию.

Затем получите отраслевые бенчмарки из API по адресу http://localhost:30206/api/benchmarks.json. Это даст вам среднюю частоту повторных покупок, среднюю частоту заказов, среднюю пожизненную ценность клиента (LTV) и порог оттока в днях. Порог оттока определяет, сколько дней неактивности означают, что клиент находится в зоне риска.

Далее выгрузите все данные о клиентах из нашего интернет-магазина InSales, включая для каждого клиента общую сумму покупок, количество заказов и историю заказов. Для каждого клиента, совершившего хотя бы одну покупку, определите дату его последнего заказа и вычислите, сколько дней прошло с этого заказа (в качестве опорной даты используйте 7 марта 2026 года). Отнесите каждого клиента к сегменту на основе его общей суммы покупок: клиенты, потратившие более $500, — это "VIP"; потратившие от $100 до $500 (включая $100) — "Regular"; потратившие менее $100 — "New". Пометьте клиента как находящегося "в зоне риска" (at risk), если число дней с момента последнего заказа превышает порог оттока в 90 дней из бенчмарков.

Создайте книгу Excel с именем Customer_Retention.xlsx в рабочей директории, содержащую три листа. Первый лист должен называться "Customer Analysis" со столбцами: Customer_Name (имя и фамилия), Email, Total_Spent (округлено до 2 знаков после запятой), Order_Count, Last_Order_Date (в формате YYYY-MM-DD), Days_Since_Last (целое число дней с момента последнего заказа), Segment (VIP, Regular или New) и At_Risk (Yes или No). Включите всех клиентов, совершивших хотя бы одну покупку. Отсортируйте по Total_Spent по убыванию.

Второй лист должен называться "Retention Metrics" с двумя столбцами: Metric и Value. Включите следующие строки: Total_Active_Customers (количество клиентов с хотя бы одним заказом), VIP_Count, Regular_Count, New_Count, At_Risk_Total (общее число клиентов в зоне риска по всем сегментам), At_Risk_VIP_Count (число VIP-клиентов в зоне риска), Store_Repeat_Rate (доля клиентов с более чем одним заказом, в процентах, округлено до 1 знака после запятой), Benchmark_Repeat_Rate (из API, 35.0), Store_Avg_LTV (средняя сумма total_spent по всем клиентам с покупками, округлено до 2 знаков после запятой) и Benchmark_Avg_LTV (из API, 250.0).

Третий лист должен называться "Action Plan" со столбцами: Customer_Name, Email, Segment, Days_Inactive, Recommended_Action. Включите только VIP-клиентов, находящихся в зоне риска, отсортированных по Days_Inactive по убыванию. Значение Recommended_Action для каждого должно быть "Send personalized retention email".

Отправьте персонализированное письмо по удержанию каждому VIP-клиенту в зоне риска. Используйте шаблон письма из рабочей директории, заменив плейсхолдеры на имя клиента и число дней его неактивности. Отправляйте каждое письмо с адреса retention@store.com на электронный адрес клиента. Тема письма должна быть "Мы скучаем по вам, [FirstName]!", где [FirstName] — имя клиента.

Наконец, создайте Google-таблицу с заголовком "Retention Outreach Log" с листом "Outreach", содержащим столбцы: Date (используйте 2026-03-07 для всех записей), Customer_Name, Email, Segment, Days_Inactive и Action_Taken (используйте "Retention email sent" для каждой записи). Включите по одной строке для каждого VIP-клиента в зоне риска, которому было отправлено письмо.
