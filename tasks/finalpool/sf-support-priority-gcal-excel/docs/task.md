Мне нужно проанализировать данные по обращениям в нашу службу поддержки и наладить регулярный процесс мониторинга. Пожалуйста, выгрузите разбивку обращений по приоритету из нашего облачного хранилища данных ClickHouse и создайте Excel-файл с именем Support_SLA_Analysis.xlsx. Файл должен содержать два листа.

Первый лист с именем Priority Analysis должен содержать колонки Priority, Ticket_Count, Avg_Response_Hours (округлённое до 2 знаков после запятой) и SLA_Status, который равен Met, если среднее время ответа укладывается в лимит SLA для данного уровня приоритета, и Breached в противном случае. Лимиты SLA: четыре часа для приоритета high, восемь часов для приоритета medium и двадцать четыре часа для приоритета low. Отсортируйте строки по количеству обращений (Ticket_Count) по убыванию.

Второй лист с именем Summary должен содержать колонки Metric и Value со строками Total_Tickets, Priorities_Met_SLA (количество уровней приоритета, для которых SLA выполнен) и Most_Common_Priority.

ВАЖНО: значения уровней приоритета (High, Medium, Low), а также значения SLA_Status (Met / Breached) записывайте в ячейки ровно как английские строковые токены данных — они являются данными, а не подлежат переводу. Имена файлов, листов и колонок также сохраняйте на английском в точности как указано.

После создания файла анализа запланируйте в календаре ежемесячные обзорные встречи на следующие три месяца, начиная с апреля 2026 года, размещая каждую встречу на первый понедельник месяца (то есть 2026-04-06, 2026-05-04 и 2026-06-01) с названием Monthly Support SLA Review, в интервале с 9:00 до 10:00.
