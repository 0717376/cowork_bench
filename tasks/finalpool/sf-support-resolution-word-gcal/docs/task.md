Привет! Подготовь, пожалуйста, для нашей команды поддержки отчёт по времени решения тикетов. Все наши обращения в поддержку хранятся в базе данных ClickHouse, и мне нужна понятная картина того, сколько времени уходит на решение тикетов — в разбивке по уровню приоритета и по категории проблемы.

Вот что нужно сделать:

Сначала запроси данные нашего центра поддержки и найди все решённые тикеты. Для каждого уровня приоритета (High, Medium, Low) посчитай общее количество решённых тикетов, среднее время решения в часах (разница между созданием тикета и его решением), минимальное время решения в часах и максимальное время решения в часах. Округли все значения в часах до 2 знаков после запятой. Отсортируй результаты по приоритету в алфавитном порядке.

Затем сделай аналогичную разбивку по категории проблемы (поле типа проблемы, ISSUE_TYPE). Для каждой категории получи общее количество тикетов и среднее время решения в часах, округлённое до 2 знаков после запятой. Отсортируй по среднему времени решения по убыванию.

Создай документ Word под названием Resolution_Analysis.docx в рабочей директории. В документе должен быть заголовок верхнего уровня "Support Ticket Resolution Analysis" (Анализ времени решения тикетов поддержки). Затем добавь заголовок раздела "Resolution Times by Priority" (Время решения по приоритету), а после него — таблицу со столбцами Priority, Total_Tickets, Avg_Hours, Min_Hours, Max_Hours. Затем добавь ещё один заголовок раздела "Resolution Times by Category" (Время решения по категориям), а после него — таблицу со столбцами Category, Total_Tickets, Avg_Hours. После таблиц добавь итоговый абзац, в котором укажи общее количество решённых тикетов, общее среднее время решения, а также среднее время решения для каждого уровня приоритета.

Важно: Word MCP не сохраняет рабочую директорию между вызовами, поэтому при создании/сохранении документа передавай АБСОЛЮТНЫЙ путь к Resolution_Analysis.docx внутри рабочей директории.
