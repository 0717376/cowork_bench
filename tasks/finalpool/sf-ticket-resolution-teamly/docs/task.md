Нашей службе поддержки нужно выявить обращения с медленным временем отклика, которые рискуют нарушить целевые показатели SLA. Обратитесь к системе тикетов поддержки в ClickHouse и найдите все обращения, у которых время отклика израсходовало более 50 процентов отведённых часов SLA (то есть где время отклика, делённое на часы SLA, превышает 0,5). Такие обращения считаются «медленными по отклику» (slow response) и требуют внимания руководства.

Данные находятся в ClickHouse в таблице `sf_data."SUPPORT_CENTER__PUBLIC__TICKETS"` (логическая база `SUPPORT_CENTER`). Используйте столбцы `PRIORITY`, `RESPONSE_TIME_HOURS`, `SLA_HOURS`, `CUSTOMER_SATISFACTION`. Названия таблиц, столбцов и значения приоритета (`High`, `Medium`, `Low`) оставляйте на английском языке как есть.

Сгруппируйте эти медленные обращения по уровню приоритета и для каждого приоритета вычислите: общее количество таких обращений, среднее время отклика в часах, среднее число часов SLA, средний процент использования SLA (utilization = время отклика / часы SLA * 100) и среднюю оценку удовлетворённости клиентов (CSAT).

Создайте в Teamly базу знаний (раздел/пространство) под названием "Slow Response Tickets Tracker" с одной страницей для каждого уровня приоритета (High, Medium, Low). Каждая страница должна быть озаглавлена названием приоритета и содержать статистику по этому приоритету: количество медленных обращений, среднее время отклика, среднее число часов SLA, средний процент использования SLA и среднюю оценку удовлетворённости клиентов.

После настройки трекера в Teamly отправьте письмо на адрес support.manager@company.example.com с темой "Slow Response Tickets Report". В теле письма кратко изложите: общее количество медленных обращений по всем приоритетам, общий процент медленных обращений от всех тикетов, а также разбивку по уровням приоритета с указанием количества и среднего процента использования SLA для каждого.
