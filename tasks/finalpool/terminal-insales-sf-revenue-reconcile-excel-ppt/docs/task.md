Финансовой команде нужна межсистемная сверка выручки между интернет-магазином InSales и корпоративным хранилищем данных ClickHouse. В рабочей директории есть PDF-файл Reconciliation_Guidelines.pdf, который описывает процедуры аудита и допустимые пороги отклонений. Также есть JSON-файл audit_thresholds.json, который задаёт уровни допусков для пометки расхождений. Пожалуйста, прочитайте оба файла, чтобы понять критерии аудита.

Сначала выгрузите данные о заказах и выручке из интернет-магазина InSales. Получите сводку заказов, сгруппированных по статусу (status), включая количество заказов и суммарную выручку для каждого статуса. Особое внимание уделите заказам в статусах completed и processing, поскольку они представляют подтверждённую выручку. Также рассчитайте средний чек (average order value) для этих подтверждённых заказов.

Затем запросите данные о заказах из корпоративного хранилища данных ClickHouse. Получите сводку заказов, сгруппированных по способу доставки (Ship_Mode), включая количество заказов и суммарную выручку для каждого. Также сделайте разбивку по регионам клиентов (Region). Рассчитайте средний чек по всем заказам из хранилища.

Напишите и запустите Python-скрипт reconciliation_engine.py в рабочей директории. Скрипт должен читать из двух JSON-файлов, которые вы создадите (wc_revenue_data.json и sf_revenue_data.json, содержащие данные из обеих систем). Скрипт должен вычислить следующие межсистемные сравнительные метрики: общее количество заказов в каждой системе, суммарную выручку каждой системы, средний чек каждой системы и процентное отклонение (percentage variance) между двумя системами по каждой из этих метрик. Он должен выводить файл reconciliation_results.json.

Создайте Excel-файл Revenue_Reconciliation.xlsx в рабочей директории с четырьмя листами.

Первый лист должен называться WC_Summary со столбцами Status, Order_Count и Total_Revenue. Включите все статусы заказов из интернет-магазина, отсортированные по Total_Revenue по убыванию. Добавьте итоговую строку (total) внизу, суммирующую только подтверждённые заказы (completed плюс processing).

Второй лист должен называться SF_Summary со столбцами Region, Order_Count и Total_Revenue. Включите все пять регионов из хранилища данных, отсортированные по Total_Revenue по убыванию. Добавьте итоговую строку (total) внизу. Также включите второй раздел ниже с пустой строкой-разделителем, показывающий разбивку по способу доставки со столбцами Ship_Mode, Order_Count и Total_Revenue.

Третий лист должен называться Cross_Audit со столбцами Source, Metric, WC_Value, SF_Value, Variance_Pct (процентная разница, вычисляемая как модуль разности, делённый на среднее двух значений, умноженный на 100, округлённый до одного знака после запятой) и Flag (установите "REVIEW", если отклонение превышает 50 процентов, иначе "OK"). Метрики для сравнения: Total_Orders, Total_Revenue и Avg_Order_Value.

Четвёртый лист должен называться Recommendations со столбцами Finding_Number (от 1 до 5), Category, Finding и Recommended_Action. Приведите пять выводов на основе результатов сверки, например: разница в масштабе между двумя системами, разрыв в выручке, наблюдения о ценовой стратегии и предложения по согласованию систем.

Создайте презентацию PowerPoint под названием Reconciliation_Presentation.pptx в рабочей директории, содержащую как минимум пять слайдов. Первый слайд должен быть титульным с заголовком "Revenue Reconciliation Audit Report". Второй слайд должен суммировать данные интернет-магазина. Третий слайд должен суммировать показатели хранилища данных. Четвёртый слайд должен представлять межсистемное сравнение (cross-audit) с конкретными процентами отклонений. Пятый слайд должен перечислять ключевые выводы и рекомендуемые дальнейшие шаги.

Примечание: при работе с инструментами Excel и PowerPoint указывайте АБСОЛЮТНЫЙ путь к файлам .xlsx и .pptx внутри рабочей директории.
