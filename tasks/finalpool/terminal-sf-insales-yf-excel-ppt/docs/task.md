Генеральный директор хочет получить единую кросс-платформенную аналитическую панель (business intelligence dashboard), которая объединяет корпоративные данные о продажах, показатели электронной коммерции и состояние фондового рынка в один executive-брифинг. В рабочей директории есть два файла для начала работы: dashboard_spec.json, который определяет отслеживаемые KPI вместе с их целевыми значениями и весами, и industry_benchmarks.csv, содержащий отраслевые бенчмарки для сравнения. Также есть файл last_quarter_summary.xlsx с показателями предыдущего квартала, которые мы используем для сравнения год к году.

Начните с чтения dashboard_spec.json, чтобы понять, какие KPI нужны и как они должны рассчитываться. Затем прочитайте industry_benchmarks.csv для получения бенчмарков. Далее извлеките корпоративные данные о продажах из хранилища данных компании (data warehouse). Нам нужна суммарная выручка в разбивке по клиентским сегментам: Государственный, Корпоративный, Малый и средний бизнес, Частные клиенты. Также извлеките помесячную динамику выручки за последние 12 месяцев из таблицы заказов вместе с топ-5 товаров по выручке.

Из платформы электронной коммерции извлеките выручку по категориям товаров, а также метрики привлечения клиентов: общее число клиентов, общее число заказов, средний чек (average order value) и суммарную выручку электронной коммерции.

Для рыночного контекста посмотрите три инструмента: индекс Московской биржи (IMOEX.ME) для оценки общего состояния рынка, Ozon (OZON.ME) как бенчмарк сектора электронной коммерции и Банк ВТБ (VTBR.ME) для оценки состояния финансового сектора. Получите последнюю цену и любую доступную информацию об изменении с начала года (year-to-date).

Напишите Python-скрипт под названием merge_data.py, который объединяет все три потока данных в единый файл unified_metrics.json. Затем напишите второй скрипт health_score.py, который вычисляет составной индекс состояния бизнеса (composite business health score), используя следующие веса: показатель корпоративной выручки — 40%, рост электронной коммерции — 30%, рыночная позиция — 30%. Показатель enterprise рассчитывается как фактическая суммарная корпоративная выручка, делённая на целевое значение 3200000, с ограничением сверху на уровне 100. Показатель e-commerce основан на суммарной выручке электронной коммерции, делённой на целевое значение 70000, с ограничением 100. Рыночный показатель равен 80, если последнее закрытие индекса IMOEX.ME выше 3000, иначе 50. Сохраните результат в business_health.json с полями enterprise_score, ecommerce_score, market_score и composite_score, все округлённые до 1 знака после запятой.

Создайте Excel-книгу под названием Executive_Dashboard.xlsx с четырьмя листами. Первый лист "Enterprise_Sales" должен содержать столбцы Segment, Revenue, Customer_Count, Avg_Deal_Size и Order_Count, отсортированные по Segment в алфавитном порядке. Revenue округляется до 2 знаков после запятой. Avg_Deal_Size — это Revenue, делённая на Order_Count, округлённая до 2 знаков. Второй лист "Ecommerce_Performance" должен содержать столбцы Category, Revenue, Product_Count и Avg_Order_Value. Перечислите каждую категорию товаров с её суммарной выручкой по заказам, округлённой до 2 знаков, числом уникальных проданных товаров в этой категории и общим средним чеком по всем заказам, округлённым до 2 знаков. Отсортируйте по Revenue по убыванию. Третий лист "Market_Context" должен содержать столбцы Symbol, Latest_Price и Asset_Name. Включите строки для трёх инструментов, упомянутых выше. Latest_Price — это самая последняя цена закрытия. Asset_Name должно быть "MOEX Russia Index" для индекса, "Ozon Holdings" для OZON.ME и "VTB Bank" для VTBR.ME. Четвёртый лист "Unified_Dashboard" должен содержать столбцы KPI_Name, Current_Value, Target, Status и Source. Включите следующие строки: Enterprise_Total_Revenue с суммой выручки всех сегментов в качестве Current_Value и 3200000 в качестве Target; Ecommerce_Total_Revenue с суммарной выручкой электронной коммерции и 70000 в качестве цели; Market_Index с последним закрытием IMOEX.ME и 3000 в качестве цели; Top_Segment с названием сегмента с наибольшей выручкой в качестве Current_Value и без цели; Top_Ecommerce_Category с названием категории электронной коммерции с наибольшей выручкой в качестве Current_Value и без цели. Status должен быть "On Track", если Current_Value достигает или превышает Target, "Below Target" в противном случае, или "N/A" для строк без цели. Source должно быть "Enterprise DW", "E-Commerce Platform", "Financial Markets", "Enterprise DW" и "E-Commerce Platform" соответственно.

Создайте презентацию PowerPoint под названием Executive_Briefing.pptx из восьми слайдов. Слайд 1 — титульный слайд с заголовком "Cross-Platform Business Intelligence Dashboard" и подзаголовком "Executive Briefing". Слайд 2 — "Business Health Score" с составным индексом и его тремя компонентами. Слайд 3 — "Enterprise Sales Overview" с обзором выручки по сегментам. Слайд 4 — "E-Commerce Performance" с разбивкой выручки по категориям. Слайд 5 — "Market Context" с тремя рыночными индикаторами. Слайд 6 — "Year-over-Year Comparison" со сравнением текущих метрик со значениями предыдущего квартала из файла summary. Слайд 7 — "Risk Factors" с указанием всех KPI, которые ниже цели. Слайд 8 — "Strategic Priorities" с рекомендуемыми действиями на основе анализа.

Наконец, напишите Python-скрипт под названием generate_alerts.py, который читает unified_metrics.json и сверяет каждый KPI с его целью из dashboard_spec.json. Для любого KPI, текущее значение которого ниже его цели, должна формироваться запись alert с полями kpi_name, current_value, target, gap_percentage, округлённым до 1 знака после запятой. Сохраните alerts в alerts.json в виде списка.
