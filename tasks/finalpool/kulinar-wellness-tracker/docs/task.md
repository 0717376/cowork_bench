Я — коуч по здоровью и составляю велнес-программу для клиента. Мне нужно оценить рецепты по критериям здорового питания, создать инструменты отслеживания и подготовить документацию программы.

Начните с посещения велнес-портала по адресу http://localhost:30235, чтобы узнать ежедневные велнес-цели. На главной странице показаны цели по сну, потреблению воды, физической активности, порциям овощей и максимальному количеству сахара. Затем перейдите на http://localhost:30235/nutrition.html, чтобы изучить подробные рекомендации по группам продуктов на один приём пищи, включая рекомендуемое количество порций для злаков, белка, овощей и фруктов.

Далее изучите кулинарную базу рецептов (используйте инструменты Kulinar, например `mcp_kulinar_getAllRecipes` и `mcp_kulinar_getRecipesByCategory`) и выберите не менее восьми рецептов из не менее чем трёх разных категорий (доступные категории: салат, закуска, суп, горячее, гарнир, выпечка, десерт, напиток). Для каждого рецепта проанализируйте его ингредиенты, чтобы оценить, сколько порций овощей он даёт и сколько сахара может содержать. Оцените каждый рецепт по велнес-шкале от 1 до 10, где более высокий балл означает больше порций овощей, меньше сахара и лучшее соответствие рекомендациям по группам продуктов с велнес-портала.

Изучите файл с целями по здоровью (wellness_goals.md) и шаблон отслеживания (tracking_template.json) в рабочей директории, чтобы понять конкретные цели клиента и желаемый формат отслеживания.

Создайте три результата в рабочей директории.

Во-первых, создайте книгу Excel с именем Wellness_Tracker.xlsx с тремя листами. Первый лист должен называться "Recipe Wellness Score" со столбцами Recipe_Name, Category, Veggie_Servings, Est_Sugar_g, Wellness_Score и Recommended. Названия рецептов (Recipe_Name) должны быть на русском языке — так, как они называются в базе Kulinar. Категории (Category) также берите из базы Kulinar (на русском). Veggie_Servings должен оценивать число порций овощей в рецепте. Est_Sugar_g должен оценивать содержание сахара в граммах. Wellness_Score должен быть числом от 1 до 10. Recommended должен быть Yes для рецептов с Wellness_Score 7 или выше и No в остальных случаях. Включите не менее восьми рецептов. Второй лист должен называться "Weekly Plan" со столбцами Day, Meal, Recipe_Name, Wellness_Score. Спланируйте пять дней приёмов пищи по два приёма в день, выбирая только из рекомендованных рецептов (Recommended=Yes). Третий лист должен называться "Progress Metrics" со столбцами Metric и Target, включая Daily_Veggie_Target (5 порций), Daily_Sugar_Limit (25 г), Weekly_Recommended_Meals (целевое число рекомендованных приёмов пищи) и Average_Wellness_Score (средний балл по всем проанализированным рецептам, округлённый до одного знака после запятой).

Во-вторых, создайте документ Word с именем Wellness_Guide.docx, который служит руководством по программе для клиента. Документ должен объяснять велнес-цели с портала, описывать, как оценивались рецепты, перечислять лучшие рекомендованные рецепты с их баллами и обоснованием выбора, представлять недельный план питания с пояснениями и включать советы по поддержанию велнес-целей. Документ должен содержать не менее четырёх абзацев. Текст пишите на русском языке. При вызове инструмента Word передавайте абсолютный путь к файлу Wellness_Guide.docx внутри рабочей директории.

В-третьих, создайте страницу в базе знаний Teamly с заголовком "Client Wellness Dashboard" (Панель велнес-клиента). Страница должна содержать сводку велнес-программы, лучшие рекомендованные рецепты с их баллами, обзор недельного плана питания и рекомендации по отслеживанию прогресса. Страница должна иметь содержательный текст объёмом не менее пяти блоков (абзацев/разделов). Создайте её в пространстве базы знаний коуча по здоровью.
