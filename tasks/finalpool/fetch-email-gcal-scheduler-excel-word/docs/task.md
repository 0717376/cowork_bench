Помогите мне с анализом email-планировщика. Доступны внешние эталонные (бенчмарк) данные, которые нужно получить, и внутренние метрики компании за 1 квартал 2026 года.

Сначала получите данные:
- Внутренние метрики: http://localhost:30331/api/data.json (поле data: список объектов с полями key и value).
- Внешние бенчмарки: http://localhost:30331/api/benchmarks.json (поле benchmarks: список объектов с полями key и external_benchmark).
Сопоставляйте внутренние значения с бенчмарками по совпадающему полю key.

С помощью терминала создайте и запустите Python-скрипт email_scheduler_processor.py в рабочей директории. Скрипт должен прочитать собранные данные из созданных вами JSON-файлов, выполнить анализ и записать результат в email_scheduler_results.json.

Создайте Excel-файл Scheduler_Report.xlsx с тремя листами.
1. Лист Data_Analysis — основная таблица сравнения. Используйте РОВНО следующие заголовки столбцов (на английском): Item (название подразделения/измерения, то есть значение key), Internal_Value (наша внутренняя метрика), External_Benchmark (внешний бенчмарк), Gap (разрыв). Разрыв вычисляется как Gap = Internal_Value - External_Benchmark. Отсортируйте строки по столбцу Item в алфавитном порядке.
2. Лист Metrics — сводные показатели в двух столбцах с заголовками Metric и Value. Включите как минимум строки Total_Items (общее число строк в Data_Analysis) и Avg_Gap (среднее значение столбца Gap).
3. Лист Recommendations — приоритетные действия по итогам анализа разрыва, со столбцами Priority и Action. В первую очередь обратите внимание на подразделение с наибольшим отрицательным разрывом (см. analysis_guide.md).

Также создайте Word-документ Scheduler_Analysis.docx с разделами: краткое резюме (executive summary), ключевые выводы (key findings) и рекомендации (recommendations). ВАЖНО: при сохранении документа через инструмент word передавайте АБСОЛЮТНЫЙ путь к файлу Scheduler_Analysis.docx внутри рабочей директории.
