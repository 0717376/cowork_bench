Вы — исследователь в ML-команде. Ваш научный руководитель попросил вас составить каталог тщательно отобранных видео о ML-технологиях с канала под названием ML Tech Reviews на видеоплатформе. У этого канала есть отдельный плейлист из 7 видео, каждое из которых посвящено конкретному open-source проекту в области машинного обучения с соответствующим репозиторием кода. Вам нужно каталогизировать эти видео, сопоставить их со связанными научными статьями из вашей локальной библиотеки статей, создать страницу базы знаний для команды в корпоративной базе `teamly`, собрать Excel-трекер исследований и отправить итоговое письмо.

Ваш рабочий процесс должен выглядеть следующим образом.

Во-первых, получите плейлист ML Tech Reviews с видеоплатформы. Получите все элементы плейлиста. Для каждого из 7 видео извлеките полную информацию, включая название (title), описание (description), теги (tags), дату публикации (publication date) и число просмотров (view count). Из описания каждого видео извлеките ссылку на репозиторий, указывающую на сервис хостинга кода. Определите основную ML-тему или технику, которой посвящено видео, например Flash Attention, LoRA, Diffusion Models, RLHF, Mixture of Experts, Quantization или Vector Databases, на основе названия и тегов.

Во-вторых, выполните поиск в вашей локальной библиотеке статей по научным статьям, связанным с каждой из 7 определённых вами ML-тем. Для каждой темы извлеките данные статьи, включая идентификатор статьи, название, авторов и дату публикации.

В-третьих, создайте Excel-файл с именем ML_Research_Tracker.xlsx. Создайте лист с именем Videos со следующими столбцами в указанном порядке: Video_ID, Title, Topic, GitHub_URL, Published_Date, View_Count. Заполните его, добавив по одной строке на каждое из 7 видео. Создайте второй лист с именем Papers со столбцами: ArXiv_ID, Title, Authors, Published, Topic, Related_Video_ID. Заполните его, добавив по одной строке на каждую найденную статью, связав каждую статью с соответствующим видео через идентификатор видео. Создайте третий лист с именем Summary со столбцами: Topic, Video_Count, Paper_Count, GitHub_Repos_Count. Заполните его, добавив по одной строке на каждую тему с агрегированными счётчиками.

В-четвёртых, создайте новую страницу в корпоративной базе знаний `teamly` с заголовком «ML Tech Research Hub». Добавьте вводный абзац, описывающий назначение страницы и каталогизируемый канал. Затем оформите в теле страницы структурированный каталог под заголовком «Research Items», содержащий все элементы (видео и статьи). Для каждого элемента укажите следующие поля: Name (название), Type (со значением Video или Paper), Topic (тема), URL (ссылка) и Status (со значением New, In Review или Archived). Добавьте запись для каждого из 7 видео и для каждой найденной статьи.

В-пятых, отправьте письмо на адрес research@lab.edu. Тема письма должна отсылать к каталогу ML-исследований. В теле письма кратко изложите количество каталогизированных видео, количество найденных статей, охваченные темы и упомяните, что полные сведения содержатся в файле ML_Research_Tracker.xlsx и в корпоративной базе знаний `teamly`.

Конфигурационный файл исследования в вашей рабочей директории называется research_config.md и описывает подход к каталогизации. Файл сопоставления тем topic_mapping.json содержит список из 7 ожидаемых ML-тем, которые следует использовать в качестве справочника.

Когда все результаты будут готовы, завершите задачу, ответив без вызова инструментов.
