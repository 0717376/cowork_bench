Стандарты обработки тикетов

  • Подтверждать получение всех тикетов с высоким приоритетом в течение 30 минут с момента создания
  • Предоставлять обновления статуса каждые 2 часа для тикетов с высоким приоритетом
  • Документировать анализ первопричин для всех тикетов типа «Ошибка» и «Инцидент»
  • Связываться с клиентом в течение 24 часов после решения обращения
  • Немедленно эскалировать тикеты, превышающие пороги SLA, руководителю команды

Стандарты коммуникации

  • Использовать профессиональный и эмпатичный язык во всех взаимодействиях с клиентами
  • Указывать шаги для воспроизведения технических проблем
  • Указывать ориентировочные сроки решения в первом ответе
  • Предлагать альтернативные решения, когда основное исправление задерживается

Методология оценки качества

Каждый агент оценивается по трём измерениям:

Оценка времени отклика: Доля тикетов, по которым первый отклик был в пределах целевого времени для соответствующего приоритета.

Оценка удовлетворённости: Средняя оценка удовлетворённости клиентов по всем решённым тикетам.

Оценка решения: Доля назначенных тикетов, успешно решённых.

Агент помечается на улучшение, если хотя бы одно измерение опускается ниже своего целевого порога.
