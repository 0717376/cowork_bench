Нашей команде контроля качества требуется комплексная оценка качества обработки тикетов поддержки. Первый шаг — зайти на внутренний портал стандартов качества по адресу http://localhost:30213 и изучить текущие целевые метрики по времени отклика, удовлетворённости клиентов и доле решённых обращений. На портале две страницы: главная с целевыми числами и страница стандартов с описанием лучших практик.

После изучения целевых показателей качества с портала выгрузите все данные о тикетах поддержки из нашей базы данных ClickHouse (через инструменты ClickHouse, схема sf_data, таблица SUPPORT_CENTER__PUBLIC__TICKETS). Мы отслеживаем тикеты с такими полями, как уровень приоритета (PRIORITY), время отклика (RESPONSE_TIME_HOURS), оценка удовлетворённости клиента (CUSTOMER_SATISFACTION), тип проблемы (ISSUE_TYPE) и статус решения (STATUS). Также у нас есть отдельная таблица с перечнем агентов поддержки, их командами и уровнями квалификации. Проанализируйте каждого агента, вычислив его среднее время отклика по всем тикетам, среднюю оценку удовлетворённости клиента и общую долю решённых обращений в процентах. Обратите внимание: в нашей системе человек, указанный в тикете (поле REPORTER), — это тот, кто его обработал. Сравните метрики каждого агента с целевыми показателями портала и определите, какие из них для каждого агента не дотягивают до стандарта.

Создайте книгу Excel с именем QA_Assessment.xlsx в вашей рабочей директории. Первый лист должен называться "Agent Scorecard" и содержать столбцы: имя агента, его среднее время отклика в часах, округлённое до двух знаков после запятой, средняя удовлетворённость клиентов, округлённая до двух знаков, доля решённых обращений в процентах, округлённая до одного знака, и текстовое поле со списком всех направлений, по которым агент не дотягивает до опубликованных стандартов (например, "Response Time, Satisfaction", если оба показателя ниже целевого; допускается и русское написание, например "Время отклика, Удовлетворённость"). Отсортируйте строки по имени агента в алфавитном порядке.

Второй лист должен называться "Issue Type Analysis" и разбивать показатели по типам проблем. Включите столбцы: название типа проблемы, общее количество тикетов этого типа, среднее время отклика в часах, округлённое до двух знаков, и средняя оценка удовлетворённости, округлённая до двух знаков. Отсортируйте по алфавиту по типу проблемы.

Третий лист должен называться "Summary" и содержать два столбца — Metric и Value. Включите следующие сводные метрики (ключи в столбце Metric оставьте на английском, ровно как указано): Total_Tickets (общее количество тикетов по всем данным), Overall_Avg_Response_Time (общее среднее время отклика, округлённое до двух знаков), Overall_Avg_Satisfaction (общая средняя удовлетворённость, округлённая до двух знаков), Overall_Resolution_Rate (общая доля решённых в процентах, округлённая до одного знака), Agents_Below_Satisfaction_Target (количество агентов, чья средняя удовлетворённость ниже целевого значения портала) и Agents_Below_Response_Target (количество агентов, чьё общее среднее время отклика превышает разумный ориентир, а именно целевое время отклика для среднего приоритета с портала).

Также изучите markdown-файл с чек-листом контроля качества (qa_checklist.md), предоставленный в вашей рабочей директории, для понимания методологии оценки, и проверьте CSV-файл с ростером команды (team_roster.csv) для списка агентов. Обратите внимание, что в CSV могут быть проблемы с качеством данных, например лишние пустые строки или неоднородное форматирование, которые следует учитывать.

Наконец, создайте Google-форму с заголовком "QA Self Assessment Survey", с помощью которой агенты смогут провести самооценку своей работы по выявленным слабым направлениям. Форма должна содержать пять вопросов. Первый вопрос запрашивает имя агента в виде короткого текстового поля и обязателен. Второй вопрос спрашивает, к какой команде относится агент, в виде вопроса с вариантами ответа, где варианты соответствуют командам из таблицы агентов (Specialist, Tier 2, Tier 3), и обязателен. Третий вопрос просит оценить собственную результативность по времени отклика по шкале — вопрос с вариантами ответа со следующими опциями (оставьте их на английском): "Exceeds Target", "Meets Target", "Below Target", "Needs Improvement"; обязателен. Четвёртый вопрос спрашивает о самой большой сложности в достижении целевых показателей качества в виде абзацного текстового поля и обязателен. Пятый вопрос просит предложить улучшения процесса контроля качества в виде абзацного текстового поля и не обязателен.

Примечание: при создании файла Excel QA_Assessment.xlsx передавайте абсолютный путь к файлу внутри вашей рабочей директории.
