Вы — финансовый аудитор, отвечающий за сверку (реконсиляцию) данных по продажам между корпоративным хранилищем данных компании и интернет-магазином. Компания использует две независимые системы учёта заказов. Хранилище данных фиксирует все транзакции по заказам в нескольких регионах, а интернет-магазин на платформе InSales обслуживает онлайн-продажи со своим собственным каталогом товаров. Ваша задача — выполнить тщательную межсистемную сверку.

Начните с прочтения документа Audit_Procedures.pdf в вашей рабочей директории. В нём описана методология сверки и конкретные метрики, которые необходимо рассчитать.

Сначала запросите хранилище данных, чтобы получить сводку по всем заказам. Сгруппируйте заказы по их статусу (Delivered, Shipped, Processing, Cancelled) и рассчитайте общее количество и суммарную выручку (в рублях) для каждого статуса. Также вычислите общую среднюю стоимость заказа по всем заказам и общее число уникальных клиентов.

Затем запросите систему интернет-магазина InSales, чтобы получить все товары и их характеристики. Получите список всех заказов магазина, включая суммы и статусы заказов. Также получите данные о клиентах магазина.

Теперь напишите и запустите Python-скрипт под названием audit_analysis.py в вашей рабочей директории. Этот скрипт должен читать данные из обеих систем (значения, которые вы собрали, можно захардкодить) и рассчитывать следующее: общее количество заказов из хранилища данных, общее количество заказов из интернет-магазина, разницу между ними, среднюю стоимость заказа в каждой системе, а также разбивку выручки хранилища данных по категориям в разрезе способов доставки (Economy, Express, Next Day, Standard).

Создайте рабочую книгу Excel под названием Order_Audit_Report.xlsx в вашей рабочей директории с четырьмя листами.

Первый лист должен называться DW_Summary и содержать столбцы Status, Order_Count и Total_Revenue. Включите по одной строке для каждого статуса заказа из хранилища данных: Cancelled, Delivered, Processing и Shipped. Добавьте итоговую строку (Total) внизу.

Второй лист должен называться Store_Summary и содержать столбцы Metric и Value. Включите строки: Total Products (количество товаров в магазине), Total Orders, Total Customers и Average Order Value (средняя стоимость заказа) из интернет-магазина.

Третий лист должен называться ShipMode_Breakdown и содержать столбцы Ship_Mode, Order_Count и Total_Revenue. Включите строки для способов доставки Economy, Express, Next Day и Standard из хранилища данных.

Четвёртый лист должен называться Reconciliation и содержать столбцы Metric, DW_Value, Store_Value и Difference. Включите строки, сравнивающие Total Order Count (общее количество заказов), Average Order Value (среднюю стоимость заказа) и Total Revenue (суммарную выручку) между двумя системами. Столбец Difference — это значение хранилища данных минус значение магазина.

Наконец, создайте документ Word под названием Audit_Findings.docx в вашей рабочей директории. Документ должен иметь заголовок "Order Reconciliation Audit Report". Включите раздел, описывающий сводку хранилища данных с количеством заказов и выручкой по статусам. Включите ещё один раздел о сводке интернет-магазина. Добавьте раздел с разбивкой по способам доставки, показывающий, как выручка распределяется по способам доставки. Завершите разделом с выводами сверки, в котором выделены ключевые различия между двумя системами и приведены рекомендации по их выравниванию.
