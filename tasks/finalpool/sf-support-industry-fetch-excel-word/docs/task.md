Мне нужно сравнить показатели работы нашего центра поддержки с отраслевыми эталонными значениями (бенчмарками). По адресу http://localhost:30340/api/support_benchmarks.json доступен API с отраслевыми бенчмарками: среднее время отклика, доля решённых обращений и оценки удовлетворённости клиентов по нашему сектору.

Выгрузите наши фактические метрики поддержки из корпоративного хранилища данных ClickHouse: объёмы обращений по приоритетам, среднее время отклика и статистику по решённым обращениям. Также получите отраслевые эталонные данные из API.

Через терминал создайте и запустите Python-скрипт под названием benchmark_analyzer.py в рабочей директории. Скрипт должен читать файлы support_metrics.json и industry_benchmarks.json (сначала создайте оба файла), сравнивать наши показатели с отраслевыми эталонами и выводить результат в файл benchmark_comparison.json.

Создайте файл Excel под названием Support_Benchmark_Report.xlsx с тремя листами. Первый лист Our_Performance должен содержать столбцы Priority, Ticket_Count, Avg_Response_Hours (округлить до 1 знака после запятой) и Pct_of_Total (округлить до 1 знака после запятой), отсортированные по Priority (Critical, High, Medium, Low). Второй лист Industry_Comparison должен содержать столбцы Metric, Our_Value, Industry_Avg, Variance и Status ("Above", если мы превосходим эталон, иначе "Below"). Включите метрики Avg_Response_Time, Resolution_Rate, Customer_Satisfaction и Tickets_Per_Agent. Третий лист Action_Plan должен содержать столбцы Priority_Area, Current_Gap и Recommended_Action, не менее чем с 3 областями для улучшения.

Создайте документ Word под названием Benchmark_Analysis.docx с заголовком "Support Center Benchmark Analysis" и разделами "Performance Overview", "Industry Comparison" и "Improvement Recommendations" с конкретными данными из анализа.

Сохраните все выходные файлы в рабочей директории. Названия столбцов в Excel и заголовки разделов могут оставаться на английском, как указано выше. ВАЖНО: при создании файлов .xlsx и .docx через соответствующие MCP-инструменты передавайте АБСОЛЮТНЫЙ путь к файлу внутри рабочей директории, так как эти инструменты не сохраняют текущую рабочую директорию.
