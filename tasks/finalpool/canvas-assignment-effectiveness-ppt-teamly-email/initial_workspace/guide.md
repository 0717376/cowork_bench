# Руководство по анализу эффективности заданий

## Обзор
В этом руководстве описано, как рассчитать индекс дискриминации (Discrimination Index, DI) и другие метрики эффективности для заданий курсов.

## Пошаговая методика

### 1. Определите целевые курсы
Сосредоточьтесь на курсах конкретного семестра (например, осень 2014). Курсы осеннего семестра 2014 года имеют коды, оканчивающиеся на '2014J'.

### 2. Соберите данные о работах
Для каждого курса соберите все оценённые работы с непустыми баллами. Вам понадобятся:
- Assignment ID, название и points_possible
- user_id студента и его балл (score)
- Учитывайте только работы, где workflow_state = 'graded'

### 3. Определите зачисленных студентов
Считайте число зачисленных студентов как количество уникальных студентов, у которых есть хотя бы одна оценённая работа в курсе.

### 4. Ранжируйте студентов по общей успеваемости
Для каждого курса независимо:
1. Рассчитайте общий средний балл каждого студента (среднее всех его баллов за задания)
2. Отсортируйте студентов по этому среднему от меньшего к большему
3. Выделите верхние 27% и нижние 27% студентов

### 5. Рассчитайте индекс дискриминации
Для каждого задания:
1. Возьмите средний балл группы верхних 27% по этому заданию
2. Возьмите средний балл группы нижних 27% по этому заданию
3. DI = (top_27_avg - bottom_27_avg) / points_possible

### 6. Рассчитайте остальные метрики
- **Completion Rate** = (число работ / число зачисленных) × 100
- **Mean Score** = среднее всех баллов по заданию
- **Score Std Dev** = популяционное стандартное отклонение баллов

### 7. Классифицируйте эффективность
- DI > 0.3 → "Good"
- 0.15 ≤ DI ≤ 0.3 → "Acceptable"
- DI < 0.15 → "Poor"

### 8. Отметьте задания для доработки
Задание требует доработки, если:
- Оно оценено как "Poor" (DI < 0.15), ИЛИ
- Его процент выполнения ниже 70%

Приоритет:
- "High", если выполнены ОБА условия
- "Medium", если выполнено только ОДНО условие
