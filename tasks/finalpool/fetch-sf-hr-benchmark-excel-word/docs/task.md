Наша компания готовится к ежегодному пересмотру вознаграждений, и вице-президент по персоналу (HR) попросил подготовить детальное сравнение наших внутренних данных о зарплатах с публикуемыми отраслевыми эталонными показателями (бенчмарками). У нас есть сервис отраслевого бенчмаркинга, который предоставляет данные о зарплатах через REST API по адресу http://localhost:30201/api/benchmarks.json и возвращает эталонную информацию о зарплатах по семи отделам, включая среднюю зарплату, медианную зарплату и перцентильные диапазоны.

Мы также ведём собственные кадровые записи в корпоративном хранилище данных ClickHouse с информацией о зарплатах примерно 50 000 сотрудников по семи отделам: НИОКР, Финансы, Операции, Продажи, Кадры, Поддержка и Инженерия.

Начните с получения отраслевых эталонных данных из API. Затем выгрузите фактические данные о зарплатах сотрудников из хранилища данных с группировкой по отделам. Для каждого отдела вычислите среднюю зарплату, медианную зарплату, минимальную зарплату, максимальную зарплату и общую численность персонала (headcount).

Создайте файл Excel под названием "HR_Salary_Benchmark.xlsx" в рабочей директории с тремя листами. Первый лист должен называться "Industry Benchmarks" и содержать эталонные данные, полученные из API, со столбцами Department, Benchmark_Avg_Salary, Benchmark_Median_Salary, Benchmark_P25 и Benchmark_P75. Второй лист должен называться "Internal Data" и включать столбцы Department, Internal_Avg_Salary, Internal_Median_Salary, Min_Salary, Max_Salary и Headcount, заполненные фактическими данными из хранилища. Третий лист должен называться "Variance Analysis" и содержать столбцы Department, Benchmark_Avg, Internal_Avg, Variance_Amount (разница между внутренней средней зарплатой и эталонной средней) и Variance_Pct (процентное отклонение, рассчитанное как Variance_Amount, делённое на эталонную среднюю и умноженное на 100). Положительное отклонение означает, что наши зарплаты выше эталона, отрицательное — что ниже эталона.

Далее создайте документ Word под названием "Salary_Benchmark_Report.docx" в рабочей директории. Это резюме для руководства должно включать: заголовок; вводный абзац, описывающий цель анализа; раздел, перечисляющий каждый отдел с его эталонной и внутренней средней зарплатой и процентом отклонения; раздел, определяющий, какие отделы находятся выше эталона, а какие ниже; и заключительный абзац с рекомендациями.

Наконец, отправьте письмо от hr-analytics@company.com на адрес vp-hr@company.com с темой "Annual Salary Benchmark Analysis Report". В теле письма обобщите ключевые выводы, упомянув отдел с наибольшим положительным отклонением (наиболее выше эталона) и отдел с наибольшим отрицательным отклонением (наиболее ниже эталона).

Сохраните все выходные файлы в рабочей директории. ВАЖНО: при создании файлов .xlsx и .docx через соответствующие MCP-инструменты передавайте АБСОЛЮТНЫЙ путь к файлу внутри рабочей директории, так как эти инструменты не сохраняют текущую рабочую директорию.
