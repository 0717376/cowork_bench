Вы — аналитик по компенсациям, готовящий обзор бонусов отдела продаж с учётом рыночной корректировки. Комитет по компенсациям хочет понять, нужно ли корректировать текущие выплаты бонусов с учётом внешних рыночных условий перед финализацией бонусного цикла за Q4.

Начните с чтения файла compensation_policy.md в вашей рабочей директории. В нём описана текущая методика расчёта бонусов, по которой бонусы назначаются исходя из совокупной выручки, сгенерированной в каждом регионе продаж. Совокупная выручка каждого региона определяет бонусный тир для всех сотрудников отдела продаж, закреплённых за этим регионом. Также прочитайте файл bonus_tiers.csv с точными порогами тиров и соответствующими процентами бонуса. Наконец, прочитайте файл budget_cap.json, в котором указан максимальный совокупный пул бонусов, утверждённый компанией.

Запросите хранилище данных ClickHouse, чтобы получить совокупную выручку и количество заказов в разбивке по региону клиента из таблиц orders и customers. Также вам нужно выгрузить список сотрудников отдела продаж (department = "Продажи") из HR-таблицы employees, включая их имя, оклад, оценку эффективности (performance rating), стаж (years of experience) и должность (role). Логические базы данных называются SALES_DW и HR_ANALYTICS; имена таблиц orders, customers, employees сохраняются на английском, имена колонок — в ВЕРХНЕМ РЕГИСТРЕ.

Напишите и запустите Python-скрипт compute_bonuses.py в вашей рабочей директории. Скрипт должен определить бонусный тир каждого региона на основе совокупной региональной выручки и порогов из bonus_tiers.csv. Для каждого сотрудника отдела продаж вычислите его текущий бонус как оклад, умноженный на процент бонуса его региона. Закрепляйте сотрудников за регионами по принципу round-robin: отсортируйте сотрудников по имени в алфавитном порядке и циклически распределите их по пяти регионам, отсортированным в алфавитном порядке (Азиатско-Тихоокеанский регион, Ближний Восток, Европа, Латинская Америка, Северная Америка). Сохраните результат как current_bonuses.json, где для каждого сотрудника указаны имя (name), регион (region), оклад (salary), процент бонуса (bonus percentage) и сумма бонуса (bonus amount).

Далее получите данные о котировках акций для Сбербанка (тикер SBER.ME) — рыночного ориентира, Газпрома (GAZP.ME) и ЛУКОЙЛа (LKOH.ME). Для каждого получите самую свежую цену закрытия и цену закрытия примерно годичной давности (период 1y), чтобы вычислить процентное изменение год к году (year-over-year). Цены указаны в рублях.

Напишите и запустите второй Python-скрипт market_adjustment.py. Скрипт должен вычислить коэффициент рыночной корректировки на основе изменения котировок Сбербанка (SBER.ME) год к году: если SBER.ME вырос более чем на 10% год к году — коэффициент 0.9 (снизить бонусы на 10%, так как сильный рынок уже выгоден для компенсаций по другим каналам); если SBER.ME упал более чем на 10% — коэффициент 1.1 (повысить бонусы для удержания талантов на слабом рынке); в остальных случаях коэффициент 1.0. Примените этот коэффициент к текущему бонусу каждого сотрудника, чтобы получить скорректированный бонус. Также проверьте, не превышает ли совокупный пул скорректированных бонусов бюджетный лимит из budget_cap.json, и если превышает — пропорционально масштабируйте все бонусы вниз так, чтобы уложиться в лимит. Сохраните результаты как market_adjusted_bonuses.json.

Проверьте общий календарь на наличие существующих событий на неделе с 9 по 13 марта 2026 года в промежутке с 9:00 до 17:00 по восточному времени (Eastern Time). Найдите первый свободный 90-минутный слот, который не конфликтует с существующими событиями, и запланируйте новое событие с названием "Q4 Compensation Review Meeting" и описанием "Review market-adjusted sales bonuses for Q4" на это время.

Создайте облачную таблицу (spreadsheet) с названием "Sales Compensation Analysis" с тремя листами. Первый лист должен называться "Rep_Performance" с колонками Employee Name, Region, Revenue in Region, Order Count in Region, Salary, Performance Rating, Current Bonus. Второй лист должен называться "Market_Adjustment" с колонками Ticker, Current Price, Year Ago Price, YoY Change Percent, Adjustment Factor. Включите строки для SBER.ME, GAZP.ME и LKOH.ME, но коэффициент корректировки (Adjustment Factor) должен быть заполнен только в строке SBER.ME, так как корректировка основана исключительно на котировках Сбербанка. Третий лист должен называться "Adjusted_Bonuses" с колонками Employee Name, Region, Current Bonus, Market Factor, Adjusted Bonus, Difference from Current.

В памятке должен быть заголовок "Q4 Compensation Review Memo", за которым следуют разделы: Background / Введение (объяснение цели обзора), Methodology / Методология (описание расчёта бонусов и подхода к рыночной корректировке), Market Analysis / Анализ рынка (резюме годового изменения по каждой акции и результирующий коэффициент корректировки), Regional Performance Summary / Сводка по регионам (совокупная выручка и количество заказов по каждому региону), Budget Impact / Влияние на бюджет (сравнение совокупных текущих бонусов и совокупных скорректированных бонусов с указанием, был ли превышен бюджетный лимит), и Recommendations / Рекомендации (не менее двух рекомендаций для комитета относительно выплат бонусов).
