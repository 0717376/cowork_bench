Вы — вице-президент по продажам, который готовит квартальный обзор стратегии, подкреплённый исследованиями. Ваша цель — объединить внутренние данные о результатах продаж с академическими исследованиями по методологиям продаж, чтобы построить убедительный обзор стратегии, основанный на доказательствах.

Начните с чтения файла strategy_brief.txt в вашей рабочей директории — в нём изложены стратегические цели и целевые показатели выручки на следующий квартал. Также прочитайте файл sales_team_roster.csv, чтобы понять, кто из региональных руководителей продаж за что отвечает и какие у них индивидуальные планы. Наконец, изучите historical_benchmarks.json, где содержатся данные о результатах за последние четыре квартала для контекста.

Из корпоративного хранилища данных ClickHouse сделайте запрос к базе продаж, чтобы получить данные о результатах по регионам. Для каждого региона извлеките суммарную выручку (total revenue), число заказов (order count) и средний чек (average order value). Также выгрузите данные о клиентах, сгруппированные по сегментам, включая число клиентов (customer count), суммарные траты (total spend) и среднюю пожизненную ценность клиента (average lifetime value) по каждому сегменту. Затем запросите данные о результатах по товарам, чтобы выделить топ-5 и худшие 5 товаров по выручке, включая информацию об их категории.

Напишите и запустите Python-скрипт под названием analyze_sales_gaps.py в вашей рабочей директории. Скрипт должен вычислить оценку изменения выручки год к году (year-over-year) для каждого региона, сравнивая текущие итоги с поквартальными бенчмарками из файла исторических данных. Он должен определить, какие регионы оказались ниже медианной выручки, как отстающие, и присвоить им тег «Priority Focus». Скрипт должен выдать файл sales_gaps.json, содержащий данные о результатах по регионам вместе с тегами приоритета.

Выполните поиск в базе академических статей по темам «sales optimization» и «customer segmentation strategies». Вы должны найти несколько статей по этим темам. Отфильтруйте только те статьи, в аннотациях которых обсуждаются практические стратегии продаж, повышение результативности по регионам или оптимизация пожизненной ценности клиентов. Игнорируйте любые статьи, посвящённые несвязанным областям, таким как здравоохранение или чисто теоретическое машинное обучение.

Напишите и запустите второй Python-скрипт match_recommendations.py. Этот скрипт должен прочитать файл sales_gaps.json и сведения о релевантных статьях, а затем сформировать research_recommendations.json. Для каждого выявленного отстающего региона или сегментного разрыва скрипт должен подобрать хотя бы одну подкреплённую исследованием рекомендацию из аннотаций статей, указав название статьи, ключевой вывод и ожидаемую область влияния.

Проверьте общий календарь на наличие существующих событий в ближайшие две недели, чтобы найти свободное время. Затем создайте событие в календаре под названием «Q1 Sales Strategy Review», запланированное в первый доступный 90-минутный интервал, который не пересекается с существующими событиями. Событие должно проходить в рабочее время с 9:00 до 17:00 в будний день. Добавьте описание, в котором упомянуто, что встреча охватывает обзор результатов по регионам и план действий, основанный на исследованиях.

Создайте Excel-книгу под названием Sales_Strategy_Analysis.xlsx в вашей рабочей директории с четырьмя листами. Первый лист с именем Regional_Performance должен содержать колонки region, revenue, order_count, avg_order_value, yoy_change_pct и priority_tag. Включите по одной строке на регион, при этом в колонке priority_tag для регионов ниже медианной выручки указывается «Priority Focus», а для остальных — «On Track». Второй лист с именем Segment_Analysis должен содержать колонки segment, customer_count, total_spend и avg_ltv. Включите по одной строке на сегмент клиентов. Третий лист с именем Research_Insights должен содержать колонки paper_title, key_finding, applicable_region и expected_impact. Включите по одной строке на каждую исследовательскую рекомендацию, сопоставленную с отстающей областью. Четвёртый лист с именем Action_Items должен содержать колонки action, region, timeline, owner и research_basis. Включите хотя бы один пункт плана действий на каждый регион со статусом «Priority Focus», при этом значение owner берётся из sales_team_roster.csv, а research_basis ссылается на конкретное название статьи.

Создайте презентацию PowerPoint под названием Sales_Strategy_Presentation.pptx в вашей рабочей директории с шестью слайдами. Первый слайд должен быть титульным с заголовком «Q1 Sales Strategy Review» и текущей датой в подзаголовке. Второй слайд должен представлять обзор результатов по регионам с ключевыми цифрами выручки. Третий слайд должен показывать анализ сегментов с числом клиентов и данными о тратах. Четвёртый слайд должен резюмировать выводы исследований и то, как они применимы к выявленным разрывам. Пятый слайд должен представлять план действий с конкретными инициативами, привязанными к исследованиям. Шестой слайд должен показывать график ключевых вех на квартал.

Наконец, напишите и запустите Python-скрипт generate_summary.py, который читает файлы sales_gaps.json и research_recommendations.json, а затем формирует файл executive_summary.txt в вашей рабочей директории. Резюме должно быть кратким абзацем, в котором отражены: суммарная выручка по всем регионам, количество регионов со статусом «Priority Focus» и число подкреплённых исследованиями рекомендаций.

Важно: при создании или сохранении файлов .xlsx и .pptx передавайте инструментам АБСОЛЮТНЫЙ путь к файлу внутри вашей рабочей директории (например, <workspace_dir>/Sales_Strategy_Analysis.xlsx), так как инструменты Excel и PowerPoint не сохраняют рабочую директорию между вызовами.
