Нашей операционной команде нужен отчёт о том, насколько хорошо мы соблюдаем соглашения об уровне обслуживания (SLA) по тикетам клиентской поддержки. Пожалуйста, выгрузите данные о тикетах из нашего хранилища данных ClickHouse вместе с определениями SLA-политик и определите для каждого уровня приоритета, сколько тикетов уложилось в целевое время ответа, а сколько — нет. Хранилище данных содержит базу данных центра поддержки SUPPORT_CENTER с таблицами TICKETS (тикеты), SLA_POLICIES (SLA-политики) и AGENTS (агенты).

Создайте файл Excel с именем SLA_Compliance_Report.xlsx в рабочей директории с двумя листами. Первый лист должен называться "Compliance" и содержать столбцы Priority, Total_Tickets, Avg_Response_Hours (округлено до двух знаков после запятой), SLA_Target_Hours, Compliant_Count (количество тикетов, у которых время ответа было меньше или равно целевому значению SLA), и Compliance_Rate (процент тикетов, уложившихся в SLA, округлённый до двух знаков после запятой). Отсортируйте строки по SLA_Target_Hours по возрастанию. Обратите внимание: включать нужно только те приоритеты, которые реально присутствуют в данных о тикетах — если для какого-то уровня приоритета, определённого в SLA-политике, нет ни одного тикета, его следует пропустить.

Второй лист должен называться "Summary" со столбцами Metric и Value. Включите следующие строки: Total_Tickets (общий итог по всем приоритетам), Overall_Compliance_Rate (процент по всем тикетам, округлённый до двух знаков после запятой), Worst_Priority (приоритет с самым низким уровнем соблюдения SLA) и Best_Priority (приоритет с самым высоким уровнем соблюдения SLA).

После создания отчёта отправьте письмо с адреса ops-report@company.com на адрес ops-manager@company.com с темой "SLA Compliance Report Summary", в тексте которого содержится общий уровень соблюдения SLA (Overall_Compliance_Rate) и приоритет с наихудшими показателями (Worst_Priority).

Важно: при работе с Excel-файлом через соответствующий инструмент указывайте полный (абсолютный) путь к файлу SLA_Compliance_Report.xlsx внутри рабочей директории, поскольку относительные пути могут не сработать.
