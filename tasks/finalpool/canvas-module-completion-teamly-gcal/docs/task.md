Вы — ассистент преподавателя курса «Creative Computing and Culture», который читается осенью 2014 года (Fall 2014). В системе управления обучением этот курс имеет идентификатор 7 (course ID 7).

Задание состоит из трёх частей.

Часть 1. Получите все модули курса 7 (course 7) и перечислите элементы внутри каждого модуля. Для каждого модуля зафиксируйте название модуля, количество входящих в него элементов и краткое описание того, какие типы элементов в нём присутствуют. У курса 5 модулей: Introduction, Week 1, Week 3, Week 5 и Week 8.

Часть 2. Создайте в базе знаний Teamly страницу-трекер с заголовком «CCC Fall 2014 - Module Tracker» для отслеживания статуса выполнения каждого модуля. На странице разместите таблицу со следующими столбцами: Module_Name (название модуля), Item_Count (количество элементов), Type_Summary (краткое описание типов элементов) и Status (статус). Добавьте по одной строке для каждого из 5 модулей курса. Значение Item_Count для каждого модуля должно соответствовать фактическому числу элементов, полученному из Canvas. В качестве начального статуса задайте для всех модулей Status = «Not Started».

Допустимые значения столбца Status: «Not Started», «In Progress», «Completed».

Часть 3. Запланируйте 4 еженедельные встречи по обзору модулей в Google Calendar. Каждое событие назовите «CCC Module Review Meeting». Первую встречу запланируйте на первую субботу после момента запуска задания, затем по одной в каждую следующую субботу (всего 4 субботы). Каждое событие должно длиться 1 час и начинаться в 10:00.

После выполнения всего перечисленного отправьте письмо на адрес teaching-team@ccc.example.com от имени ta@university.example.com с темой «Module Review Setup - Creative Computing and Culture (Fall 2014)». В теле письма должны быть: ссылка на страницу-трекер в Teamly или её название, даты 4 запланированных встреч в календаре и краткая сводка по 5 отслеживаемым модулям.

В начальной рабочей директории находится файл Module_Review_Template.md — справочный шаблон о том, как проводить обзоры модулей.
