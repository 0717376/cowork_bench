Я — руководитель отдела обучения и развития (L&D), и мне нужно провести оценку потребности в обучении, сопоставив данные об эффективности сотрудников с доступными у нас обучающими программами. Цель — выявить сотрудников с низкой эффективностью и подобрать им подходящие курсы, оставаясь в рамках бюджета.

Начните с чтения PDF-документа с политикой обучения (Training Policy) в вашей рабочей директории. В нём изложены общий годовой бюджет, максимальные расходы на одного сотрудника и правила приоритизации при выборе сотрудников для обучения. Обратите особое внимание на пороговое значение оценки эффективности (performance rating), определяющее право на обучение.

Затем загрузите каталог обучающих курсов из нашего внутреннего API по адресу http://localhost:30214/api/training_catalog.json. Он возвращает JSON-массив доступных курсов, у каждого из которых есть идентификатор курса (course_id), название (title), целевой департамент (department), уровень (level), длительность в часах (duration_hours) и стоимость (cost). Некоторые курсы привязаны к конкретному департаменту, другие помечены как общие для всех департаментов (department = "All").

Также прочитайте JSON-файл department_heads.json с руководителями департаментов в вашей рабочей директории, который сопоставляет название каждого департамента с адресом электронной почты его руководителя.

Теперь сделайте запрос к нашей HR-базе данных в ClickHouse. У нас есть таблица сотрудников `sf_data.HR_ANALYTICS__PUBLIC__EMPLOYEES` с полями, включающими имя сотрудника (EMPLOYEE_NAME), департамент (DEPARTMENT), должность (ROLE), оценку эффективности (PERFORMANCE_RATING) по шкале от 1 до 5, стаж работы в годах (YEARS_EXPERIENCE), уровень образования (EDUCATION_LEVEL) и зарплату. Выявите всех сотрудников, у которых оценка эффективности равна наименьшему возможному баллу (PERFORMANCE_RATING равно 1), — именно они являются наивысшим приоритетом для обучения согласно политике.

Обратите внимание: значения департаментов в базе данных указаны на русском языке: Инженерия, Финансы, Кадры, Операции, НИОКР, Продажи, Поддержка. В каталоге курсов поле `department` указано на английском. Соответствие между ними следующее: Инженерия = Engineering, Финансы = Finance, Кадры = HR, Операции = Operations, НИОКР = R&D, Продажи = Sales, Поддержка = Support. Используйте это соответствие, чтобы подобрать курс департамента из каталога. В итоговой таблице Excel в столбце Department указывайте русские названия департаментов ровно так, как они хранятся в базе данных.

Создайте книгу Excel под названием Training_Needs.xlsx с тремя листами. Передавайте инструменту Excel АБСОЛЮТНЫЙ путь к файлу Training_Needs.xlsx внутри вашей рабочей директории.

Первый лист должен называться "Low Performers" со столбцами Employee_Name, Department, Role, Performance_Rating, Years_Experience и Education_Level. Включите всех сотрудников с наименьшей оценкой эффективности, отсортированных по департаменту в алфавитном порядке, а внутри каждого департамента — по имени сотрудника.

Второй лист должен называться "Recommended Courses" и показывать, какой курс рекомендован для каждого департамента. Включите столбцы Department, Course_Title, Course_Cost, Employee_Count (количество сотрудников с низкой эффективностью в этом департаменте) и Total_Cost (стоимость курса, умноженная на количество сотрудников). Для каждого департамента: если существует курс, привязанный к этому департаменту, выберите наименее дорогой из них. Если курса для конкретного департамента нет, назначьте наименее дорогой общий курс (помеченный как доступный для всех департаментов, department = "All"). Отсортируйте по департаменту в алфавитном порядке. Названия курсов (Course_Title) оставляйте на английском ровно так, как они указаны в каталоге.

Третий лист должен называться "Budget Summary" со столбцами Metric и Value. Включите следующие строки: Total_Eligible_Employees (общее количество сотрудников с наименьшей оценкой), Total_Departments (количество департаментов, в которых есть хотя бы один подходящий сотрудник), Estimated_Total_Cost (сумма всех значений Total_Cost с листа Recommended Courses), Available_Budget (используйте пятьсот тысяч в качестве годового бюджета на обучение), Budget_Remaining (Available_Budget минус Estimated_Total_Cost) и Within_Budget — значение "Yes", если оценочная общая стоимость укладывается в доступный бюджет, либо "No" в противном случае.

Наконец, отправьте письмо каждому руководителю департамента, указанному в файле department_heads.json. У каждого письма должна быть тема в формате "Training Plan for [Department]" (где [Department] — название департамента), а в теле письма нужно указать количество сотрудников с низкой эффективностью в этом департаменте, название рекомендованного курса, стоимость на одного сотрудника и общую стоимость для данного департамента. Отправляйте эти письма от имени training-manager@company.com.
