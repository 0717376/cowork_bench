Вы — учебный координатор по оценке успеваемости, отвечающий за мониторинг результатов тестов (quizzes) по нескольким курсам. Вам поручили проанализировать два курса со схожим контингентом студентов: Creative Computing and Culture за осень 2014 (course ID 7) и Data-Driven Design за весну 2013 (course ID 11). По обоим курсам есть тесты, и кафедре нужен комплексный обзор успеваемости.

Авторитетным источником являются данные Canvas: используйте именно курсы с ID 7 и 11. Файлы Program_Guidelines.pdf и analysis_config.json в вашем рабочем каталоге носят справочный характер и могут содержать иллюстративные примеры (другие номера курсов, названия программ и адреса почты) — не используйте их в качестве источника истины. Адреса электронной почты для итогового письма заданы ниже в этой инструкции.

Начните с получения из Canvas всех тестов и оценок по их попыткам для курса 7 и курса 11. Для каждого теста в каждом курсе рассчитайте среднюю оценку (average score) и долю успешных сдач (pass rate), где успешной считается оценка 70 и выше.

Напишите и запустите Python-скрипт, обрабатывающий данные тестов. Скрипт должен определить, у каких тестов средняя оценка ниже 75, и пометить их для вмешательства. Он также должен вычислить общую среднюю оценку по всем тестам обоих курсов вместе взятых и определить разрыв в успеваемости между двумя курсами (разницу их средних оценок на уровне курса).

Создайте опрос для сбора обратной связи с заголовком "Academic Performance Self-Assessment", чтобы собрать мнения студентов. Опрос должен содержать ровно 5 вопросов. Первый вопрос должен спрашивать, какие темы курса были наиболее сложными (свободный текст). Второй должен спрашивать, сколько часов в неделю студент тратит на учёбу, с вариантами ответа "Less than 3", "3 to 5", "5 to 8" и "More than 8". Третий должен спрашивать о предпочитаемом формате обучения с вариантами "Lectures", "Hands-on labs", "Group projects" и "Self-paced online". Четвёртый вопрос должен просить студента оценить общую сложность курса по шкале от 1 до 5. Пятый вопрос должен запрашивать предложения по улучшению (свободный текст).

Для каждого теста со средней оценкой ниже 75 запланируйте в календаре часовую сессию разбора. Назовите каждое событие "Quiz Review: [Quiz Title]", где quiz title — фактическое название теста из системы. Запланируйте первую сессию на 16 марта 2026 года в 14:00, а каждую последующую сессию — на день позже в то же время.

Создайте Excel-файл с именем Quiz_Performance_Report.xlsx в рабочем каталоге, состоящий из трёх листов. Первый лист должен называться "Quiz_Performance" со столбцами Course_ID, Course_Name, Quiz_ID, Quiz_Title, Avg_Score, Pass_Rate и Needs_Review (Yes, если средняя ниже 75, иначе No). Включите все тесты обоих курсов. Второй лист должен называться "Feedback_Summary" и перечислять 5 вопросов опроса со столбцами Question_Number, Question_Text и Question_Type. Третий лист должен называться "Remediation_Schedule" и перечислять запланированные сессии разбора со столбцами Quiz_Title, Session_Date, Start_Time и Duration_Hours.

Наконец, отправьте письмо от coordinator@assessment.example.com на адрес faculty@assessment.example.com с темой "Quiz Performance Analysis - Cross-Course Review". В теле письма укажите общее количество проверенных тестов по обоим курсам, количество помеченных для коррекции (remediation), общую среднюю оценку по программе, а также упомяните, что опрос для сбора обратной связи создан и сессии разбора запланированы.

Ваш рабочий каталог содержит Program_Guidelines.pdf и analysis_config.json для справки.
