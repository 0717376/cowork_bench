# Руководство по аудиту налогового соответствия

## Методика сопоставления ставок

Заказы сопоставляются с налоговыми ставками по полю региона выставления счёта (billing state) из данных заказа. Магазин поддерживает ставки для отдельных регионов. Для всех остальных регионов применяется ставка по умолчанию (федеральная). У этой ставки по умолчанию поле региона пустое в конфигурации налоговых ставок.

Если для региона настроено несколько ставок, их нужно просуммировать, чтобы получить итоговую применимую ставку для этого региона.

## Анализ расхождений

1. Для каждого заказа рассчитайте: Expected Tax = Order Total * (Applicable Rate / 100)
2. Рассчитайте Discrepancy = Actual Tax - Expected Tax
3. Классифицируйте каждый заказ по порогам из PDF с налоговым регламентом
4. Округляйте все денежные значения до 2 знаков после запятой
5. Округляйте ставки до 4 знаков после запятой
6. Округляйте проценты до 1 знака после запятой

## Отчётность по регионам

Агрегируйте результаты на уровне заказов по региону выставления счёта, чтобы определить:
- Суммарную выручку и суммы налога по каждому региону
- Уровень соответствия (процент заказов в пределах допуска $0.50)
- Требуется ли отдельная подача отчётности (суммарный фактический налог > $100)
