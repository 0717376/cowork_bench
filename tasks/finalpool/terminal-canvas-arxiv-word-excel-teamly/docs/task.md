Учебный отдел (деканат) российского вуза хочет оценить, насколько действующие учебные программы курсов соответствуют современным направлениям научных исследований. В рабочей директории есть PDF-файл Curriculum_Review_Guidelines.pdf, в котором описана методология обзора и критерии оценки. Также есть JSON-файл review_criteria.json с пороговыми значениями оценок релевантности. Пожалуйста, сначала изучите оба файла.

Начните с выгрузки информации о курсах из системы управления обучением (LMS). Получите список всех курсов вместе с их модулями. Сосредоточьтесь на уникальных предметах курсов (например, Прикладная аналитика и алгоритмы, Проектирование на основе данных, Основы финансов и т. д.) и определите ключевые тематические области, охватываемые каждым предметом, на основе названий модулей и описаний курсов. Названия предметов берите verbatim — в точности так, как их возвращает LMS (без перевода и перефразирования).

Затем выполните поиск в архиве научных статей по недавним публикациям, связанным с темами курсов. Ищите статьи по анализу данных, машинному обучению, проектированию алгоритмов, подходам на основе данных, финансовому анализу и вычислительным методам. Найдите не менее 5 релевантных статей и зафиксируйте их названия, категории и даты публикации.

С помощью командной строки напишите и запустите Python-скрипт, выполняющий анализ пробелов учебной программы (curriculum gap analysis). Скрипт должен сопоставить каждый уникальный предмет курса с релевантными ему научными статьями, присвоить каждой паре «статья — курс» оценку релевантности от 0 до 10 на основе пересечения ключевых слов между аннотацией статьи и предметной областью курса, а также выявить пробелы, где действующие курсы не охватывают новые исследовательские темы. Сохраните результат анализа в файл curriculum_gap_output.txt в рабочей директории.

Создайте документ Word с именем Curriculum_Enhancement_Proposal.docx в рабочей директории. Документ должен содержать заголовок "Curriculum Enhancement Proposal", введение с пояснением цели обзора, раздел для каждого предмета курса с конкретными рекомендациями на основе связанных научных статей и заключение с обобщением наиболее критичных пробелов учебной программы. При создании документа через MCP-инструмент Word передавайте АБСОЛЮТНЫЙ путь к файлу .docx внутри рабочей директории.

Создайте файл Excel с именем Curriculum_Gap_Analysis.xlsx в рабочей директории с тремя листами. Первый лист должен называться "Course_Topics" со столбцами Course_Subject и Current_Topics, перечисляющими каждый уникальный предмет курса и сводку тем его модулей. Второй лист должен называться "Research_Frontiers" со столбцами Paper_Title, Primary_Category и Relevance_Score, перечисляющими каждую найденную научную статью с общей оценкой релевантности от 1 до 10. Третий лист должен называться "Gap_Matrix" со столбцами Course_Subject, Paper_Title, Relevance_Score и Gap_Type, где Gap_Type равно "Covered", если оценка релевантности 7 и выше, "Partial", если от 4 до 6, или "Gap", если ниже 4. При создании файла через MCP-инструмент Excel передавайте АБСОЛЮТНЫЙ путь к файлу .xlsx внутри рабочей директории.

Наконец, создайте в базе знаний teamly раздел (трекер) с заголовком "Curriculum Review Tracker", оформленный как таблица со столбцами Course Subject, Review Status (одно из значений Not Started, In Progress, Completed), Priority (одно из значений High, Medium, Low) и Notes. Добавьте по одной записи (строке) для каждого уникального предмета курса: для всех записей Review Status должен быть равен Not Started, а Priority определяется тем, сколько исследовательских пробелов выявлено для данного предмета (>=3 пробелов — High, >=1 — Medium, 0 — Low).
