Вы — аналитик службы поддержки, которому поручено сравнить показатели работы службы поддержки вашей компании с отраслевыми стандартами. Внешняя исследовательская компания опубликовала отраслевые эталонные показатели удовлетворённости клиентов (CSAT), доступные в виде JSON API по адресу http://localhost:30408/api/csat_benchmarks.json. Эти данные содержат среднеотраслевые показатели удовлетворённости и целевые значения времени отклика в разбивке по уровню приоритета тикетов.

Начните с получения эталонных данных CSAT через API. Данные содержат отраслевые средние значения для тикетов приоритета High, Medium и Low, включая целевые показатели удовлетворённости (target_satisfaction), допустимые пороги времени отклика (target_response_hours) и эталонное время разрешения (target_resolution_hours).

Затем выполните запрос к базе данных тикетов поддержки (хранилище данных ClickHouse) для получения фактических показателей вашей компании. Нужно получить общее количество тикетов, среднюю оценку удовлетворённости клиентов, среднее время отклика в часах, а также количество разрешённых и неразрешённых тикетов — всё в разбивке по уровню приоритета (High, Medium, Low). Также получите данные по каждому отдельному агенту поддержки: количество его тикетов, средняя оценка удовлетворённости и среднее время отклика. Данные находятся в схеме sf_data, в таблицах SUPPORT_CENTER__PUBLIC__TICKETS и SUPPORT_CENTER__PUBLIC__AGENTS (значения PRIORITY и STATUS остаются на английском: High/Medium/Low; имена агентов в столбце AGENT_NAME — на русском).

Напишите и выполните Python-скрипт, который проведёт анализ соответствия SLA. Скрипт должен сравнить фактические показатели вашей компании с отраслевыми эталонами по каждому уровню приоритета. Для каждого приоритета вычислите разрыв по удовлетворённости (ваша оценка минус эталон), разрыв по времени отклика (ваше время минус целевое значение) и определите статус соответствия SLA: "Compliant", если и удовлетворённость, и время отклика соответствуют эталонам; "At Risk", если один из показателей близок к пороговому значению; "Non-Compliant", если хотя бы один показатель значительно не достигает цели. Пороговые значения для оценки соответствия указаны в reporting_template.json (satisfaction_tolerance, response_time_tolerance_pct). Также определите 3 лучших и 3 худших агентов по оценке удовлетворённости.

Создайте презентацию PowerPoint с именем Support_Performance_Review.pptx в рабочем каталоге, содержащую не менее 7 слайдов. Первый слайд — титульный, с заголовком "Support Performance Benchmark Review" и текущей датой. Второй слайд должен представлять эталонные данные отрасли, полученные через API. Третий слайд — общие показатели тикетов вашей компании по приоритетам. Четвёртый слайд — сравнение показателей компании с отраслевыми эталонами с выделением разрывов. Пятый слайд — статус соответствия SLA для каждого уровня приоритета. Шестой слайд — рейтинг производительности агентов. Седьмой слайд — ключевые выводы и рекомендации по улучшению.

ВАЖНО: при вызове инструмента pptx указывайте АБСОЛЮТНЫЙ путь к файлу Support_Performance_Review.pptx внутри рабочего каталога (инструмент pptx не сохраняет текущий рабочий каталог).

Отправьте письмо с адреса analytics@support-team.example.com на адрес managers@support-team.example.com с темой "Support Performance Benchmark Analysis". Текст письма должен резюмировать ключевые выводы, включая общий показатель удовлетворённости в сравнении с отраслевым эталоном, какие уровни приоритета соответствуют SLA, имена лучших агентов и области, требующие улучшения.

В вашем рабочем каталоге находятся файлы SLA_Policy_Reference.pdf с внутренними целевыми показателями SLA компании и reporting_template.json с настройками форматирования.
