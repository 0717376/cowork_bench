Помоги с анализом результатов опроса. Доступны внешние эталонные (benchmark) данные: зайди на http://localhost:30330 и извлеки оттуда нужные показатели (категории и значения Typical_Duration_Min).

Используй терминал, чтобы создать и запустить Python-скрипт `notion_survey_processor.py` в рабочей директории. Скрипт должен читать собранные данные из создаваемых тобой JSON-файлов, выполнять анализ и записывать результат в `notion_survey_results.json`.

Создай Excel-файл `Survey_Report.xlsx` с тремя листами. Первый лист `Data_Analysis` должен содержать основную сравнительную таблицу. Второй лист `Metrics` — сводку ключевых метрик. Третий лист `Recommendations` — список приоритетных действий.

Лист `Data_Analysis` должен содержать следующие колонки: `Item` (основное измерение — название категории), `Internal_Value` (наши внутренние значения метрики), `External_Benchmark` (внешнее эталонное значение, взятое с дашборда http://localhost:30330) и `Gap` (разница между ними). Правило расчёта: `Gap = Internal_Value - External_Benchmark`. Значения `External_Benchmark` обязательно бери из живого дашборда (колонка Typical_Duration_Min), а не придумывай. Внутренние значения `Internal_Value` определи самостоятельно как разумные оценки. Отсортируй строки по алфавиту по колонке `Item`.

Лист `Metrics` должен содержать две колонки — `Metric` и `Value` — со сводными показателями: общее число позиций `Total_Items` (равно числу строк данных на листе `Data_Analysis`) и средний разрыв `Avg_Gap` (среднее по колонке `Gap`), а также другие ключевые статистики.

Лист `Recommendations` должен содержать колонки `Priority` и `Action` и перечислять приоритетные действия на основе анализа разрывов.

Также создай Word-документ `Survey_Analysis.docx` с разделами: краткое резюме (executive summary), ключевые выводы и рекомендации. ВАЖНО: при создании Word-документа передавай инструменту АБСОЛЮТНЫЙ путь к `Survey_Analysis.docx` внутри рабочей директории (Word MCP теряет текущую директорию).

Создай в Teamly страницу с заголовком, содержащим слово «Dashboard» (например, «Notion Survey Dashboard»), с кратким резюме анализа на русском языке.

Создай форму (Forms) с заголовком, содержащим слово «Feedback» (например, «Notion Survey Feedback»), хотя бы с одним вопросом — для сбора отзывов команды по результатам анализа.
