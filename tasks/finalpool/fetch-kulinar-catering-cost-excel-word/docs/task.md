Мне нужна помощь с анализом питательной ценности блюд для кейтеринга. Сначала загрузите данные с внешнего источника нормативов питания по адресу http://localhost:30319/api/data.json и извлеките дневные нормы по нутриентам (раздел nutrition_guidelines, в частности норму по белку Protein — daily_g).

Затем подберите рецепты из нашей кулинарной базы рецептов (kulinar). Выберите одну категорию блюд и возьмите не менее 5 рецептов этой категории. Названия рецептов и их категория ДОЛЖНЫ браться именно из кулинарной базы рецептов (kulinar), а не выдумываться.

Для каждого выбранного рецепта оцените его пищевую ценность: калорийность (Calories, ккал на порцию) и содержание белка (Protein_g, граммы белка на порцию). Поскольку в базе рецептов нет готовых цифр по калориям и белку, дайте разумную оценку на основе ингредиентов рецепта. Затем сравните содержание белка в каждом рецепте с дневной нормой по белку, полученной из загруженного источника (Protein daily_g), и заполните столбец Meets_Guidelines:
- значение "Да" (или "Yes"), если Protein_g больше или равен дневной норме по белку;
- значение "Нет" (или "No"), если Protein_g меньше дневной нормы по белку.

С помощью терминала создайте и запустите Python-скрипт под названием cook_catering_processor.py в рабочей директории. Скрипт должен читать собранные данные из создаваемых вами JSON-файлов, выполнять анализ и выводить результат в файл cook_catering_results.json.

Создайте Excel-файл под названием Catering_Cost_Report.xlsx с тремя листами.

Первый лист, Data_Analysis, должен содержать по одной строке на рецепт со столбцами: Recipe (название рецепта), Category (категория), Calories (оценка калорийности), Protein_g (оценка содержания белка в граммах) и Meets_Guidelines (Да/Нет — соответствует ли рецепт дневной норме по белку). Отсортируйте строки по алфавиту по столбцу Recipe.

Второй лист, Metrics, должен быть таблицей со столбцами Metric и Value и содержать сводные показатели, рассчитанные по листу Data_Analysis, включая как минимум:
- Total_Recipes — количество рецептов;
- Avg_Calories — средняя калорийность по всем рецептам, округлённая до целого;
- Recipes_Meeting_Guidelines — количество рецептов со значением "Да" в столбце Meets_Guidelines;
- Avg_Protein — среднее содержание белка по всем рецептам.

Третий лист, Recommendations, должен содержать список приоритетных действий со столбцами Priority (номер приоритета) и Action (описание действия). Укажите не менее двух рекомендаций на основе проведённого анализа (например, добавить больше высокобелковых блюд).

Также создайте Word-документ под названием Catering_Cost_Analysis.docx с тремя разделами: краткое резюме (executive summary), ключевые выводы (key findings) и рекомендации (recommendations). Заголовки разделов можно писать по-русски или по-английски.

Важно: при работе с инструментом Word передавайте АБСОЛЮТНЫЙ путь к файлу .docx внутри рабочей директории, так как инструмент Word может не учитывать текущую рабочую директорию.
