Координатору программы кулинарного искусства нужно подготовить комплексный обзор курсов, объединяющий данные о студенческих заданиях с анализом базы рецептов. Обзор охватывает два курса по биохимии и биоинформатике из системы управления обучением (course IDs 3 и 4) и информацию о рецептах из кулинарной базы (русская база рецептов kulinar).

Начните с изучения методички по оцениванию (assessment rubric) и файла конфигурации курса (course config) в рабочей директории.

По данным курсов выгрузите все задания из обоих курсов, включая название задания, количество баллов и число студенческих сдач. По данным рецептов получите сводку по всем категориям рецептов с указанием количества рецептов и среднего уровня сложности по каждой категории. Категории и сложность берите из кулинарной базы kulinar (например, через инструменты получения списка рецептов и сведений по отдельному рецепту); сложность (difficulty) — это числовое поле рецепта.

Запустите аналитические скрипты, которые вычисляют сводную статистику: общее число заданий по каждому курсу, среднее количество баллов на задание, суммарное число сдач, а также разбивку категорий рецептов по количеству и сложности.

Создайте Excel-файл с названием Nutrition_Course_Assessment.xlsx с четырьмя листами. Первый лист "Student_Assignments" должен содержать столбцы Course_Name, Assignment_Name, Points_Possible и Submission_Count, перечисляя все задания из обоих курсов, отсортированные по Course_Name, затем по Assignment_Name. Второй лист "Recipe_Analysis" должен содержать столбцы Category, Recipe_Count и Avg_Difficulty (округлённую до 1 знака после запятой), перечисляя все категории рецептов, отсортированные по Category. Третий лист "Course_Summary" должен содержать столбцы Metric и Value со следующими строками: Total_Courses (значение 2), Course_3_Assignments (число заданий в курсе 3), Course_4_Assignments (число заданий в курсе 4), Course_3_Total_Submissions (суммарное число сдач по всем заданиям курса 3), Course_4_Total_Submissions (суммарное число сдач по всем заданиям курса 4), Total_Recipe_Categories (число категорий), Total_Recipes (общее количество рецептов). Четвёртый лист "Grading_Schedule" должен содержать столбцы Date, Time, Course_Name, Assignment_Name и Duration_Minutes, планируя сессии проверки для каждого задания курса 3, начиная с 10 марта 2026 года в 09:00, по одному заданию на сессию продолжительностью 60 минут, по одной сессии в день и только по будням, отсортированные по дате.

Также напишите документ Word с названием Assessment_Feedback.docx, который кратко описывает структуру заданий курсов, обсуждает распределение категорий рецептов и уровней сложности, а также даёт рекомендации о том, как база рецептов могла бы дополнить учебную программу курсов. При работе с инструментом Word передавайте АБСОЛЮТНЫЙ путь к файлу .docx внутри рабочей директории агента.

Наконец, создайте события в календаре для каждой сессии проверки из листа Grading_Schedule, чтобы координатор мог их отслеживать.
